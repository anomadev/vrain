Las herramientas de salida en PHP (`echo` y `print`) son constructos del lenguaje diseñados para mostrar texto y variables en pantalla, donde `echo` admite múltiples argumentos y no retorna valor, mientras que `print` acepta sólo uno y siempre retorna `1`, lo que le permite formar parte de expresiones. Para depurar y examinar el contenido de variables complejas, PHP ofrece `var_dump`, que muestra tipo y valor de forma recursiva, y `print_r`, que presenta la información en un formato más legible para humanos, opcionalmente capturable mediante un parámetro de retorno. Conocer sus diferencias y saber cuándo usar cada uno te permitirá depurar eficazmente y mantener un código más claro.
## `echo`
### Definición y características
`echo` es un constructo del lenguaje (no función) que imprime una o más expresiones convertidas a cadena sin añadir espacios ni salto de línea adicionales.
- **Sintaxis**: `echo expr1, expr2, ...;`
- **Retorno**: Ninguno (`void`)
- **Atajo corto**: `<?= expr =>` equivale a `<?php echo expr ?>`
- **Conversión de tipos**: Cualquier valor no `string` se fuerza a cadena incluso con `declare(strict_types=1)`
### Ejemplos prácticos

```php
<?php

echo "Hola", " mundo";   // Hola mundo
echo 1 + 2;              // 3 (coerción implícita)

?>
```

```php
<!doctype html>
<html>
	<body>
		<?= "<p>Saludo con atajo</p>" ?>  <!-- equivalente a echo -->
	</body>
</html>
```

## `print`
### Definición y características
`print` también es un constructo que muestra una única expresión convertida a cadena.
- Sintaxis: `print expr;`
- Retorno: Siempre `1`, por lo que puede utilizarse en expresiones condicionales.
- Conversión de tipos: Igual que `echo`, fuerza a `string` ([[2.4 Type Casting]]).
### Ejemplos prácticos
```php
<?php

if (print "Hola") {
	// Siempre entra aquí porque print retorna 1
}

?>
```

```php
<?php

print "Valor: " . (2 * 3); // Valor: 6

?>
```
*permite usarse en expresiones, p. ej. `($cond) ? print 'A' : print 'B';`*
## `var_dump`
### Definición y características
`var_dump` es una función para depuración que muestra información estructurada (tipo y valor) de una o varias expresiones, recorriendo recursivamente `arrays` ([[3.1 Arrays]]) y objetos ([[11.1 Classes and Objects]]).
- Sintaxis: `var_dump(mixed $value, mixed ...$values): void`
- Salida: Imprime directamente al navegador o consola
- Profundidad: Explora todas las propiedades públicas, privadas y protegidas, salvo que el objeto defina `__debugInfo()`
### Ejemplos prácticos
```php 
<?php

$a = [1, 2, ["x", "y"]];
var_dump($a);
/*
array(3) {
  [0]=> int(1)
  [1]=> int(2)
  [2]=> array(2) {
    [0]=> string(1) "x"
    [1]=> string(1) "y"
  }
}
*/

?>
```

```php
<?php

var_dump(3.14, true);
// float(3.14)
// bool(true)

?>
```
*Envuelve en `<pre>` para mejorar legibilidad*
## `print_r`
### Definición y características
`print_r` muestra información de una variable en formato legible para humanos, mostrando claves y valores de `arrays` ([[3.1 Arrays]]) y objetos ([[11.1 Classes and Objects]]); opcionalmente puede devolver la salida en lugar de imprimirla si se le pasa `true` como segundo parámetro
- Sintaxis: `print_r(mixed $value, bool $return = false): string|true`
- Salida:
	- Por defecto imprime y retorna `true`
	- Si `$return === true`, retorna la cadena en lugar de imprimir
### Ejemplos prácticos
```php
<?php

$arr = ['a' => 'manzana', 'b' => 'banana'];
print_r($arr);
/*
Array
(
    [a] => manzana
    [b] => banana
)
*/

?>
```

```php
<?php

$txt = print_r($arr, true); // $txt contiene la representación

?>
```

### Comparativa y buenas prácticas
- **Rendimiento**: `echo` es marginalmente más rápido que `print`
- **Depuración**:  preferentemente `var_dump` cuando necesites conocer tipos exactos; usa `print_r` para estructuras más limpias.
- **Evitar**: `print_r` en producción sin condicional (`if(DEBUG)`) para no exponer datos sensibles.
- **Tip**: Combina `var_dump` con `die()` en entornos de desarrollo para hacer *`dump and die`* y detener la ejecución en el punto de inspección.
## Notas
---
- Evitar mezclar depuración y lógica de negocio; usar siempre flags de entorno para habilitarla solo en desarrollo.
