Las constantes en PHP son identificadores para valores simples que no pueden cambiar durante la ejecución del script (las *magic constants* son la excepción pues varían según su contexto). Se definen mediante la función `define()` o la palabra clave `const`, siendo esta última evaluada en tiempo de compilación y con restricciones sintácticas más estrictas. Su ámbito es siempre global y no llevan el prefijo `$` ([[2.1 Basic PHP Syntax]]), lo que simplifica su uso en todo el código. Además de las constantes globales ([[8.2 Super Global Variables]]), PHP permite definir constantes de clase que son estáticas, de solo lectura y accesibles con la sintaxis `ClassName::CONSTANT`. A partir de PHP 5.6 se introdujeron **arrays contantes** que no pueden modificarse tras su definición. Las *magic constants* (por ejemplo, `__LINE__` o `__FILE__`) se resuelven en tiempo de compilación y proporcionan información del entorno de ejecución. Se recomienda nombrar las constantes en mayúsculas con guiones bajos para mejorar la legibilidad y evitar colisiones. PHP también ofrece numerosas **constantes predefinidas** (como `PHP_VERSION` o `PHP_OS`) útiles para consultar detalles del propio intérprete y del sistema operativo.
### Definición
Las contantes en PHP son un identificador (nombre) para un valor simple que permanece inmutable durante toda la ejecución del script.
- No emplea el prefijo `$` y su nombre debe comenzar con letra o guión bajo, seguido de letras, dígitos o guiones bajos.
- Es sensible a mayúsculas (`FOO` != `foo`) y por convención, se escriben en `uppercase`.
### Conceptos clave
- **Definición de constantes**
	- Con `define("NOMBRE", valor)`, evaluada en tiempo de ejecución
	- Con `const NOMBRE = valor;`, evaluando en tiempo de compilación y usable dentro de clases.
- **Ámbito global**: accesibles desde cualquier parte del script, sin importar funciones o clases.
- **Constantes de clase**:
	- Declaradas con `const` dentro de una clase, son implícitamente `static` y `read-only`
	- Desde PHP 7.1 pueden tener `public`, `protected` o `private` ([[11.4 Access Specifiers]])
- **Arrays constantes**: permitidos desde PHP 5.6; no se pueden modificar tras su definición.
- **Magic constants**: especiales, cambian su valor según el contexto (`__LINE__`, `__FILE__`, etc.) y se resuelven al compilar.
- **Constantes predefinidas**: integradas en el núcleo y extensiones (p. ej., `PHP_VERSION`, `PHP_OS`, `PHP_INT_MAX`).
- **Convención de nombres**: `uppercase` con guiones bajos para mejorar legibilidad y evitar colisiones.
### Ejemplos prácticos
##### Definir constantes con `define()` y `const`
```php
<?php

define("MAX_USUARIOS", 100);
const TIEMPO_LIMITE  = 60;

?>
```
##### Acceso global
```php
<?php

echo MAX_USUARIOS; // 100

?>
```
##### Constante de clase
```php
class Config {
	public const VERSION = "1.2.3";
}

echo Config::VERSION;
```
##### Array constante
```php
<?php

const ROLES = ["admin", "editor", "usuario"];
echo ROLES[1];

?>
```
##### Magic constant
```php
<?php

echo "Esto está en la línea " . __LINE__;
// Imprime el número de la línea actual

?>
```
##### Constante predefinida
```php
<?php

echo "PHP versión: " . PHP_VERSION;

?>
```
