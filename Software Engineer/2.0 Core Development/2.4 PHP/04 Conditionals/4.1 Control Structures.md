Las estructures condicionales en PHP permiten ejecutar distintos fragmentos de código según el resultado de expresiones que evalúan a verdadero o falso. El bloque `if` inicia la condición, `else` cubre el caso contrario, y `elseif` (o `else if`) encadena múltiples comparaciones ([[4.2 if-else]]). PHP también ofrece una sintaxis alternativa usando dos puntos (`:`) y `endif;` para mezclar con bloques HTML de forma clara. Para comparaciones múltiples sobre la misma expresión, el `switch` ([[4.3 switch]]) es más limpio y puede ser más eficiente, recordando siempre usar `break` para evitar el `fall-through` no deseado. Finalmente, el operador ternario (`?:`) ([[4.4 Null Coalescing Operator]]) condensa un `if...else` simple en una sola línea, mejorando la legibilidad en asignaciones rápidas.
### Definición
- `if (expr) {...}`: ejecuta el bloque si `expr` evalúa a `true`.
- `else {...}`: opcional; se ejecuta si la condición del `if` (y de cualquier `elseif`) fue `false`.
- `elseif (expr) {...}` (o `else if`): añade una segunda comprobación que solo se evalúa si la anteriores fueron `false`.
- **Sintaxis alternativa**: usar `if (expr): ... endif;` (y análogamente `elseif:`, `else:`, `endswitch;`) para mezclar con HTML sin llaves.
- `switch (valor) { case X: ... break; default: ... }`: compara `valor` contra cada `case` usando comparación laxa, ejecuta desde el primer `case` coincidente hasta encontrar un `break`.
### Conceptos clave
- **Evaluación booleana**: PHP convierte expresiones a `true` o `false` según sus reglas internas.
- **Orden y alcance**: un `else` siempre se asocia al `if` más cercano; usar llaves o sintaxis alternativa para evitar ambigüedades.
- **Corto circuito en `&&` / `||`**: las expresiones lógicas se evalúan de izquierda a derecha y paran al primer resultado determinante.
- **Cascada de `elseif`**: solo se ejecuta el bloque de la primera condición verdadera.
- **Sintaxis alternativa**: reemplaza `{}` por `:` y `endif;`/`endfor;`/`endforeach;` para legibilidad en plantillas.
- **`switch` vs `if...elseif`**: el `switch` evalúa la expresión una sola vez y puede ser más rápido en bucles intensivos.
- **Operador ternario**: `condicion ? valor_true : valor_false` para expresiones simples.
### Ejemplos prácticos
##### `if` básico
```php
<?php

$a = 5; $b = 3;
if ($a > $b) {
    echo "a es mayor que b";
}
```
##### `if...else`
```php
<?php

if ($a > $b) {
    echo "a > b";
} else {
    echo "a ≤ b";
}

```
##### `if...elseif...else`
```php
<?php

if ($a > $b) {
  echo "a > b";
} elseif ($a == $b) {
  echo "a = b";
} else {
  echo "a < b";
}

```
##### Ternario
```php
<?php

$max = ($a > $b) ? $a : $b;
echo "El mayor es $max";

```
##### Sintaxis alternativa en plantillas
```php
<?php if ($a == 5): ?>
  <p>a vale 5</p>
<?php elseif ($a == 6): ?>
  <p>a vale 6</p>
<?php else: ?>
  <p>a no vale ni 5 ni 6</p>
<?php endif; ?>

```
##### `switch` con `break`
```php
<?php

switch ($i) {
  case 0:
    echo "i = 0"; break;
  case 1:
    echo "i = 1"; break;
  default:
    echo "i no es 0 ni 1";
}

```
##### `switch` sin `break`
```php
<?php

switch ($i) {
  case 0:
  case 1:
    echo "i es 0 o 1"; break;
  case 2:
    echo "i = 2";
}

```
## Notas
---
- Usar siempre `{}` en `if...else` para evitar el "`dangling else`".
- Utilizar el ternario en asignaciones cortas, pero evitar anidar más de dos niveles para no reducir legibilidad.
