La sentencia `switch` en PHP permite comprar una misma expresión contra múltiples valores forma más legible que una serie de `if...else` anidados. Evalúa la expresión una sola vez, luego recorre cada `case` aplicando una comparación laxa (`==`) hasta encontrar coincidencia; al omitir el `break`, continúa la ejecución en el siguiente `case` (`fall-through`) hasta topar con un `break` o el fin del bloque. Incluye un `default` opcional que captura cualquier valor no coincidente, y admite una sintaxis alternativa con dos puntos `switch ...: ... endswitch` para mezclar con HTML. Para comparaciones complejas puede usarse `switch(true)` y expresiones booleanas en los `case`, o bien optar por el nuevo operador `match` en PHP 8 ([[1.1 What is PHP]]), que ofrece semántica más estricta sin necesidad de `break`.
### Definición
La estructura **`switch (expresión) { case valor: … break; … default: … }`** compara la **expresión** con cada **valor** de `case` usando **comparación laxa** (`==`) y ejecuta el bloque asociado al primer match, deteniendo el flujo con `break` o continuando en `fall‑through` si no se incluye. El `default` se ejecuta si ningún `case` coincide y puede ubicarse en cualquier posición, aunque por convención va al final. PHP también soporta **sintaxis alternativa**:

```php
<?php
switch ($valor):
    case 1:
        echo "uno";
        break;
    default:
        echo "otro";
endswitch;
```
### Conceptos clave
- **Evaluación única**: la **expresión del `switch`** se evalúa **solo una vez**, a diferencia de un `if…elseif` donde cada condición se calcula de nuevo.  
- **Comparación laxa**: los `case` usan `==`, por lo que `"1"` coincide con `1`.  
- **`break` y `fall‑through`**: si falta `break`, la ejecución “cae” en el siguiente `case`.  
- **`default`**: caso por defecto para valores no coincidentes; su omisión deja sin efecto el `switch` para esos valores.  
- **Sintaxis alternativa**: permite `switch …: … endswitch;` para plantillas mixtas con HTML.  
- **`continue` en `switch`**: dentro de loops, `continue` actúa como `break`; usar `continue 2` para saltar al siguiente ciclo del bucle externo.  
- **Rendimiento**: en bucles intensivos, `switch` puede ser más rápido que varios `if…elseif` porque evita reevaluaciones repetidas.  
- **`switch(true)`**: truco para evaluar múltiples expresiones booleanas complejas dentro de `case`.
### Ejemplos prácticos
##### `switch` simple
```php
<?php
$i = 1;
switch ($i) {
	case 0:
	    echo "i = 0";
        break;
    case 1:
        echo "i = 1";
        break;
    default:
        echo "i no es 0 ni 1";
}

```
##### `fall-through` intencional
```php
<?php

$i = 0;
switch ($i) {
    case 0:
    case 1:
    case 2:
        echo "i está entre 0 y 2";
        break;
    default:
        echo "i es mayor que 2";
}
```
##### Sin `break` (caída completa)
```php
<?php

$i = 0;
switch ($i) {
    case 0:
        echo "i = 0";
    case 1:
        echo " o i = 1";
    case 2:
        echo " o i = 2";
}

// Salida: "i = 0 o i = 1 o i = 2"
```
##### `switch(true)` para condiciones
```php
<?php

$edad = 20;
switch (true) {
    case $edad < 13:
        echo "Niño"; break;
    case $edad < 18:
        echo "Adolescente"; break;
    default:
        echo "Adulto";
}

```
##### Sintaxis alternativa
```php
<?php switch ($a): ?>
  <?php case 1: ?>
    <p>a vale 1</p>
    <?php break; ?>
  <?php default: ?>
    <p>a no vale 1</p>
<?php endswitch; ?>
```
## Notas
---
- Siempre incluir `break` salvo que el `fall-through` sea intencional
- `switch(true)` para validaciones por rangos
- Revisar `match` en PHP 8 cuando esperemos retornos de valores en lugar de bloques de código
