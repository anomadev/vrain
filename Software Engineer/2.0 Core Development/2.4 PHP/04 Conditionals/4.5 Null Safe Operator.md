### Definición
El operador `nullsafe` (`?->`) introducción en PHP 8.0 ([[1.1 What is PHP]]) permite realizar llamadas a métodos o acceder a propiedades en cadenas de objetos que podrían ser `null`. Cuando el operando izquierdo de `?->` evalúa a `null`, la ejecución de toda la cadena se detiene y el resultado es `null`, sin lanzar errores de "llamada a método en `null`".
### Conceptos clave
- **Sintaxis y semántica**: funciona igual que `->` si el operando no es `null`, devolviendo el valor de propiedad o el resultado del método; de lo contrario retorna `null`.
- **Cortocircuito completo**: Ni los argumentos del método ni los accesos subsiguientes se evalúan si algún elemento en la cadena es `null`.
- **Versionamiento**: Implementado en la versión 8.0.0 de PHP.
- **RFC y diseño**: Definición en el PHP RFC "`Nullsafe operator`" (RFC 5619), que especifica `full short circuiting` y uso de `?->` para acceder a propiedades/métodos con seguridad.
- **Limitaciones**: No se usa puede ser en contexto de asignaciones, `unset()` o referencias (`&`) sobre el resultado de una cadena `?->`.
- **Compatibilidad con otras características**: No es combinable con la sintaxis de `callables` de primera clase (`$obj?->method(...)`).
### Ejemplos prácticos

```php
<?php

// Asigna país solo si todos los niveles existen
$country = $session?->user?->getAddress()?->country;

```

```php
<?php

// Al ser $session null, nada se evalúa y $country será null
$session = null;
$country = $session?->user?->getAddress()?->country; //null
```

```php
<?php

// Evita llamadas a funciones costosas si previa es null
$foo = null;
$foo?->bar(expensiveFunction())?->baz(); // ni bar ni baz se ejecutan

```

```php
<?php

// No permitido: nullsafe en el lado izquierdo de asignación
$foo?->bar = 'valor'; // Error de parseo

```
## Notas
---
- Usar `?->` para encadenar llamadas seguras en modelos `Eloquent` de `Laravel`.
- Evitar bloques `if` ([[4.2 if-else]]) anidados al verificar autenticación: `$user?->profile?->email`.
- No sustituye a validaciones de error; para flujos de trabajo críticos usar excepciones.
