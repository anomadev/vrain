En PHP existen cuatro tipos principales de bucles para iterar código: `while`, que evalúa la condición al inicio de cada iteración; `do-while`, que garantiza al menos una ejecución antes de comprobar la condición; `for`, con tres expresiones (inicialización, condición y avance) que se evalúan en momentos diferenciados; y `foreach`, diseñado específicamente para recorrer `arrays` y objetos iterables de manera sencilla. Cada bucle ofrece además una sintaxis alternativa basada en dos puntos (`while...endwhile`, `for...endfor`, etc.) útil cuando se mezcla PHP con HTML. Asimismo , funciones como `break` y `continue` permiten controlar el flujo dentro de estos bucles, interrumpiendo o saltando iteraciones según se requiera.
### Definición
- `while (expr) {...}` ejecuta el bloque repetidamente mientras `expr` evalúe a `true`, comprobando la condición antes de cada iteración.
- `do {...} while (expr);` ejecuta primero el bloque al menos una vez y luego evalúa `expr` al final de cada iteración, asegurando al menos una ejecución.
- `for (expr1; expr2; expr3) {...}` realiza `expr1` una sola vez al inicio, luego por cada iteración comprueba `expr2` para continuar o detenerse, y al final de cada ciclo ejecuta `expr3`.
- `foreach (iterable as $value)` y `foreach (iterable as $key => $value)` recorren `arrays` o `Trasversable`, asignando sucesivamente el valor (y opcionalmente la clave) al bloque interno sin afectar el puntero interno del `array`.
### Conceptos clave
- **Comprobación de condición**
	- En `while` la condición se revisa **antes** de cada ejecución, pudiendo no ejecutarse nunca si es `false` desde el inicio.
	- En `do-while` se comprueba **después**, garantizando la primera pasada.
- **Ciclo controlado vs no controlado**
	- Un `for` puede convertirse en un bucle infinito si su condición está vacía (`for (;;)`), usando `break` para salir cuando convenga.
- **Sintaxis alternativa**
	- Todas las estructuras de bucle admiten `...expr: ... end...;` para mezclarse con plantillas HTML sin llaves.
- **Recorrido de colecciones**
	- `foreach` es la forma más segura y legible de iterar `arrays` y objetos que implementen `Traversable`, lanzando error si se usa en otro tipo de dato.
- **Control de flujo interno**
	- `break` detiene el bucle inmediatamente; `continue` salta a la siguiente iteración, útil para omitir casos específicos sin salir del bucle.
### Ejemplos prácticos
##### `while` con sintaxis clásica y alternativa
```php
<?php

$i = 1;
while ($i <= 5) {
	echo $i++;
}

// equivalente en sintaxis alternativa
$i = 1;
while ($i <= 5):
	echo $i++;
endwhile;

```
##### `do-while` asegurando una pasada
```php
<?php

$n = 0;
do {
	echo "Número: $n\n";
} while ($n > 0);
// Imprime "Número: 0" una vez

```
##### `for` con inicialización, condición y avance
```php
<?php

for($i = 1; $i <= 5; $i++) {
	echo "Iteración $i\n";
}

```
##### `for` infinito y `break`
```php
<?php

for (;;) {
	$x++;

	if ($x > 3) {
		break;
	}
	echo "$x ";
}
// Salida: 1 2 3

```
##### `foreach` con clave y valor
```php
<?php

$edades = ["Ana" => 25, "Luis" => 30];
foreach ($edades as $nombre => $edad) {
	echo "$nombre tiene $edad años\n";
}

```
##### Desestructuración en `foreach` (PHP 7.1+)
```php
<?php

$pares = [[1,2], [3,4]];
foreach ($pares as [$a, $b]) {
	echo "$a + $b = " . ($a + $b) . "\n";
}

```
## Notas
---
- Usar `foreach` siempre que recorra colecciones porque evita errores de índice y mejora la legibilidad.
- En bucles grandes, extraer `count($array)` a una variable para no recalcular el tamaño en cada iteración.
