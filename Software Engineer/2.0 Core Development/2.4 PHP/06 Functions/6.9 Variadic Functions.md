Las **Funciones Variádicas** en PHP permiten que una función acepte un número **ilimitado** de argumentos usando el operador de `splat` (`...`), al tiempo que simplifican el manejo de listas de parámetros sin recurrir a las antiguas funciones `func_get_args()` [PHP](https://www.php.net/manual/en/migration56.new-features.php?utm_source=chatgpt.com). Introducidas en PHP 5.6, capturan todos los argumentos adicionales en un **array**, que puede tiparse, pasarse por referencia o recibir valores por defecto como en cualquier parámetro normal [PHP](https://www.php.net/manual/en/migration56.new-features.php?utm_source=chatgpt.com). Paralelamente, PHP sigue apoyando las funciones legado `func_num_args()`, `func_get_arg()` y `func_get_args()` para compatibilidad tras bambalinas [PHP](https://www.php.net/manual/en/function.func-get-args.php?utm_source=chatgpt.com). Desde PHP 8.0 se añadieron **`trailing commas`** en la declaración de parámetros y la posibilidad de **reemplazar** múltiples parámetros fijos por un parámetro variádico en la **herencia** de métodos [PHP](https://www.php.net/manual/en/migration80.new-features.php?utm_source=chatgpt.com). Para meta‑programación, la reflexión ofrece `ReflectionParameter::isVariadic()` para detectar programáticamente parámetros `variádicos` [PHP](https://www.php.net/manual/en/reflectionparameter.isvariadic.php?utm_source=chatgpt.com).
### Definición
Una función variádica se declara anteponiendo (`...`) al último parámetro de la firma, de modo que:
```php
function example($fijo, ...$extras) { ... }
```
- `$extras` se convierte automáticamente en un **array** con todos los argumentos sobrantes.
- Solo **el último parámetro** de lista puede ser variádico; ubicar (`...`) en otra posición produce un error.
- Internamente, la sintaxis (`...`) reemplaza la necesidad de `func_get_args()` y `func_num_args()`.
### Conceptos clave
##### Sintaxis y Captura
- `function f($a, ...$args)` captura en `$args` todos los parámetros extras en un **array** [PHP](https://www.php.net/manual/en/migration56.new-features.php?utm_source=chatgpt.com).
- Compatibilidad con **tipado**: `function sum(int ...$nums): int` obliga a que cada argumento sea `int` y retorna un `int` [PHP Tutorial](https://www.phptutorial.net/php-tutorial/php-variadic-functions/?utm_source=chatgpt.com).
- **Paso por referencia**: `function incAll(&...$vals)` (aunque raro) permite modificar el array interno, aunque se recomienda pasar el array completo por referencia [PHP](https://www.php.net/manual/en/migration56.new-features.php?utm_source=chatgpt.com).
##### Funciones Legado
- `func_num_args()`: retorna el conteo de argumentos pasados [PHP](https://www.php.net/manual/en/function.func-num-args.php?utm_source=chatgpt.com).
- `func_get_arg($i)`: obtiene el argumento en la posición `$i` [PHP](https://www.php.net/manual/en/function.func-get-arg.php?utm_source=chatgpt.com).
- `func_get_args()`: devuelve un **array** con todos los argumentos [PHP](https://www.php.net/manual/en/function.func-get-args.php?utm_source=chatgpt.com).
##### Unpacking de Argumentos
- Con **`splat`** en llamadas: `f(...$array)` expande los elementos del array como argumentos posicionales [PHP](https://www.php.net/manual/en/migration56.new-features.php?utm_source=chatgpt.com).
##### Mejoras en PHP 8.x
- **`Trailing comma`** en declaración de parámetros: `function foo($a, $b, ...$c,) {}` [PHP](https://www.php.net/manual/en/functions.arguments.php?utm_source=chatgpt.com).
- **Herencia flexible**: en clases hijas, un método puede reemplazar parámetros fijos por un variádico si las firmas son compatibles [PHP](https://www.php.net/manual/en/migration80.new-features.php?utm_source=chatgpt.com).
- **Detección de variádico**: `ReflectionParameter::isVariadic()` retorna `true` si el parámetro fue declarado con `...` [PHP](https://www.php.net/manual/en/reflectionparameter.isvariadic.php?utm_source=chatgpt.com).
### Ejemplos prácticos
##### Función que suma cualquier cantidad de números
```php
<?php

function sum(int ...$nums): int {
	return array_sum($nums);
}

echo sum(1,2,3,4); // 10

```
##### Alternativa legado con `func_get_args()`
```php
<?php

function legacySum() {
	return array_sum(func_get_args());
}

echo legacySum(5, 10, 15); // 30

```
##### Unpacking de argumentos
```php
<?php

$vals = [2, 4, 6];
echo sum(...$vals); // 12

```
##### Tipado y recuento interno
```php
<?php

function info(string $label, ...$items) {
	printf("%s (%d items): %s\n", $label, count($items), implode(', ', $items));
}

info('Frutas', 'manzana', 'pera', 'uva');
// Frutas (3 items): manzana, pera, uva

```
##### Remplazo de parámetros en herencia (PHP 8.0+)
```php
<?php

class A { function m($a, $b) {} }
class B extends A { function m(...$all) {} }

```
## Notas
--- 
- Usar variádicos tipados (`int ...$vals`) para garantizar la integridad de datos en funciones de agregación.
- Usar siempre unpacking cuando los datos ya existen en un array, evitando bucles manuales.
