En Laravel, todas las opciones de configuración viven en archivos dentro del directorio `config`, donde encontrarás desde las conexiones de base de datos hasta la URL de tu aplicación y clave de cifrad. La plataforma facilita la separación de valores por entrono mediante archivos `.env` gestionados con la librería `DotEnv`, y ofrece funciones (`env()`, `config()`) y `facades` (`Config`, `App`) para acceder y modificar estas configuraciones en tiempo de ejecución. Además, para optimizar el arranque en producción, Laravel permite `cachear` todos los archivos de configuración en un solo (`php artisan config:cache`) y publicar configuraciones adicionales que no vienen por defecto en `config/`. Por último, encontrarás opciones para controlar el `modo debug` (`APP_DEBUG`) y el modo mantenimiento, con comandos `Artisan` (`down`, `up`) que ayudan a gestionar despliegues con vistas pre-renderizadas y `bypass` mediante `tokens`.
### Definición 
Los archivos de configuración de Laravel residen en la carpeta `config` de tu proyecto, cada uno documentado con sus respectivas opciones.
El archivo `.env` (copiado de `.env.example` al instalar) define variables de entorno que luego son leídas por la función `env()` dentro de estos archivos de configuración. 
Para entornos múltiples, Laravel puede cargar un archivo `.env.[APP_ENV]` si `APP_ENV` está definido o se pasa `--env` al `CLI`.
### Conceptos clave
- **`DotEnv` y archivos `.env`**: Laravel usa la librería **`DotEnv`** para cargar variables definidas en `.env` o `.env.[APP_ENV]` antes de `bootstrap`.
- **Tipos en `env()`**: Aunque todo llega como `string`, `env()` convierte valores como `true`, `false`, `null` y `empty` a sus tipos correspondientes.
- **Acceso a valores**: Usa el `helper` `config('file.option', 'defaultValue')` o el `facade` `Config::get('file.option')` para leer cualquier opción.
- **Modificación en `runtime`**: Con `config(['app.timezone' => 'UTC'])` o `Config::set('app.timezones', 'UTC')` cambias valores dinámicamente.
- **Cache de configuración**: El comando `php artisan config:cache` une todos los archivos en uno solo, acelerando la carga en producción.
- **Publicación de `config`**: Con `php artisan config:publish --all` puedes copiar al directorio `config/` cualquier archivo de configuración que no venga por defecto.
- **`Debug Mode`**: La opción `debug` en `config/app.php` refleja `APP_DEBUG` del `.env`, mostrando errores ricos en desarrollo pero nunca en producción.
- **`Maintenance Mode`**: Usando `php artisan down` y `php artisan up` se activa/desactiva un `middleware` que arroja un 503 con vista de mantenimiento, configurable con `--reflesh`, `--retry`, `--secret` y pre-renderizado de la plantilla.
### Ejemplos prácticos
##### Mostrar secciones de la configuración
```bash
php artisan about --only=environment
php artisan config:show database
```
Presenta un resumen por sección o un archivo específico.
##### Recuperar con valor por defecto
```php
$zona = config('app.timezone', 'UTC');
```
Si `app.timezone` no existe, retorna `UTC`.
##### Asignar en `runtime`
```php
config(['mail.from.address' => 'no-reply@example.com']);
```
Cambia la dirección de envío sin reiniciar la aplicación.
##### `Caching` de la configuración
```bash
php artisan config:cache
php artisan config:clear
```
Genera un único archivo compilado y lo limpia cuan sea necesario.
##### Activar modo mantenimiento con `token` y `pre-render`
```bash
php artisan down --secret="abc123" --render="errors:503"
```
Emite una ruta oculta y un `view` pre-renderizado para cero dependencia.

## Notas
---
- Mantener siempre el `.env.example` actualizado con placeholders claros para onboarding de nuevos desarrolladores.
- Usar `config()` en lugar de `env()` fuera de los archivos `config/` para evitar problemas al hacer `caching`.
