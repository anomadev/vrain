Laravel ofrece dos enfoques principales para desarrollar la capa de presentación de la aplicación: usando PHP (con Blade y Livewire) o aprovechando `frameworks` JavaScript modernos como `Vue` o `React` mediante `Inertia` [laravel.com](https://laravel.com/docs/12.x/frontend). Con PHP y Blade puedes seguir una metodología clásica de servidor, donde cada petición devuelve un HTML completo renderizado en el servidor [laravel.com](https://laravel.com/docs/12.x/frontend). Si necesitas interactividad dinámica sin abandonar PHP, `Livewire` permite crear componentes reactivos usando atributos `wire:` y sin escribir JavaScript complejo [laravel.com](https://laravel.com/docs/12.x/frontend). Para una SPA con `Vue` o `React`, `Inertia` actúa como puente, reutilizando rutas y controladores de Laravel y renderizando páginas `frontend` en JavaScript dentro de un mismo repositorio [laravel.com](https://laravel.com/docs/12.x/frontend). Independientemente de tu elección, Laravel utiliza `Vite` para empacar CSS y JavaScript de manera eficiente, con HMR rápido en desarrollo y compilación optimizada para producción [laravel.com](https://laravel.com/docs/12.x/frontend).

---
## Using PHP
### PHP y Blade
Laravel conserva la potencia de la vistas basadas en servidor con `Blade`, su motor de plantillas ligero y expresivo. `Blade` ofrece directivas como `@foreach` y `{{}}` para iterar y mostrar datos de forma concisa, reemplazando los tradicionales `<?php echo ?>`. Al renderizar formularios o acciones de usuario, el servidor devuelve un documento HTML completo, ideal para aplicaciones con navegación basada en recarga de página.
### Livewire
`Laravel Livewire` permite crear componentes PHP que mantienen estado y respondan a eventos del `frontend` sin escribir JavaScript explícito. Un componente `Livewire` define propiedades públicas y métodos en PHP, y su vista Blade puede incluir atributos como `wire:click="increment"` para invocar esos métodos de forma reactiva. Por ejemplo, un contador `Livewire` incrementa `$count` en el servidor y re-renderiza solo el fragmento necesario logrando una experiencia de usuario fluida.
### Starter Kits (PHP)
Laravel ofrece un **`starter kit Livewire`** que `scaffoldea` autenticación, al tiempo que instala y configura `Alphine.js` para complementar con JavaScript mínimo donde se requiera.

## Using `React` o `Vue`
### Inertia
`Inertia` actúa como un adaptador que permite usar controladores de Laravel para renderizar página en `React` o `Vue`, pasando datos directamente como `props` a tus componentes JS. En el controlador, en lugar de `return view()`, usas `Inertia::render('users/show', ['user' => $user])` y en tu componente `Vue/React` recibes `user` como propiedad. Este enfoque mantiene rutas y autenticación en Laravel, eliminando la necesidad de `APIs REST` y repositorios separados. `Inertia` también soporta `server-side rendering` y se integra fácilmente con hosting en `Laravel Cloud` o `Forge` para mantener SEO y rendimiento.
### Starter Kits (JS)
Los `starter kits Inertia` para `Vue` o `React` `scaffoldean` autenticación, rutas, `layouts` con `Tailwind CSS` y configuración `Vite`, permitiendo comenzar el proyecto `full-stack` en minutos.

---
## Bundling Assets
Laravel usa `Vite` por defecto para empacar CSS y JavaScript, proporcionando tiempos de compilación ultrarápidos y Hot Module Replacement (HMR) instantáneo durante el desarrollo. El archivo `vite.config.js` incluido en nuevas aplicaciones carga el plugin oficial de Laravel, simplificando la integración y manteniendo la configuración mínima. Para producción, `Vite` genera `assets` optimizados, minificados y versionados automáticamente, facilitando la implementación en cualquier entorno. 

## ✍️ Notas
---
- Preferible comenzar con `Blade` y `Livewire` para proyectos con lógica principalmente en el servidor.
- Usar `Inertia` cuando requiera interactividad compleja o `SPA`, pero quiere mantener `routes` y `auth` en Laravel.
- Aprovechar `Vite` y los `starter kits` para no perder tiempo configurando `bundlers` o `presets` de `React/Vue`.
