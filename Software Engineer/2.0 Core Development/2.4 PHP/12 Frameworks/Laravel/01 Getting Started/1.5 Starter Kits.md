Los Starter Kits de Laravel ofrecen plantillas listas para iniciar rápidamente tu aplicación, incluyendo rutas, controladores y vistas de registro y autenticación de usuarios [laravel.com](https://laravel.com/docs/12.x/starter-kits?utm_source=chatgpt.com). Puedes crear un proyecto nuevo con `laravel new` y elegir entre los kits oficiales de `React`, `Vue` o `Livewire` para adaptarlo a tu pila de `frontend` preferida [laravel.com](https://laravel.com/docs/12.x/starter-kits?utm_source=chatgpt.com). Cada kit integra `Tailwind CSS`, `Vite` y componentes de interfaz (`shadcn/ui` para `React/Vue`o `FluxUI` para `Livewire`), y también existen variantes potentes con `WorkOS AuthKit` para autenticar con SSO, `passkeys` y redes sociales. Los kits incluyen opciones de `Server Side Rendering` (SSR) mediante `Inertia` y comandos dedicados (`npm run build:ssr` o `composer dev:ssr`) para probar localmente el SSR. Además la comunidad mantiene kits adicionales disponibles en `Packagist`, que puede usar con `laravel new --using=vendor/kit`, y el instalador oficial te permitirá seleccionarlos directamente.
### Introducción
Laravel proporciona Starter Kits para esqueletar aplicaciones con funcionalidades de autenticación ya implementadas, evitando tener que configurar manualmente rutas, controladores o vistas de login y registro. Aunque son opcionales, facilitan la puesta en marcha de nuevos proyectos y sirven como punto de partida sólido.

---
### Crear un aplicación con un Starter Kit
Instala el instalador de Laravel con `Composer`
```bash
composer global require laravel/installer
```
Ejecuta el comando interactivo que te preguntará qué kit deseas usar:
```bash
laravel new my-app
```
Instala dependencias de `frontend` y arranca el servidor de desarrollo:
```bash
cd my-app
npm install && npm run build
composer run dev
```

---
### Kits disponibles
- `React Starter Kit`: aplica `Inertia 2`, `TypeScript`, `Tailwind` y `shadcn/ui` para `React`.
- `Vue Starter Kit`: utilizar `Inertia 2`, `Vue 3`, `Composition API`, `TypeScript` y `shadcn-vue`
- `Livewire Starter Kit`: implementa `Livewire 3`, `Laravel Volt`, `TypeScript`, `Tailwind` y `Flux UI`

---
### Personalización de Starter Kits
Cada kit ofrece archivos de configuración y vistas base que puedes modificar:
- `React` y `Vue`, adapta `resources/js/Pages` y `vite.config.js` según tus necesidades.
- `Livewire`, ajusta componentes en `app/Http/Livewire/` y estilos en `resources/views`.
- Los kits de autenticación incluyen diseños variantes (simple, `card`, `split`) que puedes cambiar editando `resources/views/components/layouts/auth.blade.php`.

---
### Autenticación con WorkOS AuthKit
Laravel 12 añade variantes de cada kit con `WorkOS AuthKit`, que provee social login, SSO y passkeys gratuitas para hasta 1 millón de usuarios activos. Para configurarlo, define en tu `.env`:
```env
WORKOS_CLIENT_ID=tu-client-id  
WORKOS_API_KEY=tu-api-key  
WORKOS_REDIRECT_URL="${APP_URL}/authenticate"
```

---
### Inertia SSR
Para habilitar `server-side rendering` con `Inertia` en `React` o `Vue`, ejecuta:
```bash
npm run build:ssr
composer dev:ssr
```
Esto compila un `bundle SSR` y arranca simultáneamente el servidor Laravel y el de `Inertia` para pruebas locales.

---
### Kits Comunitarios
Puedes usar cualquier kit disponible en `Packagist` agregando `--using`:
```bash
laravel new my-app --using=vendor/kit-name
```
Por ejemplo, hay kits para `e-commerce`, `Angular`, o generados de `boilerplate` como `tnylea/laravel-new`. Próximamente, el instalador oficial soportará listarlos automáticamente.

## ✍️ Notas
- Elige `Livewire` para lógica centrada en PHP sin aprender JS avanzado
- Usar `Inertia` + `Vue` si necesita un SPA robusta con SSR y `props` fuertemente tipado
