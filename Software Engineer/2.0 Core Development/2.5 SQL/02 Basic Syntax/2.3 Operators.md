Los operadores en SQL son símbolos o palabras reservadas que permiten realizar diversas operaciones sobre datos, como compara valores, realizar cálculos, combinar condiciones lógicas o manipular cadenas. Funcionan como los botones de una calculadora o los filtros de búsqueda, pero aplicados sobre filas y columnas de una base de datos.

---
### Conceptos Clave
1. **Operadores Aritméticos**
	- Permiten realizar operaciones matemáticas básicas (`+`, `–`, `*`, `/`, `%`).
	- Se aplican sobre columnas numéricas o en expresiones dentro de consultas.
2. Operadores de Comparación
	- Comparan dos valores y devuelven `TRUE`, `FALSE` o `UNKNOWN`:
		- `=` (igualdad), `<>` o `!=` (diferente)
	    - `<`, `>`, `<=`, `>=`
	    - `BETWEEN … AND …`, `NOT BETWEEN … AND …`
	    - `IN (lista_de_valores)`, `NOT IN (lista_de_valores)`
	    - `LIKE`, `NOT LIKE` (patrones con `%` y `_`)
	    - `IS NULL`, `IS NOT NULL`
	- Se usan generalmente en la cláusula `WHERE` para filtrar filas.
3. Operadores Lógicos
	- Permiten combinar varias condiciones booleanas:
	    - `AND` (ambas condiciones deben ser verdaderas)    
	    - `OR` (al menos una debe ser verdadera)
	    - `NOT` (niega la condición)
    - Se usan en `WHERE` o `HAVING` para cirujanos filtros más complejos.
4. Operadores de Cadena (`String`)
	- Permiten comparar o concatenar texto:
	    - `LIKE` / `NOT LIKE` con `%` y `_` para patrones.
	    - En algunos motores: `||` o `CONCAT(...)` para concatenar cadenas.
	- Muy útiles para buscar por coincidencias parciales en nombres, correos, direcciones, etc.
5. Operadores de Existencia y Subconsulta
	- `EXISTS (subconsulta)`: devuelve filas cuando la subconsulta retorna al menos un registro.
	- `NOT EXISTS (subconsulta)`: opuesto a `EXISTS`.
	- Permiten verificar la existencia de registros relacionados antes de devolver filas.
6. Operadores Bitwise (Bit a Bit)
	- Se usan raramente en aplicaciones cotidianas, pero son importantes en ámbitos de bajo nivel o manipulación de flags:
        - `&` (AND bit a bit), `|` (OR bit a bit), `^` (XOR bit a bit), `~` (NOT bit a bit), `<<`, `>>`.    
	- Transforman enteros en forma binaria, ejecutan la operación en cada bit y devuelven un nuevo entero.
7. Operadores de Rango y Conjunto
	- `BETWEEN x AND y`: evalúa si un valor está dentro de un rango (incluye extremos).
    - `IN (valor1, valor2, ...)`: evalúa si un valor está dentro de una lista de valores.
    - Simplifican muchas comparaciones sucesivas (ej.: `columna = 1 OR columna = 2 …`).
8. Operadores Compuestos y Ternarios
	- **Operadores de asignación combinada** (según DBMS, menos comunes en SQL puro):
        - Por ejemplo, `+=, -=` en algunas extensiones de SQL.
    - **Operador condicional** `CASE WHEN … THEN … ELSE … END`: similar al ternario, permite devolver distintos valores según condiciones.

---
### Ejemplos Prácticos
##### Filtrar ventas por rango de fecha (Operador `BETWEEN`)
`Escenario`: Necesitamos mostrar todas las ventas realizadas entre 1 de mayo de 2025 y el 15 de mayo de 2025
```sql
SELECT ventaID, FechaVenta, Total
FROM Ventas
WHERE FechaVenta BETWEEN '2025-05-01' AND '2025-05-15'
```
El operador `BETWEEN` filtra las filas cuya columna `FechaVenta` cae entre las fechas dadas (incluyendo ambos extremos).
##### Buscar clientes cuyo nombre comience con "Mar" (Operador ``LIKE`)
`Escenario`: Queremos listar clientes cuyos nombre empiecen con "Mar" (ej. "Mario", "Maria").
```sql
SELECT ClienteID, Nombre
FROM Clientes
WHERE Nombre LIKE 'Mar%';
```
- El símbolo `%` en `LIKE 'Mar%'` indica que cualquier secuencia de caracteres después de "Mar"
- Devuelve filas donde la columna `Nombre` empiece con "Mar"
##### Actualizar inventario con operadores aritméticos
`Escenario`: En un sistema de inventario, queremos aumentar en 10 unidades el stock de todos los productos de la categoría "Electrónica"
```sql
UPDATE productos
SET Stock = Stock + 10
WHERE Categoria = "Electrónica";
```
- Se utiliza el operador `+` para incrementar la columna `Stock`
- Solo afecta las filas donde `Categoria = 'Electrónica'`
##### Uso de operadores lógicos `AND` y `OR` en un reporte de empleados
`Escenario`: Obtener el listado de empleados cuyo salario esté por encima de 20000 y que pertenezcan al departamento de "Ventas", o tengan más de 5 años de antigüedad.
```sql
SELECT empID, Nombre, Salario, FechaIngreso
FROM Empleados
WHERE
	(Salario > 20000 AND Departamento = 'Ventas')
	OR DATE_PART('year', AGE(CURRENT_DATE, FechaIngreso)) > 5;
```
- `(Salario > 20000 AND Departamento = 'Ventas')` filtra a los de “Ventas” con salario alto
- `OR DATE_PART(...) > 5` añade a quienes llevan más de 5 años, sin importar su salario o departamento.
##### Verificar existencia de órdenes de un cliente (`EXISTS`)
`Escenario`: Mostrar todos los clientes que han realizado al menos una orden
```sql
SELECT c.ClientID, c.Nombre
FROM Clientes c
WHERE EXISTS (
	SELECT 1
	FROM Ordenes o
	WHERE o.ClienteID = c.ClienteID
);
```
- La subconsulta `SELECT 1 FROM Ordenes o WHERE o.ClienteID = c.ClienteID` se evalúa para cada fila de `Clientes`
- Si existe al menos una fila en `Ordenes` con ese `ClienteID`, `EXISTS` retorna `TRUE` y el cliente se incluye en el resultado
##### Filtra productos cuyos IDs estén en una lista (`IN`)
`Escenario`: Mostrar datos de productos con IDs específicos que recibimos en un pedido: 3, 7 y 15
```sql
SELECT ProductoID, Nombre, Precio
FROM Productos
WHERE ProductoID IN (3, 7, 15);
```
- El operador `IN` compara `ProductoID` contra la lista `(3, 7, 15)`
- Equivale a escribir `WHERE ProductoID = 3 OR ProductoID = 7 OR ProductoID = 15`
##### Combinar cadenas con `||` (concatenación)
> En SQL estándar y en muchos motores (PostgreSQL, Oracle), se usa `||`. En MySQL se podría usar `CONCAT(...)`.

`Escanario`: Queremos mostrar el nombre completo de clientes concatenando `"Nombre"` y `"Apellido"`
```sql
SELECT ClienteID, Nombre || ' ' || Apellido AS NombreCompleto
FROM Clientes;
```
- El operador `||` concatena cadenas con espacio en medio
- Devuelve una columna `NombreCompleto` como "Juan Pérez", etc.
##### Operadores de comparación con valores nulos
`Escenario`: Obtener todos los empleados cuyo campo "`FechaSalida`" sea nulo (aún trabajan en la empresa)
```sql
SELECT EmpID, Nombre
FROM Empleados
WHERE FechaSalida IS NULL;
```
- El operador `IS NULL` filtra filas donde `FechaSalida` no tiene valor
- Para desigualdad: `FechaSalida IN NOT NULL`
