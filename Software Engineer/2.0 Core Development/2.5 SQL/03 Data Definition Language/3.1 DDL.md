El lenguaje de definición de datos (DDL) es un subconjunto de SQL utilizando para definir y administrar la estructura de los objetos de la base de datos. Los comandos DDL incluyen `CREATE`, `ALTER`, `DROP` y `TRUNCATE`, que se utilizan para crear, modificar, eliminar y vaciar estructuras de bases de datos como tablas, índices, vistas y esquemas. Estos comandos permite a los administradores y desarrolladores de bases de datos, establecer relaciones entre tablas y administrar la estructura general de base de datos. Las declaraciones DDL generalmente generan cambios inmediatos en la estructura de la base de datos y pueden afectar los datos existentes.

A diferencia de las consultas que manipulan datos (DML), DDL trabaja a nivel de esquema: crea, altera o elimina objetos como tablas, índices y vistas.

> **Analogía**: Imagina que una base de datos es una casa.
> **DDL es un conjunto de herramientas y acciones para construir o remodelar la casa**: poner paredes, ventanas, puertas (`CREATE`), cambiar el tamaño de la habitación (`ALTER`), demoler una parte (`DROP`), o limpiar todo el interior sin tocar la estructura (`TRUNCATE`).
> DML, en cambio, sería como amueblar las habitaciones y mover muebles (`INSERT`, `UPDATE`, `DELETE`).
