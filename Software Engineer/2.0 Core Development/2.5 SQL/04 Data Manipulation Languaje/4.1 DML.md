El Data Manipulation Language (DML) es el subconjunto de SQL encargado de acceder y modificar los datos almacenados en las tablas de una base de datos. Incluye principalmente las sentencias:
- `SELECT` (lectura/consulta)
- `INSERT` (inserción de nuevas filas)
- `UPDATE` (actualización de filas existentes)
- `DELETE` (borrado de filas)
Algunos textos diferencian a `SELECT` como parte de DQL (Data Query Language), en la práctica se considera dentro de DML porque implica manipulación de datos.

> Analogía: Imagina que tu base de datos es una tabla de Excel con muchas filas y columnas.
> - DDL sería el equivalente a dar formato a la hoja: crearla, cambiar el ancho de las columnas, eliminarla por completo.
> - DML es como trabajar sobre las celdas: leer valores, agregar nuevas filas, modificar celdas existentes o eliminar filas.

- Transacciones y DML
	- Las operaciones DML forman parte de transacciones: un conjunto de sentencias que se ejecutan como unidad
	- Hasta que no se ejecute `COMMIT`, los cambios `INSERT`, `UPDATE` o `DELETE` pueden revertirse con `ROLLBACK`
Las declaraciones DML operan sobre los datos en sí en lugar de sobre la estructura de la base de datos, lo que permite a los usuarios interactuar con la información almacenada.
Estos comandos son esenciales para las operaciones diarias de la base de datos, el análisis de datos y el mantenimiento de la precisión y relevancia de los datos dentro de un sistema de base de datos.
