> [!info] WHERE (Roadmap SH)
> SQL proporciona una cláusula `WHERE` que se utiliza básicamente para filtrar los registros. Si se cumple la condición especificada en la cláusula `WHERE`, solo se devuelve el valor específico de la tabla. Debe utilizar la cláusula `WHERE` para filtrar los registros y obtener solo los registros necesarios.
> La cláusula `WHERE` no solo se utiliza en la instrucción `SELECT`, sino que también se utiliza en las instrucciones `UPDATE`, `DELETE`, etc.
### Definición
La cláusula `WHERE` en SQL permite filtrar filas según condiciones específicas. Funciona como un colador: de un conjunto grande de datos, solo deja pasar las filas que cumplen la(s) condición(es) definidas, evitando que el resto aparezca en el resultado de la consulta, actualización o eliminación.

> [!info] Analogía
> Si tienes una gran hoja de cálculo con clientes, aplicar `WHERE` es como usar el filtro de Excel para mostrar únicamente aquellas filas donde la columna "Cuidad" sea "Guadalajara" o la columna "Edad" sea mayor a 30.
### Conceptos clave
##### Filtros básicos
```sql
SELECT *
FROM empleados
WHERE salario > 50000;
```
- Trae solo empleados cuyo salario supere 50,000.
- `=` (igual), `<>` o `!=` (diferente), `<`, `>`, `<=`, `>=`.
##### Rangos con `BETWEEN ... AND ...`
```sql
SELECT nombre
FROM productos
WHERE precio BETWEEN 100 AND 200;
```
- Trae productos cuyo precio sea entre 100 y 200, incluidos ambos.
- Simplifica la comparación de valores entre dos límites inclusivos.
##### Valores en lista con `IN`
```sql
SELECT *
FROM pedidos
WHERE estado IN ('Pendientes', 'En camino', 'Entregado');
```
- Trae pedidos con estado "Pendiente", "En camino", "Entregado".
- Compara una columna contra varios valores posibles, es equivalente a varias comparaciones `OR`.
##### Patrones con `LIKE` y comodines
```sql
SELECT nombre
FROM clientes
WHERE nombre LIKE 'J%';
```
- Trae clientes cuyo nombre empiece con "J".
- `%` representa cualquier secuencia de caracteres, `_` un solo carácter
```sql
SELECT correo
FROM clientes
WHERE correo LIKE '%gmail.com';
```
- Trae usuarios cuyo correo termine en `@gmail.com`.
##### Valores nulos con `IS NULL` / `IS NOT NULL`
```sql
SELECT *
FROM usuarios
WHERE telefono IS NULL;
```
- Trae usuarios que no tiene teléfono registrado.
- Permite filtrar donde una columna no tiene valor asignado (`NULL`)
##### Combinar condiciones con `AND`, `OR`, `NOT`
```sql
SELECT *
FROM empleados
WHERE salario > 30000
	AND departamento = 'Ventas';
```
- Trae empleados de "Ventas" cuyo salario supere 30,000
- `AND` todas las condiciones deben ser verdaderas
- `OR` al menos una debe ser verdadera
- `NOT` invierte la condición
### Ejemplos Prácticos
##### Filtrar clientes según ciudad y edad
```sql
SELECT cliente_id, nombre, ciudad, edad
FROM clientes
WHERE cuidad = 'Monterrey'
	AND edad >= 30
ORDER BY nombre;
```
- Muestra solo clientes que vivan en Monterrey y tengan al menos 30 años
- Ordena alfabéticamente por nombre
##### Listar productos en promoción (precio entre un rango)
```sql
SELECT producto_id, nombre_producto, precio
FROM productos
WHERE precio BETWEEN 50 AND 150;
```
- Muestra productos cuyo precio esté entre 50 y 150 (incluidos)
##### Buscar empleados cuyo nombre contenga "ana"
```sql
SELECT empleado_id, nombre
FROM empleados
WHERE nombre LIKE '%ana%';
```
- Muestra empleados cuyo nombre incluya la secuencia "ana" en cualquier parte (por ejemplo, "Diana", "Anabel").
##### Ver órdenes con estados específicos
```sql
SELECT orden_id, cliente_id, estado
FROM ordenes
WHERE estado IN ('Pendiente', 'En camino', 'Entregado');
```
- Trae órdenes cuyo estado sea exactamente "Pendiente", "En camino" o "Entregado".
##### Identificar usuario sin correo registrado (`NULL`)
```sql
SELECT usuario_id, nombre
FROM usuarios
WHERE correo IS NULL;
```
- Muestra usuarios que no tienen correo electrónico (columna `correo` es `NULL`).
