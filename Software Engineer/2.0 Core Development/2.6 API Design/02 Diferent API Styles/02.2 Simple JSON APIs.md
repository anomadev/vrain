## Definición
`JSON:API` es una especificación formal para construir `APIs` en `JSON` de manera estandarizada. Su principal objetivo es ser una herramienta "`anti-bikeshedding`" (evitar discusiones triviales sobre el formato) al definir convenciones estrictas sobre cómo se deben solicitar y enviar los recursos, cómo manejar las relaciones y cómo estructurar los errores. Utiliza el tipo de medio especializado `application/vnd.api+json`.

## Conceptos Clave
- `Resource Objects`: El corazón del documento. Cada recurso DEBE tener un `type` y un `id`.
- `Attributes`: Contiene los datos del recurso. No debe incluir relaciones aquí.
- `Compound Documents (Included)`: Permite enviar recursos relacionados en una sola respuesta para reducir el número de peticiones `HTTP`.
