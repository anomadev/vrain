## Definición
`HATEOAS` es la restricción final y más avanzado del estilo arquitectónico `REST`. Establece que un cliente debe interactuar con una aplicación de red enteramente a través de la `hypermedia` (enlaces y metadatos) proporcionada dinámicamente por el servidor en las respuestas. En un sistema que implementa `HATEOAS`, el cliente no necesita conocer de antemano las `URLs` de los recursos ni las reglas de negocio complejas; simplemente "navega" la `API` siguiendo los enlaces disponibles, de la misma forma que un humano navega por la web haciendo clic en hipervínculos.

## Conceptos Clave
- `Richarson Maturity Model (Level 3)`: `HETAOAS` representa el nivel más alto de madurez de una `API REST`.
- `Hypermedia Controls`: Son los elementos (enlaces, formularios, botones en `HTML` o nodos `links` en `JSON`) que indican al cliente qué acciones puede realizar.
- `Self-Discovery`: La capacidad de la `API` de guiar al cliente sobre su propia estructura y flujos de trabajo sin necesidad de documentación externa (`out-of-band information`).
- `Decoupling (Desacoplamiento)`: El cliente no `hardcodea` las `URLs`. Si el servidor cambia una ruta (ej. `/v1/users` a `/users`), el cliente no se rompe porque siempre usa la `URL` que el servidor le entregó en la respuesta anterior.
- `State Transitions`: Los enlaces devueltos dependen del estado actual del recurso. Si una cuenta bancaria está bloqueada, el servidor no incluirá el enlace para "retirar dinero", solo para "desbloquear".

## Ejemplos Prácticos
##### `Standard JSON Response (No HATEOAS)`
El cliente debe saber de antemano cómo construir la `URL` para ver los depósitos.
```json
{
	"account_number": "12345",
	"balance": 100.00,
	"status": "active"
}
```
##### `HATEOAS Response (Using HAL - JSON Hypermedia API Language)`
El servidor indica explícitamente qué puede hacer el cliente a continuación.
```json
{
  "account_number": "12345",
  "balance": 100.00,
  "status": "active",
  "_links": {
    "self": { "href": "/accounts/12345" },
    "deposits": { "href": "/accounts/12345/deposits" },
    "withdrawals": { "href": "/accounts/12345/withdrawals" },
    "transfer": { "href": "/accounts/12345/transfers" }
  }
}
```
##### `State-Driven HATEOAS (Overdrawn Account)`
Si la cuenta está en negativo, el servidor "apaga" las opciones de retiro y transferencia.
```json
{
  "account_number": "12345",
  "balance": -50.00,
  "status": "overdrawn",
  "_links": {
    "self": { "href": "/accounts/12345" },
    "deposits": { "href": "/accounts/12345/deposits" }
  }
}
```

## Analogía
Imagina que vas a un Restaurante con un Menú Dinámico.
- **Sin `HATEOAS`**: Tienes que memorizar de antemano que para pedir pizza debes ir a la cocina y decir "Pizza". Si el chef cambia el nombre del plato o mueve la estación de pizza, te quedas con hambre.
- **Con `HATEOAS`**: Te sientas y te entregan un Menú (la respuesta de la `API`). Si es de mañana, el menú solo muestra "Desayunos". Si no tienes dinero, el menú tiene tachada la opción "Langosta". Solo puedes pedir lo que el menú te ofrece en ese momento. No necesitas saber cómo se llama el plato en la cocina; solo señalas la opción en el papel.

## Buenas Prácticas
- **`Use Link Relations (rel)`**: Define nombres de relación estándar para que el cliente sepa qué significa cada enlace (ej. `next`, `prev`, `edit`, `author`).
- **`Adopt a Standard Format`**: No inventes tu propia estructura de enlaces. Usa `HAL (application/hal+json)`, `Siren`, o `JSON-LD`.
- **`Avoid Out-of-Band Knowledge`**: Diseña la `API` para que la documentación sea secundaria. El cliente ideal debería poder realizar un flujo completo (ej. comprar un producto) solo siguiendo enlaces desde el `root` de la `API`.
- **`Consistent Link Structure`**: Mantén la estructura de los objetos de enlace igual en toda la `API` para facilitar el `parseo` en el cliente.

## Notas Personales
- `HATEOAS` es técnicamente lo que hace que una `API` sea "`RESTful`". Sin esto, es solo una `API HTTP` con `JSON`.
- En el mundo real, los desarrolladores odian "adivinar" enlaces y prefieren `Swagger`. Sin embargo, en arquitecturas donde el `backend` cambia constantemente, `HATEOAS` es un salvavidas.
- Proyectos como `HTMX` están reviviendo el interés en `HATEOAS` porque devuelven `HTML` directamente, que es el `hypermedia` "natural" por excelencia.
