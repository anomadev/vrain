## Definición
El `Control de Acceso Obligatorio (MAC)` es un modelo de seguridad donde el acceso a los recursos es regulado por una autoridad central basándose en políticas estrictas, y no por el dueño del recurso. En este sistema, tanto los sujetos (usuarios/procesos) como los objetos (archivos/datos) tienen etiquetas de seguridad. El sistema compara estas etiquetas para permitir o denegar el acceso, impidiendo que un usuario pueda relajar las restricciones por su cuenta.

## Conceptos Clave
Para un Arquitecto de Software, el valor de `MAC` reside en su capacidad de imponer políticas globales inquebrantables:
- **`Security Labels & Clearance`**: Cada objeto tiene una clasificación (ej. `Secret`) y cada sujeto tiene un nivel de autorización o "`clearance`".
- **`Centralized Policy Entorcement`**: Las reglas son definidas por un administrador de seguridad y aplicadas directamente por el `Security Kernel` del sistema operativo.
- **`Modelo Bell-LaPadula (Confidencialidad)`**: Enfocado en proteger el secreto de la información. Sus reglas son:
	- **`No Read Up`**: No puedes leer datos de un nivel superior al tuyo.
	- **`No Write Down`**: No puedes escribir datos en un nivel inferior para evitar fugas de información sensible.
- **`Modelo Biba (Integridad)`**: Enfocado en la veracidad de los datos. Sus reglas son:
	- **`No Read Down`**: No lees datos de baja integridad para no contaminar tu análisis.
	- **`No Write Up`**: No puedes escribir en niveles superiores para no corromper datos de alta fidelidad.
- **`Modelo Clark-Wilson`**: Se basa en "transacciones bien formadas" y la separación de funciones para asegurar que los datos solo cambien de un estado válido a otro.

## Ejemplos Prácticos
##### Entornos Gubernamentales y Militares
Es el uso clásico. Un documento etiquetado como "`Top Secret`" solo puede ser abierto por alguien con esa credencial específica. El autor del documento no tiene poder para bajar la categoría a "Público".
##### Seguridad en Linux (`SELinux` y `AppArmor`)
Implementaciones modernas de `MAC` que confinan procesos. Por ejemplo, aunque un servidor web sea comprometido, `SELinux` puede impedir que ese proceso acceda a archivos fuera de su etiqueta, aunque el proceso corra como `root`.
##### Sistemas de Alta Integridad (Bancos/Salud)
Protección de registros médicos o transacciones financieras donde la integridad es más importante que el secreto, asegurado que solo procesos verificados puedan modificar saldos o historiales.

## Analogía
Imagina un **Búnker de Alta Seguridad**.
- `DAC`: Sería como si cada persona tuviera la llave de su propia oficina y pudiera invitar a quien quiera.
- `MAC`: Es un sistema donde un **Oficial de Seguridad Central** te pega un carnet en el pecho con un color (ej. Rojo). Todas las puertas del búnker tienen un lector que solo se abre si tu carnet es Rojo o superior. No importa si tu jefe te invita a pasar a la "Sala Azul"; si tu carnet no tiene el nivel, la puerta **simplemente no se abrirá**.

## Buenas Prácticas
- **Define una Política de Etiquetado Clara**: Antes de implementar, debes mapear el flujo de información y definir los niveles de sensibilidad de forma exhaustiva.
- **Principio de Mínimo Privilegio**: Asigna el nivel de "`clearance`" más bajo posible que permita al usuario cumplir su función.
- **Uso de Perfiles Estándar**: En servidores, utiliza perfiles de `SELinux` o `AppArmor` ya probados para servicios comunes (`Nginx`, `MySQL`) antes de crear los tuyos.
- **Auditoria Continua**: Dado que es un sistema rígido, monitorea los logs de denegación para ajustar etiquetas que puedan estar causando cuellos de botella operativos.

## Notas Personales
- `MAC` es la "fortaleza digital". Es extremadamente difícil de gestionar a escala debido al `overhead` administrativo de mantener etiquetas actualizadas.
- Para la mayoría de los SaaS comerciales, `MAC` es excesivo como modelo principal, pero es vital aplicarlos a nivel de infraestructura (`SO`) para mitigar el radio de explosión de un posible ataque.
- **Dato Clave**: Si decides usar `SELinux`, prepárate para una curva de aprendizaje empinada, pero tendrás la tranquilidad de que ni un `root` comprometido podrá desmantelar todo el sistema fácilmente.
