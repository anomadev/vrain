## Definición
`Stoplight` es una plataforma integral de gobernanza y documentación de `APIs` que cubre todo el ciclo de vida del desarrollo. Su filosofía central es el `Design First`, proporcionando un entorno donde los arquitectos pueden diseñar contratos `OpenAPI` de forma visual, aplicar reglas de estilo automáticas y generar prototipos (`mocks`) instantáneos, asegurando que la `API` sea consistente y de alta calidad desde concepción.

## El "Trío de Oro" (`Open Sorce Core`)
El ecosistema de `Stoplight` se sostiene sobre tres herramientas de código abierto que son estándares en la industria:
- **`Spectra (Linter)`**: Un validador de archivos `JSON/YAML` que aplica "`Style Guides`". Asegura que todos tus `endpoints` sigan las mismas reglas (ej.  "usar siempre `kebab-case`, "incluir siempre descripciones").
- **`Prism (Mock Server)`**: Un servidor de pruebas que lee tu archivos `OpenAPI` y simula el comportamiento de la `API`. Permite que el equipo de `frontend` empiece a trabajar sin esperar a que el `backend` esté terminado.
- **`Elements (Documentation UI)`**: Componentes web (`React` o `Web Components`) para renderizar documentación interactiva de "tres columnas" (estilo `Stripe`), con búsqueda avanzada y ejemplos de código automáticos.

### Características Principales
1. **`Stoplight Studio (Visual Designer)`**: Es un editor híbrido: puedes escribir el `YAML` directamente o usar una interfaz visual (formularios) para definir rutas, modelos de datos y seguridad. Esto democratiza el diseño de la `API`, permitiendo de perfiles no técnicos colaboren en la definición del contrato.
2. **`API Governance & Style Guides`**: `stoplight` permite crear y compartir guías de estilo a través de toda una organización. Si un desarrollador intenta crear un `endpoint` que no cumple con el estándar de la empresa, `Spectral` lanzará una advertencia en tiempo real dentro del editor.
3. **`Git-Centric Workflow`**: Se integra profundamente con `GitHub`, `GitLab` y `Bitbucket`. La documentación no vive en una base de datos aislada, sino que es un reflejo de los archivos en tu repositorio. Cada `commit` en el diseño puede disparar actualizaciones automáticas en el portal de desarrolladores.

## Analogía
Imagina que estás construyendo un `Rascacielos (API)`
- **`Swagger`** es el papel y la regla para hacer el dibujo técnico.
- **`Stoplight`**: es un Software de Modelado 3D (como `AutoCAD` o `Revit`). No solo dibujas las líneas; el sistema te avisa si una viga es muy delgada (`Spectral`), te permite entrar a una habitación virtual para ver cómo se siente antes de construirla (`Prism`) y genera automáticamente los folletos de venta para los compradores (`Elements`).

## Buenas Prácticas
- **`DRY (Dont Repeat Yourself)`**: Utiliza los Modelos Reutilizables de `Stoplight`. Define un esquema de `User` una vez y llámalo mediante `$ref` en todos tus `endpoints` para mantener la consistencia.
- **`Linting en CI/CD`**: No solo uses `Spectral` en el editor; intégralo en tus `pipelines` de integración continua para rechazar cualquier cambio que rompa las reglas de estilo de la `API`.
- **`Mocks para Feedback Temprano`**: Comparte la `URL` del `mock` de `Prism` con los consumidores de tu `API` antes de programar el `backed`. Es más fácil cambiar un `YAML` que refactorizar una base de datos.
- **`Stoplight-flavored Markdown`**: Aprovecha los componentes especiales de `Stoplight` en tus guías para añadir pestañas, alertas de seguridad y ejemplos de código interactivos.

## Notas
`Stoplight` es la elección de los puristas de la arquitectura. Si tu objetivo es crear un ecosistema de `APIs` que se sientan como una unidad coherente y no como un grupo de servicios aislados, sus herramientas de gobernanza son inmejorables. Especialmente útil si estás en una fase de refactorización o diseño de sistemas complejos donde la consistencia es el activo más valioso.
