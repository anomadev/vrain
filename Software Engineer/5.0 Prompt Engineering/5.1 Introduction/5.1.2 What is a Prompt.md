## Definición
Un Prompt es el conjunto de instrucciones, contexto y datos de entrada proporcionados a un Modelo de Lenguaje (LLM) para guiar su generación de respuestas. Desde la arquitectura de software, es el input vector que condiciona la distribución de probabilidad del modelo para que el output sea determinista dentro de los límites de nuestra lógica de negocio.

## Conceptos Clave
Para dominar el prompting como ingeniero, enfócate en la anatomía del prompt:
- **Instruction (Instrucción)**: La tarea específica que el modelo debe realizar (ej. "Resume", "Traduce", "Extrae").
- **Context (Contexto)**: Información externa o de dominio que el modelo no conoce de antemano (ej. "Regulaciones del SAT 2024").
- Input Data (Datos de entrada): El dato crudo a procesar (ej. El cuerpo de un correo o un JSON de una factura).
- **Output Indicators (Restricciones de formato)**: Definir cómo queremos el resultado (ej. "Responde solo en formato JSON", "Usa un tono profesional").
- **Zero-shot vs. Few-shot**: 
	- **Zero-shot**: Solo das la instrucción.
	- **Few-shot**: Proporcionas ejemplos de entrada/salida para "entrenar" contextualmente al modelo.

## Notas personales
Al principio cometía el error de tratar los prompts como conversaciones casuales. Como ingeniero, debes tratarlos como Configuración. Si un prompt no es consistente, tu sistema no es confiable.
