## Definición
La ingeniería de Prompts es el proceso de estructurar y refinar las entradas de un LLM para activar patrones específicos aprendidos durante su entrenamiento, permitiendo que el modelo ejecute tareas complejas sin necesidad de reentrenarlo (fine-tuning) o de conectar bases de datos externas (RAG). Es la técnica de guiar los "mecanismos de atención" del modelo hacia el contexto relevante para maximizar la precisión de la respuesta.

## Conceptos Clave
- **Activación de Patrones**: Un prompt bien diseñado no da "órdenes", sino que activa secuencias lógicas pre-existentes en el modelo (ej. piensa paso a paso, activa el razonamiento metódico).
- **Limitación de Conocimiento**: Es vital recordar que la ingeniería de prompts no añade información nueva al modelo; solo optimiza el acceso a lo que ya sabe dentro de su fecha de corte.
- **Zero Infrastructure Cost**: A diferencia de RAG o Fine-tuning, esta técnica no requiere cambios en el backend ni bases de datos vectoriales, lo que la hace ideal para iteraciones rápidas.
- **Trial and Error**: Es un proceso iterativo. Cómo arquitecto, debes ver los prompts como código que requiere "debugging" contante para encontrar la instrucción óptima.

## Notas Personales
Desde la perspectiva de diseño de sistemas, la ingeniería de prompts es el "Unit Testing" del lenguaje natural. Un prompt mal diseñado es un "bug" silencioso. Trata cada prompt como una interfaz (**Interface**): define qué entra y qué debe salir, pero mantén el "cómo" dentro de la caja negra del LLM, siempre validando que el output.
