## Definición
Un Web Framework es una estructura de soporte predefinida (un "esqueleto" de software) que proporciona herramientas, librerías y convenciones para agilizar el desarrollo. Laravel es un framework de PHP ([[1.1 What is PHP]]) diseñado bajo una filosofía "`progressive`", lo que significa que es lo suficientemente sencillo para prototipos rápidos y lo suficientemente robusto para aplicaciones de escala empresarial con `Clean Architecture`.

## Conceptos Clave
- `Progressive Framework`: Laravel escala en complejidad junto con tus necesidades. No te obliga a usar herramientas avanzadas desde el inicio, pero están ahí cuando las requieras.
- `Convention over Configuration (CoC)`: Laravel asume decisiones por ti (dónde van los archivos, cómo se nombran las tablas). Esto reduce la fatiga de decisión y facilita el `Agentic Development` (IA entendiendo tu código).
- `Zero Configuration Resolution`: El uso de `Dependenncy Injection` automático que permite instanciar objetos complejos sin configuración manual.
- `Batteries Included`: Viene con soluciones nativas para problemas comunes: `Authentication`, `Queue Management`, `ORM (Eloquent)`, y `Routing`.

## Ejemplos prácticos
En **`Vanilla PHP`**, tendrías que gestionar manualmente la conexión, la limpieza de datos y el `fetching`: 
```php
// Vanilla PHP - Risk of SQL Injection, manually handling results
$id = $_GET['id'];
$conn =  new mysqli($host, $user, $pass, $db);
$result = $conn->query("SELECT * FROM users WHERE id = " . $id);
$user = $result->fetch_assoc();
```
En `Laravel`, el framework se encarga de la seguridad y la abstracción.
```php
// Laravel - Clean, Secure, and Expressive
public function show(User $user)
{
	// The User object is autommatically fetched via Route Model Binding
	return view('users.profile', ['user' => $user]);
}
```

## Analogía
Imagina que vas a construir una casa. `Vanilla PHP` es como si tuvieras que fabricar tus propios ladrillos, talar los árboles para la madera y forjar tus clavos. Un `Framework` como `Laravel` es como comprar una `Prefab House` de alta gama: ya tienes los cimientos, las tuberías y la estructura eléctrica instalada; tú solo te encargas de la distribución de las habitaciones y la decoración (la lógica de tu negocio).

## Buenas Prácticas
- `Follow the Conventions`: No intentes "pelear" contra el framework. Si `laravel` espera que tus modelos estén en `app/Models`, mantenlos ahí. Esto facilita que otros developers (y herramienta de IA) entiendan tu proyecto de inmediato.
- `Don't Reinvent the Wheel`: antes de escribir una función compleja de validación o manejo de archivos, revisa el `Service Container` o los `Helpers` de `Laravel`; es muy probable que ya exista una solución optimizada.

## Escenarios y Aplicaciones
- `Rapid Prototyping (MVP)`: Ideal cuando necesitas validar una idea de negocio rápido gracias a los `Starter Kits` (`Breeze`, `Jetstream`).
- `Enterprise Applications`: Gracias a su soporte para `Horizontal Scaling` con `Redis` y sistemas de `Queues`, puede manejar millones de `requests`.
- `AI-Assisted Projects`: `Laravel` es "`Agent Ready`". Al seguir convenciones estrictas, herramientas como Cursor o Claude pueden predecir con exactitud dónde añadir lógica o cómo generar una migración.

## Notas Personales
- Un error común al aprender es pensar que el framework "pesa" demasiado (`Performance Overhead`). La realidad es que el tiempo que ahorras en desarrollo y seguridad compensa por mucho los milisegundos de ejecución en el 99% de los casos.
- La documentación menciona el `Agentic Development`. Esto es clave: en la era de la IA, usar el framework con convenciones clara (como `Laravel`) te da una ventaja competitiva brutal porque los modelos de lenguaje entienden tu código mejor que uno "custom" desordenado.
