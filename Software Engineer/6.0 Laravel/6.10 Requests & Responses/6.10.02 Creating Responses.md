### Tipos de Respuestas Básicas
##### Cadenas y `Arrays`
Laravel permite devolver `strings` directamente desde las rutas o controladores. Si devuelves un `array`, el `framework` lo convertirá automáticamente en una respuesta `JSON` con la cabecera `Content-Type: application/json`.
```php
Route::get('/', function () {
    return 'Hola Mundo'; // Devuelve texto plano
});

Route::get('/api/status', function () {
    return ['status' => 'OK', 'code' => 200]; // Devuelve JSON automáticamente
});
```
##### Objetos de Respuesta
Para tener un control total sobre las cadenas y los códigos de estado, se utiliza el `helper` `response()`.
```php
return response('Contenido no encontrado', 404)
            ->header('Content-Type', 'text/plain')
            ->header('Custom-Header', 'Value');
```
### Respuesta Especializada
##### Respuesta de Vista
Es la forma más común de devolver `HTML` generado por el motor de plantillas `Blade`.
```php
return response()
            ->view('hello', $data, 200)
            ->header('Content-Type', 'text/html');
```
##### Respuesta `JSON`
Aunque Laravel convierte `arrays` automáticamente, usar el método `json()` permite definir códigos de estado y cabeceras de forma más explícita, algo vital para la construcción de `APIs` profesionales.
```php
return response()->json([
    'name' => 'Abigail',
    'state' => 'CA',
], 201);
```
##### Respuestas de Archivo (Descargas y Visualización)
- `Download`: Fuerza al navegador a descargar el archivo.
- `File`: Muestra el archivo directamente en el navegador (útil para `PDFs` o imágenes).
```php
return response()->download($pathToFile);
return response()->file($pathToFile);
```
### Redirecciones 
Las redirecciones son respuestas especiales que indican al navegador que debe navegar a una `URL` diferente.
- A una `URL` externa: `return redirect('https://google.com')`
- A una ruta con nombres: `return redirect()->route('login')`;
- A la página anterior: `return back()->withInput();` (muy útil tras fallos de validación).
### Códigos de Estado Comunes
Entender qué códigos devolver es la marca de un ingeniero con criterio técnico.
- `200 (OK)`: Petición exitosa
- `201 (Created)`: Recurso creado con éxito (típico en `POST`)
- `204 (No Content)`: Éxito, pero no hay nada que devolver (típico en `DELETE`)
- `400 (Bad Request)`: Error del cliente (petición mal formateada)
- `401 (Unauthorized)`: Falta de autenticación
- `403 (Forbidden)`: Autenticado pero sin permisos
- `404 (Not Found)`: Recurso inexistente
- `422 (Unprocessable Entity)`: Error de validación de datos
- `500 (Internal Server Error)`: Error crítico en el servidor
### `Response` Macros (`Senior Approach`)
Si te encuentras repitiendo la misma estructura de respuesta en mucho controladores, puedes definir un "Macro" en el `AppServiceProvider` para reutilizarlo.
```php
// app/Providers/AppServiceProvider.php
public function boot(): void
{
    Response::macro('caps', function (string $value) {
        return Response::make(strtoupper($value));
    });
}

// Uso en controlador
return response()->caps('foo');
```

## Buenas Prácticas
1. **Consistencia en `APIs`**: Define un formato estándar para tus respuestas de error (ej. siempre incluir una clave `message` y una clave `errors`).
2. **No fuerces `JSON` manual**: Evita usar `json_encode()` dentro de tus controladores; deja que Laravel maneje la serialización y las cabeceras por ti.
3. **Encadenamiento de Cabeceras**: Laravel utiliza una interfaz fluid, lo que te permite encadenar múltiples llamadas a `header()` o usar `withHeaders([])` para pasar un `array`.
4. **Cuidado con las `cookies`**: Las `cookies` enviadas en la respuesta deben ser adjuntadas mediante el método `cookie()` para que Laravel las encripte y firme correctamente.
