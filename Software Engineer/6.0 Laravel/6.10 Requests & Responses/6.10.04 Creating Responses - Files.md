### Recuperación de Archivos (`Request`)
Cuando un cliente envía un archivo a través de un formulario con el atributo `enctype="multipart/form-data"`, Laravel lo envuelve en una instancia de la clase `Illuminate\Http\UploadedFile`.
##### Acceso y Validación
Para obtener el archivo desde la petición, se utilizar el método `file` o el acceso dinámico a propiedades. Es fundamental verificar siempre que el archivo esté presente y sea válido antes de procesarlo.
```php
// Obtener el archivo
$file = $request->file('avatar');

// O mediante propiedad dinámica
$file = $request->avatar;

// Verificar presencia y validez
if ($request->hasFile('avatar') && $request->file('avatar')->isValid()) {
    // Procesar archivo
}
```
##### Rutas y Extensiones 
La instancia de `UploadedFile` ofrece métodos para inspeccionar el archivo antes de guardarlo:
- **`path()`**: Devuelve la ruta temporal completa del archivo.
- **`extension()`**: Intenta determinar la extensión basada en el contenido `MIME` del archivo.
- **`getClientOriginalName()`**: Devuelve el nombre original del archivo tal como estaba en la computadora del usuario (no confiable para seguridad, pero útil para visualización).
##### Almacenamiento
El método `store` mueve el archivo desde la ubicación temporal a un disco configurado, generando un nombre único (`hash`) para evitar colisiones.
```php
$path = $request->avatar->store('avatars');
// O especificando un nombre de archivo
$path = $request->avatar->storeAs('avatars', 'user_1.jpg');
```
### Creación de Respuestas de Archivo (`Response`)
Laravel permite devolver archivos al cliente de dos manera principales: mostrándolos en el navegador (visualización) o forzando la descarga.
##### Mostrar Archivos (`file`)
El método `file` se utiliza para mostrar imágenes, `PDFs` o vídeos directamente en el navegador, en lugar de iniciar una descarga. Este método establece automáticamente las cabeceras `Content-Type` adecuadas.
```php
return response()->file($pathToFile);

// Con cabeceras personalizadas
return response()->file($pathToFile, $headers);
```
##### Forzar Descargas (`download`)
El método `download` genera una respuesta que obliga al navegador a descargar el archivo. Puedes personalizar el nombre que verá el usuario y las cabeceras.
```php
return response()->download($pathToFile);

// Personalizando el nombre del archivo al descargar
return response()->download($pathToFile, 'reporte_final.pdf', $headers);
```
##### Respuestas de descarga eliminables
Un caso de uso común es generar un archivo temporal (como un `CSV`) y borrarlo inmediatamente después de que el usuario lo descargue. Laravel ofrece el método `deleteFileAfterSend` para esto.
```php
return response()->download($pathToFile)->deleteFileAfterSend();
```
### Consideraciones de Seguridad
1. **Validación de `MIME`**: Nunca confíes solo en la extensión del archivo. Valida siempre el tipo de contenido (ej. `mimes:jpg,png,pdf`) en tus reglas de validación.
2. **Visibilidad**: Los archivos almacenados en la carpeta `storage/app` no son accesibles públicamente. Para que lo sean, debe estar en `storage/app/public` y debes ejecutar el comando `php artisan storage:link`.
3. **Nombres de Archivos**: Evita usar el nombre original del cliente al guardar en el servidor para prevenir ataques de inyección de rutas. Deja que Laravel genere el `hash` por defecto.
