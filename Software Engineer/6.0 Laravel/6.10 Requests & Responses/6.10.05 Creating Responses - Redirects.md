### `Redirects` Básicas
El `helper` global `redirect` es la puerta de entrada para enviar al usuario a una nueva ubicación. Por defecto, Laravel utiliza un código de estado `HTTP 302` (redirección temporal).
```php
Route::get('/dashboard', function () {
    return redirect('/home');
});
```
##### Redirección a la página anterior
Es extremadamente común después de un fallo en la validación de un formulario. El método `back()` utiliza la sesión para determinar la ubicación previa.
```php
return back()->withInput();
```
### Redirecciones a Rutas Nombradas
Esta es la mejor práctica en Laravel. Al usar nombres en lugar de `URLs` manuales, tu código no se rompe si decides cambiar la estructura de las `URLs` en el futuro.
```php
// Redirección simple
return redirect()->route('login');

// Redirección con parámetros
return redirect()->route('profile', ['id' => 1]);
```
### Redirecciones a Acciones de Controlador
Si prefieres apuntar directamente a un método de un controlador, puedes usar el método `action`
```php
use App\Http\Controllers\UserController;

return redirect()->action([UserController::class, 'index']);
```
### Redirecciones con Datos de Sesión (`Flash Data`)
Normalmente querrás enviar un mensaje de éxito al usuario después de redirigirlo. Laravel permite "encadenar" mensaje que solo durarán una petición.
```php
return redirect()->route('dashboard')
    ->with('status', '¡Perfil actualizado con éxito!');
```
En tu vista de `Blade`, podrías mostrarlo así:
```php
@if (session('status'))
    <div class="alert alert-success">
        {{ session('status') }}
    </div>
@endif
```
### Redirecciones Externas
Si necesitas enviar al usuario fuera de tu aplicación (por ejemplo, a una pasarela de pagos o a un sitio de documentación), utiliza el método `away`.
```php
return redirect()->away('https://www.google.com');
```
### Comparativa de Códigos de Estado

| Código | Tipo                 | Uso Sugerido                                                          |
| :----- | -------------------- | --------------------------------------------------------------------- |
| `301`  | `Moved Permanently`  | SEO: Cuando una página ha cambiado de `URL` para siempre              |
| `302`  | `Found (Temporary)`  | El estándar: Redirecciones tras un `login` o procesar un `form`       |
| `303`  | `See Other`          | Común en `APIs` para indicar que el resultado está en otra `URI`      |
| `307`  | `Temporary Redirect` | Similar al `302`, pero garantiza que el método `POST`/`GET` no cambie |
## Mejores Prácticas
1. **Usa siempre rutas nombradas**: Evita `redirect('/user/profile/edit')`. Si cambias el `slug` en el archivo de rutas, tendrías que buscar y reemplazar en todo el proyecto. Con `route('user.edit')`, solo cambias el archivo de rutas.
2. **`Redirect after POST`**: Siempre redirige al usuario después de una petición `POST` exitosa. Esto evita el molesto mensaje de "Confirmar reenvío de formulario" si el usuario refresca la página.
3. **Encadenamiento fluido**: Aprovecha la interfaz fluida de Laravel para añadir cabeceras o cookies a la dirección si es necesario: `return redirect('/')->cookie('name', 'value', $minutes);`.
4. **Validación y `Back`**: Cuando uses `back()`, asegúrate de usar `withInput()` para que el usuario no pierda lo que ya escribió en el formulario en caso de error.
