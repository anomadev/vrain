### El Enfoque Profesional: `Form Request`
Aunque es posible validar directamente en el controlador usando `$request->validate()`, la mejor práctica para aplicaciones escalables es crear una clase dedicada. Esto separa la lógica de validación del flujo de ejecución del controlador.
##### Crear un `Form Request`
Se utiliza el comando de `Artisan` para generar la clase en `app/Http/Requests`:
```bash
php artisan make:request StorePostRequest
```
##### Estructura de la Clase
Un `Form Request` contiene dos métodos fundamentales:
- **`authorize()`**: Determina si el usuario tiene permiso para realizar esta petición. Debe devolver un booleano (`true` o `false`).
- **`rules()`**: Define las reglas de validación que se aplicarán a los datos de entrada.
```php
public function rules(): array
{
    return [
        'title' => 'required|unique:posts|max:255',
        'body' => 'required',
    ];
}
```
### Implementación en el Controlador
Una vez creado el `Form Request`, solo necesitas inyectarlo como una dependencia en el método de tu controlador. Laravel se encargará de validar los datos automáticamente antes que el código dentro del método se ejecute.
```php
public function store(StorePostRequest $request)
{
    // Si llegas aquí, los datos ya son válidos.
    $validated = $request->validated();
    
    Post::create($validated);
}
```
### Reglas de Validación Comunes
Laravel incluye decenas de reglas listas para usar:
- **`required`**: El campo no puede estar vacío.
- **`email`**: Debe ser un formato de correo válido.
- **`unique:table,column`**: Asegura que el valor no exista ya en la base de datos.
- **`confirmed`**: Útil para contraseñas; busca un campo con el `subfijo` `_confirmation`.
- **`sometimes`**: Valida el campo solo si está presente en la petición.
### Gestión de Errores y UX
Cuando la validación falla:
1. **Redirección Automática**: Laravel devuelve al usuario a la página anterior.
2. **Mensajes de Error**: Los errores se `"flashean"` a la sesión y están disponibles en la variable `$errors` en `Blade`.
3. **Persistencia de Datos**: Usa la función `old('field')` para que el usuario no tenga que rellenar todo el formulario de nuevo.
```php
<input type="text" name="title" value="{{ old('title') }}">
@error('title')
    <div class="alert alert-danger">{{ $message }}</div>
@enderror
```

## Buenas Prácticas
1. **Usar `$request->validated()`**: En lugar de `$request->all()`, usa siempre el método `validated()`. Esto te asegura trabajar únicamente con los datos que han pasado el filtro de tus reglas, evitando inyecciones de campos no deseados.
2. **Mensajes Personalizados**: Puedes sobrescribir el método `messages()` en tu `Form Request` para devolver textos más amigables que los predeterminados.
3. **Lógica de Autorización**: Si la lógica para `authorize()` es compleja, considera usar `Policies` o `Gates` dentro de ese método para mantener la coherencia con el sistema de permisos de Laravel.
