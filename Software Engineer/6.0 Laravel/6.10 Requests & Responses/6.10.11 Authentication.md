El sistema de autenticación de Laravel es uno de los más robustos y completos en el ecosistema de PHP. Su arquitectura está diseñada para manejar desde el inicio de sesión tradicional basado en sesiones hasta la autenticación de `APIs` mediante `tokens`, todo bajo una configuración centralizada y extensible.

### Conceptos Fundamentales: `Guards` y `Providers`
Para entender la autenticación en Laravel, es imperativo distinguir entre `Guards` (Guardias) y `Providers` (Proveedores), definidos en el archivo `config/auth.php`.
- **`Guards`**: Definen cómo se autentican los usuarios para cada petición. Por ejemplo, el guardia `web` utiliza `cookies` y sesiones, mientras que el guardia `sanctum` utiliza `tokens` de `APIs`.
- **`Providers`**: Definen quién es el usuario y de dónde se recupera. Por defecto, Laravel utiliza el proveedor de `eloquent` para buscar usuario en la base de datos mediante el modelo `User`.

### Autenticación Manual
Aunque existen `kits` de inicio, entender el flujo manual es vital para cualquier arquitectura personalizada. El `facade Auth` es el punto de entrada principal.
##### Intento de Inicio de Sesión
El método `attempt` acepta un `array` de credenciales (usualmente email y `password`). Laravel se encarga de buscar al usuario, verificar el `hash` de la contraseña y, sí es correcto, iniciar una sesión.
```php
use Illuminate\Support\Facades\Auth;

public function authenticate(Request $request)
{
    $credentials = $request->validate([
        'email' => ['required', 'email'],
        'password' => ['required'],
    ]);

    if (Auth::attempt($credentials)) {
        $request->session()->regenerate(); // Previene ataques de fijación de sesión

        return redirect()->intended('dashboard'); // Redirige a la URL que el usuario intentó visitar
    }

    return back()->withErrors([
        'email' => 'Las credenciales proporcionadas no coinciden con nuestros registros.',
    ])->onlyInput('email');
}
```
##### El Usuario Autenticado
Una vez iniciada la sesión, se puede acceder a la instancia del usuario o verificar su estado:
- **`Auth::user()`**: Devuelve el modelo del usuario actual.
- **`Auth::check()`**: Devuelve el booleano indicando si el usuario está autenticado.
- **`Auth::id()`**: Devuelve el `ID` del usuario autenticado (más eficiente que pedir el objeto completo).

### Protección de Rutas (`Middleware`)
La forma estándar de proteger una ruta o grupo de rutas es mediante el `middleware auth`. Si un usuario no autenticado intenta acceder, el `middleware` lo redigirá automáticamente a la ruta nombrada `login`.
```php
Route::get('/profile', function () {
    // Solo usuarios autenticados...
})->middleware('auth');
```

### Ecosistema de Autenticación (`Starter Kits`)
Laravel ofrece soluciones "llave en mano" para evitar reconstruir la rueda, cada una con un enfoque distinto:
- **`Laravel Breeze`**: Es la implementación mínima y simple. Utiliza `Blade` o `Inertia` (`Vue`/`React`) y es ideal para proyectos que requieren control total sobre el código.
- **`Laravel Jetstream`**: Un sistema avanzado con características como autenticación de dos factores (2FA), gestión de equipos y sesiones de navegador.
- **`Laravel Fortify`**: Un motor de autenticación sin `frontend`. Ideal si estás construyendo tu propio diseño pero quieres que Laravel maneje la lógica de `login`, registro y `2FA` por detrás.

### Autenticación de `APIs`: `Sanctum` vs. `Passport`
- **`Laravel Sanctum`**: Es el estándar para `SPAs (Single Page Applications)` o aplicaciones móviles sencillas. Utiliza `cookies` para `SPAs` en el mismo dominio o `tokens` ligeros.
- **`Laravel Passport`**: Una implementación completa de una servidor `OAuth2`. Se recomienda solo si necesitas ofrecer una `API` a terceros o integraciones complejas de `OAuth`.

## Buenas Prácticas
1. **Regeneración de Sesión**: Siempre llama a `$request->session()->regenerate()` tras un `login` exitoso para mitigar el riesgo de *`Session Fixation`*
2. **Límite de Intentos (`Rate Limiting`)**: Usa el middleware `throttle` en tus rutas de `login` para prevenir ataques de fuerza bruta.
3. **Contraseñas Seguras**: Nunca manejes el `hashing` manualmente; Laravel usa `Bcrypt` p `Argon2` por defecto a través del `cast hashed` en el modelo.
4. **Uso de `instended()`**: Al redirigir tras el `login`, usa `redirect()->intended()` para mejorar la experiencia de usuario, devolviéndolo a la página exacta que quería ver antes de ser interrumpido por el `login`.
