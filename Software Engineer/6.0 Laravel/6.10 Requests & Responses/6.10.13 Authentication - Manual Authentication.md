## Definiciﾃｳn
La autenticaciﾃｳn manual consiste en utilizar los servicios internos de Laravel (principalmente el facade `Auth`) para verificar credenciales y gestionar sesiones de usuario sin depender de andamiajes automﾃ｡ticos. Se basa en el patrﾃｳn de "Intento y Respuesta", donde la aplicaciﾃｳn evalﾃｺa un conjunto de datos y decide el estado de la sesiﾃｳn.

### El Mﾃｩtodo Central: `Auth::attempt`
El pilar de este sistema es `Auth::attempt()`. Este mﾃｩtodo recibe un array de credenciales y realiza tres acciones automﾃ｡ticas:
1. Busca al usuario por el identificador (ej. `email`).
2. Compara la contraseﾃｱa (`hasheada` en la DB) con el `string` proporcionado.
3. Si coinciden, inicia la sesiﾃｳn y devuelve `true`.

```php
if (Auth::attempt(['email' => $email, 'password' => $password], $remember)) {
    // El usuario ahora estﾃ｡ autenticado.
    $request->session()->regenerate();
    return redirect()->intended('dashboard');
}
```

### Conceptos de Flujo Crﾃｭtico
- **`intended()`**: Redirige al usuario a la URL que intentﾃｳ visitar originalmente antes de ser interceptado por el middleware de autenticaciﾃｳn. Es vital para una buena experiencia de usuario (UX).
- **Regeneraciﾃｳn de Sesiﾃｳn**: Siempre se debe llamar a `$request->session()->regenerate()` tras un `login` exitoso. Esto previene ataques de **`Session Fixation`** al asignar una nueva ID de sesiﾃｳn al usuario reciﾃｩn autenticado.
- **`Persistent Login (Remember Me)`**: El segundo argumento booleano de `attempt` permite mantener la sesiﾃｳn abierta indefinidamente mediante una cookie de "recuﾃｩrdame".

### El Proceso de `Logout` (Cierre de Sesiﾃｳn)
Cerrar sesiﾃｳn requiere un protocolo de limpieza para asegurar que no queden rastros de la sesiﾃｳn anterior en el servidor o el navegador.

```php
public function logout(Request $request)
{
    Auth::logout();
    $request->session()->invalidate();
    $request->session()->regenerateToken(); // Limpia el token CSRF
    return redirect('/');
}
```

### Seguridad y Buenas Prﾃ｡cticas (Punto Crﾃｭtico)
- **`Throttle` (Limitaciﾃｳn de Intentos)**: Nunca expongas una ruta de `login` manual sin un middleware de limitaciﾃｳn. Laravel ofrece el middleware `throttle:6,1` (6 intentos por minuto) para mitigar ataques de fuerza bruta.
- **Validaciﾃｳn de Credenciales**: No pases el objeto `$request->all()` directamente a `attempt`. Usa `$request->only(['email', 'password'])` para asegurar que solo se procesen los campos necesarios.
- **`CSRF Protection`**: Es obligatorio que el formulario de `login` incluya `@csrf`. Sin esto, el sistema es vulnerable a falsificaciones de peticiﾃｳn.

- **`Auth::attempt`** es el portero que revisa tu identificaciﾃｳn (email) y tu clave secreta (`password`).
- **`Regenerate Session`** es como si, una vez dentro, el portero te quitara el pase temporal y te diera una pulsera nueva (ID de sesiﾃｳn) para que nadie pueda usar el pase que dejaste afuera.
- **`Intended`** es como si el portero recordara que ibas directo a la zona VIP (`Dashboard`) y te guiara allﾃｭ de inmediato, en lugar de dejarte en la entrada.
