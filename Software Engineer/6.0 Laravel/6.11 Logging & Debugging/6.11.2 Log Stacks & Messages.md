Los `Log Stacks` (pilas de registro) son una funcionalidad de Laravel que permite agrupar múltiples canales de `log` bajo un solo nombre. Esto permite que, con una sola instrucción de código, un mensaje se distribuya simultáneamente a diversos destino (como un archivo local, `Slack` y un servicio externo), facilitando la redundancia y la categorización de los mensajes  según su importancia.

## Conceptos Clave
- **Canal (`Channel`)**: Un destino individual para los `logs` (ej. `daily`, `slack`, `papertrail`). 
- **`Stack` (Pila)**: Un contenedor virtual que combina varios canales. Es el "canal por defecto" en la mayoría de las aplicaciones Laravel.
- **`Tap`**: Una forma de personalizar las instancias de `Monolog` antes de que el mensaje sea procesado por un canal específico.
- **`Bubble`**: Concepto de `Monolog` donde un mensaje "sube" por los niveles de `log`. Si un canal tiene un nivel mínimo de `error`, ignorará los mensajes de nivel `info`. 
- **Mensajes con Contexto**: La capacidad de pasar un `array` de datos adicionales para enriquecer la información del evento sin ensuciar el `string` del mensaje.

## Ejemplos Prácticos
##### Configuración de un `Stack` en `logging.php`
```php
'channels' => [
    'stack' => [
        'driver' => 'stack',
        'channels' => ['daily', 'slack'], // Messages sent here go to both
        'ignore_exceptions' => false,
    ],

    'daily' => [
        'driver' => 'daily',
        'path' => storage_path('logs/laravel.log'),
        'level' => 'debug',
        'days' => 14,
    ],

    'slack' => [
        'driver' => 'slack',
        'url' => env('LOG_SLACK_WEBHOOK_URL'),
        'username' => 'Laravel Log',
        'level' => 'critical', // Only critical or higher goes to Slack
    ],
],
```

##### Uso de mensajes estructurados
```php
use Illuminate\Support\Facades\Log;

// This message will go to the 'daily' file but NOT to Slack (level is info)
Log::info('User updated profile', [
    'user_id' => $user->id,
    'changed_fields' => ['email', 'timezone']
]);

// This message will go to BOTH daily and Slack (level is critical)
Log::critical('Database connection lost on production server', [
    'cluster' => 'us-east-1',
    'attempt' => 3
]);
```

## Analogía
Imagina un Sistema de Megafonía en una empresa.
- Los Canales son los altavoces individuales en la oficina, el almacén y el móvil del jefe.
- El `Stack` es el micrófono principal en recepción.
- Cuando el recepcionista anuncia "Hay café en la cocina" (nivel `info`), solo se escucha en la oficina. Pero si el recepcionista grita "Fuego!" (nivel `critical`), el sistema está configurado para que suene en los altavoces de la oficina, del almacén y demás envíe una alerta automática al móvil del jefe.

## Buenas Prácticas
- **Niveles de Filtrado**: Configura niveles mínimos por canal dentro del `stack`. No querrás que `Slack` se inunde con mensajes de `debug`, pero sí que registre los `error` o `critical`.
- **Log Contextual**: Siempre pasa datos variables en el `array` de contexto, no los concatenes en el `string` del mensaje. Esto facilita la búsqueda en herramientas como `Elasticsearch` o `Datadog`.
- **Evitar redundancia innecesaria**: No satures el `stack` con demasiados canales pesados que puedan afectar el rendimiento de la petición (`I/O`).
- **`Environment-Specific Stacks`**: Usa diferentes configuraciones de `stack` para `local` (solo archivo) y `production` (archivo + monitorización externa).

## Escenarios y Aplicaciones
- **Notificaciones de Emergencia:** Usar un `stack` que incluya `Slack` o Telegram para errores `500` en producción.
- **Auditoría y `Debug`:** Un `stack` que guarde `debug` en archivos locales para desarrollo, pero solo `info` en producción para auditoría de acciones de usuario.
- **Cumplimiento Legal:** Enviar logs de transacciones financieras a un archivo local y simultáneamente a un servicio de almacenamiento inmutable en la nube para auditoría.

## Notas Personales
- Si usas el driver `stack` y olvidas definir los canales que lo componen en el array `channels`, Laravel no lanzará error pero no registrará nada.
- Recuerda que `Log::stack(['canal1', 'canal2'])->info(...)` permite crear `stacks` sobre la marcha en el código sin definirlos en el archivo de configuración.
- El orden de los canales en el `stack` no suele importar, a menos que uses lógica de procesamiento personalizada (`Taps`).
