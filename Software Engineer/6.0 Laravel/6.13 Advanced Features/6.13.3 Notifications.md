## Definición
Las notificaciones en Laravel son clases dedicadas que representan un mensaje informativo enviado a un usuario o entidad. A diferencia de los `Mailables` (enfocados solo en correo), las Notificaciones son multicanal por naturaleza; pueden ser entregadas a través de conductores (`drivers`) como `mail`, `database`, `broadcast`, `sms` (vía `Vonage` o `Twilio`) y `slack`, o incluso canales personalizados creados por la comunidad.

## Conceptos Clave
- **`Trait Notifiable`**: Se añade a los modelos (usualmente `User`) para permitirles recibir notificaciones mediante el método `$user->notify()`.
- **Método `via()`**: Define por qué canales se debe enviar la notificación en un momento dado (puede ser dinámico según preferencias del usuario).
- **Formateo por Canal**: Cada canal tiene su propio método de representación (ej. `toMail`, `toDatabase`, `toSlack`). 
- **`On-Demand Notifications`**: Capacidad de enviar una notificación a una dirección o número "al vuelo" sin tener un usuario en la base de datos mediante el `facade` `Notication::route()`.
- **`Database Channel`**: Almacena la notificación en una tabla de la base de datos para mostrar alertas dentro de la aplicación (`UI`).

## Ejemplos Prácticos

##### Clase de Notificación Multicanal
```php
namespace App\Notifications;

use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Notifications\Messages\MailMessage;
use Illuminate\Notifications\Notification;

class InvoicePaid extends Notification implements ShouldQueue
{
    use Queueable; // Allows the notification to be processed in the background

    public function __construct(public $invoice) {}

    // Determine which channels to use
    public function via(object $notifiable): array
    {
        return ['mail', 'database'];
    }

    // Format for Email
    public function toMail(object $notifiable): MailMessage
    {
        return (new MailMessage)
                    ->subject('Invoice Paid')
                    ->line('Your payment for invoice #' . $this->invoice->id . ' was successful.')
                    ->action('View Invoice', url('/invoices/' . $this->invoice->id))
                    ->line('Thank you for your business!');
    }

    // Format for Database (Stored as JSON)
    public function toArray(object $notifiable): array
    {
        return [
            'invoice_id' => $this->invoice->id,
            'amount' => $this->invoice->total,
        ];
    }
}
```

##### Disparo de la Notificación
```php
// Using the Notifiable trait
$user->notify(new InvoicePaid($invoice));

// Using the Notification Facade (for multiple users)
use Illuminate\Support\Facades\Notification;

Notification::send($users, new InvoicePaid($invoice));
```

## Analogía
Imagina que eres un Director de Orquesta que necesita dar un aviso a todos los músicos.
- La notificación es el mensaje: "Ensayo mañana a las 8 AM".
- El `Trait` `Notifiable` es el instrumento que cada músico tiene para recibir avisos.
- El método `via()` es el medio: a algunos les manda un `WhatsApp (SMS)`, a otros les gritas (`Broadcast/Real-time`) y a otros les dejas una notas en su casillero (`Database`). Tú no escribes tres mensajes distintos; escribes la instrucción una vez y dejas que el sistema de mensajería del teatro (Laravel) se encargue de traducirlo al formato correcto para cada receptor.

## Buenas Prácticas
- **Usar Colas (`ShouldQueue`)**: Siempre que envíes notificaciones externas (Mail, SMS, Slack), implementa `ShouldQueue` para evitar que los tiempos de respuesta de la red bloqueen la experiencia del usuario.
- **Personalizar el Remitente**: En el modelo `User`, puede definir métodos como `routeNotificationForMail` para personalizar a qué email se envía la notificación en lugar de usar el campo `email` por defecto.
- **`Database Pruning`**: Si usas el canal `database`, implementa una tarea programada para limpiar notificaciones antiguas y evitar que la tabla crezca infinitamente.
- **Localización**: Usa el método `locale()` al enviar la notificación para asegurarte de que el mensaje llegue en el idioma preferido del destinatario: `$user->notify((new InvoicePaid($invoice))->locale('es'));`.

## Escenarios y Aplicaciones
- **Alertas de Seguridad**: Enviar un código de verificación (`OTP`) por SMS o un aviso de "Inicio de sesión desde dispositivo desconocido" por email.
- **Sistemas de Cobro**: Avisar automáticamente cuando una suscripción ha fallado o una factura ha sido pagada.
- **Interacción Social**: Notificar a un usuario cuando alguien comento su publicación (`In-app` vía `Database` + `Real-time` vía `Broadcast`).
- **Monitoreo de Infraestructura**: Enviar errores críticos del sistema directamente a un canal de `Slack` del equipo de ingeniería.

## Notas Personales
- El canal `database` no crear la migración automáticamente en versiones antiguas. Asegúrate de ejecutar `php artisan make:notifications-table` si la tabla no existe.
- Al trabajar con `SPA` (`Vue/React`), el canal `broadcast` combinado con `Laravel Echo` es la forma más limpia de mostrar notificaciones `"toast"` en tiempo real sin recargar la página.
- Las notificaciones enviadas a través de colas serializan los modelos. Si los datos del modelo cambian entre el despacho y la ejecución del `Job`, el `Worker` cargará la versión más fresca de la `DB`.
