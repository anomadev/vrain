##  Definición
`Laravel Sail` es un entorno de desarrollo basado en `Docker` que se integra directamente con las aplicaciones Laravel. Utilizan un archivo `docker-compose.yml` para definir y levantar contenedores que incluyen `PHP`, `MySQL`, `Redis`, `Meilisearch` y otros servicios necesarios. `Sail` permite ejecutar comandos de `Artisan`, `Composer` y `Node.js` "dentro" de estos contenedores de forma transparente para el desarrollador.

## Conceptos Clave
- `Docker & Docker Compose`: La tecnología subyacente que crea "paquetes" aislados para cada servicio de la aplicación.
- `Runtime`: El contenedor principal que ejecuta PHP y la aplicación. `Laravel Sail` permite elegir entre diferentes versiones de PHP (8.2, 8.3, 8.4).
- `Sail Binary`: Un `script` ubicado en `vendor/bin/sail` que actúa como un puente para redirigir tus comandos al contenedor de `Docker` correcto.
- `Environment Parity`: La garantía de que todos los desarrolladores del equipo están usando exactamente las mismas versiones de software, eliminando el "en mi maquina funciona".
- `Customization`: La capacidad de "expulsar" (*`publish`*) los archivos de `Docker` para personalizar las imágenes base (`Dokerfiles`).

## Ejemplos Prácticos

##### Iniciar el entorno
```bash
# Start the containers in the background
./vendor/bin/sail up -d

# Stop the containers
./vendor/bin/sail stop
```

##### Ejecutar comandos comunes
En lugar de `php artisan`, usas `sail`
```bash
# Run migrations inside the container
./vendor/bin/sail artisan migrate

# Install a new composer package
./vendor/bin/sail composer require laravel/sanctum

# Run frontend assets compilation
./vendor/bin/sail npm run dev
```

##### Alias para mayor rapidez
Para no escribir `./vendor/bin/sail` cada vez, se recomienda añadir un alias en tu archivo `.zshrc` o `.bashrc`:
```bash
alias sail='[ -f sail ] && sh sail || sh vendor/bin/sail'
```

## Analogía
Imagina que eres un **Pintor** que necesita herramientas muy específicas (pinceles de marta, óleos franceses, un caballete de roble).
- **Instalación tradicional:** Tienes que ir a buscar cada herramienta a diferentes tiendas y guardarlas en tu casa. Si te mudas o un amigo quiere ayudarte, tiene que hacer todo el proceso de nuevo.
- **`Laravel Sail`:** Es como si te entregaran un **Maletín de Pintura Profesional (Contenedor)**. Todo lo que necesitas está dentro del maletín, organizado y listo. No importa en qué casa estés, solo abres el maletín y tienes exactamente las mismas herramientas que usaste ayer.

## Buenas Prácticas
- **No instales PHP local**: Si usas `Sail`, evita tener `PHP` o `Node` instalados directamente en tu sistema operativo para no causar conflictos de versiones.
- **Usa Volúmenes**: Asegúrate de que los datos de tu base de datos persistan usando volúmenes de `Docker` (`Sail` lo hace por defecto en `docker-compose.yaml`).
- **Optimiza recursos**: Si no necesitas servicios como `Selenium` o `Meilisearch` para un proyecto pequeño, elimínalos de tu archivo `docker-compose.yml` para ahorrar memoria `RAM`.
- Versiones de PHP: Si necesitas probar cómo se comporta tu app en una versión de PHP distinta, simplemente cambia la imagen del `context` en el archivo de configuración y reinicia `Sail`.

## Escenarios y Aplicaciones
- **`Onboarding` de desarrolladores:** Un nuevo programador puede clonar el repositorio y estar trabajando en minutos con solo ejecutar `sail up`.
- **Proyectos con múltiples versiones:** Trabajar en un proyecto con PHP 7.4 y otro con PHP 8.4 simultáneamente en la misma máquina sin que se interfieran.
- **Pruebas Integradas:** Ejecutar `tests` en un entorno limpio que incluye base de datos y servicios de caché reales en lugar de `mocks`.

## Notas Personales
- Si usas Windows, **es obligatorio usar `WSL2 (Windows Subsystem for Linux)`** y clonar tu código dentro del sistema de archivos de Linux para evitar problemas de rendimiento extremos con `Docker`.
- A veces, el puerto 80 (el de la web) está ocupado por otros servicios. Puedes cambiarlo fácilmente en el archivo `.env` usando la variable `APP_PORT`.
- `Sail` es excelente para desarrollo, pero **no está diseñado para producción**. Para eso, existen herramientas como `Laravel Forge` o `Vapor`.
