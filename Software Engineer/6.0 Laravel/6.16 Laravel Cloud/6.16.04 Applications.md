Representan aplicaciones Laravel desplegadas en Laravel Cloud. Se pueden crear a partir de kits de Laravel o de repositorios existentes en `GitHub`, `Bitbucket` o `GitLab`. Las aplicaciones pueden contener muchos entornos y recursos.

Ambos métodos requieren seleccionar una región donde se implementará su aplicación. Una aplicación solo se puede implementar en una única región a la vez. Laravel Cloud creará automáticamente un entorno predeterminado para tu aplicación independientemente del método de creación que elijas.

> [!info] Una vez creadas, las solicitudes no se pueden transferir entre organizaciones. Las solicitudes quedan asociadas de forma permanente a la organización en la que se crean.

### Creating from starter kits
Laravel Cloud proporciona las siguientes plantillas de kit de inicio:
- **`Laravel`**
- **`React Starter Kit`**: `React`, `TypeScript`, `Inertia`, `shadcn/ui`
- **`Vue Starter Kit`**: `Vue`, `TypeScript`, `Inertia`, `shadcn-vue`
- **`Livewire Starter Kit`**: `Livewire`, `Laravel Volt`, `Flux UI`

El kit de inicio creará automáticamente un nuevo repositorio en tu cuenta de proveedor `Git` conectada, clonará el código oficial del kit de inicio de Laravel y configurará tu aplicación para su implementación.

### Creating form existing repository
Deberá seleccionar un repositorio de control de versiones desde el que se implementará la aplicación, asignarle un nombre y seleccionar una región. Por lo general, la estructura organizativa de Laravel Cloud debería reflejar la estructura organizativa de su proveedor de control de versiones.

> [!info] Una vez autenticado el proveedor de control de versiones, el repositorio (o los repositorios) seleccionados estarán disponibles para que todos los usuarios de la organización puedan verlos y crear aplicaciones desde ellos.

### Regions
Laravel Cloud actualmente admite las siguientes regiones de aplicación:
- US East (Virginia) / `us-east-1`
- US East (Ohio) / `us-east-2`
- CA Central (Central) / `ca-central-1`
- EU Central (Frankfurt) / `eu-central-1`
- EU West (Ireland) / `eu-west-1`
- EU West (London) / `eu-west-2`
- Asia Pacific (Singapore) / `ap-southeast-1`
- Asia Pacific (Sydney) / `ap-southeast-2`
- Asia Pacific (Tokyo) / `ap-northeast-1`
Para minimizar la latencia, debe asegurarse de que su aplicación y sus recursos asociados se encuentren dentro de la misma región.

### Deleting applicactions
Al eliminar aplicaciones, se eliminarán todos los entornos que contengan. Sin embargo, recursos como bases de datos, almacenes de `key-value` y depósitos de almacenamiento no se eliminan, ya que pueden ser utilizados por otras aplicaciones dentro del entorno. Para eliminar estos recursos, diríjase a la pestaña `Resources` de su organización.
