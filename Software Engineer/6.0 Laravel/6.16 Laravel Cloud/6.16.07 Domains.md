Laravel Cloud proporciona dos tipos de dominios para cada entorno: un domino `laravel.cloud` asignado automáticamente y cualquier dominio personalizado que conectes.

### Cloud domains
A cada entorno se le asigna un dominio gratuito de `laravel.cloud`. Esto resulta ideal para realizar pruebas antes de migrar a tu propio dominio. Se crean a partir del nombre de la aplicación y del entorno. Si cambia alguno de ellos, el dominio cambiará y deberá volver a implementar la aplicación para que el cambio surta efecto.

Dado que los dominios se utilizan con frecuencia para entornos que no son de producción se establece automáticamente un encabezado `X-Robots: noindex, nofollow` para que no sean indexados por los motores de búsqueda. Este encabezado no se configura para dominios personalizados.

### Custom domains
Permite que los visitantes accedan a tu aplicación a través de tu propia dirección. Laravel Cloud aprovisiona y renueva automáticamente los certificados SSL para cada dominio personalizado, de modo que su sitio web ser sirve a través de HTTPS.

#### Add a custom domain
- `Wildcard support`: decidir si debe aceptar tráfico en todos los subdominios.
- `Redirect behavior`: Si Laravel Cloud debería redirigir `www.site.com` a `site.com` (o viceversa).
- `Downtime preference`: Si es aceptable un breve tiempo de inactividad durante la configuración, o si necesita una transición ininterrumpida.
- `Cloudflare DNS settings`: Si su DNS es administrado por `Cloudflare` y si su nombre de host ya está siendo enrutado a través de la red de `Cloudflare`.

1. Agregue el dominio a Laravel Cloud
2. (Opcional) Verifique previamente la propiedad y el certificado SSL
3. Apunta el domino a Laravel Cloud
4. Actualizar hasta que se conecte

#### Verification paths
Laravel Cloud admite dos métodos de verificación dependiendo de si puede tolerar una breve interrupción del servicio mientras se implementa los cambios de DNS: 
- `Real-time verification (flexible)`: Apunta el dominio a Laravel Cloud, verifica la propiedad y emite el certificado SSL, el método más sencillo pero puede haber un periodo breve donde no funcione con HTTPS.
- `Pre-verification (uninterrupted)`: Demuestra la propiedad, emite el certificado SSL antes de redirigir el tráfico. Esto evita interrupciones del servicio, pero requiere configuración previa de registros DNS adicionales.

### DNS records you will see in Cloud
Al abrir la configuración de un dominio, Laravel Cloud mostrará los registros DNS necesario para su configuración.

#### Origin records
Son las entradas DNS principales que le indican a Internet que envíe el tráfico de su dominio a Laravel Cloud:
- `A` asocia tu dominio a una dirección IP y es común cuando no utilizas el servicio de DNS de `Cloudflare`.
- `CNAME` asigna tu domino a otro nombre de host y es común cuando utiliza `Cloudflare` DNS.

#### Ownership verification record
Durante la pre-verificación, Laravel Cloud puede redirigir un registro `TXT` temporal para confirmar que controla el dominio. Este registro tiene el siguiente aspecto: `TXT _cf-custom-hostname.<your-domain> -> <value>`. 
Una vez confirmada la propiedad, este registro ya no es necesario, aunque dejarlo no causará ningún problema.

#### SSL validation records
Para emitir un certificado SSL Laravel Cloud puede requerir uno o más registros de validación:
- `TXT _acme-challenge.<your-domain> -> <value>`
- `CNAME _acme-challenge.<your-domain> -> <value>` (común para delegación de comodines/DCV)
Si utiliza compatibilidad con `wildcard` mantenga el registro `CNAME` de delegación de DCV de forma permanente. Este registro permite que Laravel Cloud renueve automáticamente su certificado SSL; eliminarlo impedirá futuras renovaciones.

### Understanding verification status
El dominio está completamente conectado una vez que se verifican tres componentes:
- `Hostname` - se confirma la propiedad del dominio
- `SSL` - Se ha emitido y activado un certificado SSL
- `Origin` - DNS enrutado correctamente a Laravel Cloud
A medida que avanza la verificación, el panel mostrará uno de los siguientes estados:
- `Verifying ownership` - esperando validación de registros DNS y SSL
- `Verifying` - Se ha confirmado el certificado SSL, pero aún no se ha detectado enrutamiento
- `Connected` - Componentes verificados y dominio activo
- `Time out` - Verificación no se completó dentro del plazo permitido
Laravel Cloud reintenta las comprobaciones de verificación durante aproximadamente 12 horas antes de que se agote el tiempo de espera, por lo que es necesario volver a activarlas manualmente durante este período.

### Wildcard domains
Un dominio comodín (`wildcard`) permite cualquier subdominio para enrutar a su `environment` sin configurar cada uno individualmente. Requieren verificación previa a menos que el nombre de host esté siendo gestionado por la red de `Cloudflare`.
- Configure el dominio raíz `example.com` de modo que sus subdominios queden cubiertos.
- Agregue el registro de delegación `DCV` que se muestra en la UI. Este permite a Laravel Cloud seguir emitiendo y renovado certificados SSL.
Si su DNS está gestionado por `Cloudflare`, asegúrese de que los registros de delegación `DCV` en `_acme-challenge` estén configurados solo como DNS. El uso de un proxy para estos registros bloqueará la emisión de certificados.

### Managing domains
Desde la configuración del `environment`, puede desvincular y reasignar dominios entre entornos de la misma organización. También puede establecer un dominio raíz verificado como dominio principal para un `environment`. 

### Limitations
No existen limitaciones en cuanto al número de `environments` en la nube a los que se les puede asignar un dominio personalizado. Cada plan incluye varios dominios personalizado. Cada plan incluye varios dominios personalizado en toda su organización.
- `Starter`: 10
- `Growth`: 50
- `Business`: 250
- `Enterprise`: Personalizada
Los dominios personalizados adicionales que superen la cantidad incluida en su plan se facturaran a 0.25 DLS por dominio al mes. Si requiere un caso especial que requiera muchos dominios, póngase en contacto con Laravel Cloud para aumentar sus límites.
