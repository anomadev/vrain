### Introducción 
- **Cuando ocurren?** Los despliegues se ejecutan cada vez que hay un código por liberar, nuevos recursos que adjuntar o configuraciones del entorno que actualizar.
- **El Mecanismo**: Al activarse, Laravel Cloud toma el código y los ajustes del entorno, construye una imagen Docker optimizada con la versión de PHP elegida y ejecuta los comando de construcción (`build`) y despliegue (`deploy`).
- **`Zero Downtime` (Cero tiempo de inactividad)**: Una vez que la nueva compilación finaliza con éxito, el despliegue anterior se termina de manera progresiva y contralada (*`gracefully terminated`*), permitiendo que los procesos HTTP o tareas en ejecución terminen antes de apagarse, mientras el nuevo despliegue entra en línea inmediatamente sin interrumpir el servicio.

### Deploy options
Laravel Cloud ofrece tres mecanismos principales para detonar un despliegue:
#### Push to deploy
- Cada que envías (`push`) nuevo código a la rama remota de Git configurada para ese entorno, se dispara automáticamente un despliegue. Esta opción viene activada por defecto en todos los entornos, pero se puede modificar en `Settings > Deployments`.
#### Deploy hooks
- Permite dispara despliegues mediante un `endpoint HTTP` (un `webhook`). Al activarlo, la plataforma genera una URL única a la cual se le debe enviar una petición de tipo `POST`.
- `Control Commits`: Por defecto, el `hook` despliega el último `commit` de la rama del entorno. Sin embargo, se puede desplegar un `commit` específico pasando el parámetro `commit_hash` en la `query` (ej. `?commit_hash=abc123def456`). Dicho `commit` debe pertenecer a la rama asignada.
#### Example using GitHub Actions
- **Ventaja competitiva (Análisis Crítico)**: A diferencia de los esquemas tradicionales de `CI/CD` (donde tu `pipeline` en GitHub debe instalar PHP, correr `Composer`, compilar `assets` con `Node` y luego empaquetar), con Laravel Cloud el `pipeline` externo solo actúa como un `trigger`.
- **Flujo simplificado**: Al enviar `POST` a través de `GitHub Actions` con el secreto `LARAVEL_CLOUD_DEPLY_HOOK` y el `hash` (`${{ github.sha }}`), Laravel Cloud se encargara de todo el trabaja pesado de forma nativa e interna (clonar el `commit`, resolver las dependencias, compilar y publicar sin caídas).
#### Manual
- Es posible forzar un despliegue en cualquier momento desde el `dashboard web` haciendo clic en el botón `Deploy`. Asimismo, al alterar parámetros de infraestructura o variables, el botón `Save & Deploy` guardará los cambios y compilará la nueva versión manualmente.

### Troubleshooting
#### Framework or rutime version not supported
- **Restricciones del Core de Laravel**: Laravel Cloud exige de forma mandatoria utilizar Laravel 9 o superior. Adicionalmente, requiere que estés utilizando la última versión *`minor`* (de mantenimiento) de dicho `tag`. Los mínimos estrictos requeridos son:
	- Laravel 11: v11.41.3
	- Laravel 10: v10.48.28
	- Laravel 9: v9.52.20
- **Solución al error común**: Si el `deployment` falla arrojando el mensaje de que el paquete `laravel/framework` en el `composer.lock` no está soportando, se corrige de inmediato ejecutando localmente en la terminal el comando: `composer update laravel/framework` y subiendo el archivo `lock` actualizado.
