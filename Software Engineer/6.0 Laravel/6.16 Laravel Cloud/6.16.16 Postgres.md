### Introduction 
- Laravel Cloud ofrece una solución de base de datos `Serverless PostgreSQL` totalmente gestionada, diseñada específicamente para acoplarse a la naturaleza elástica y de rápido crecimiento de la plataforma.
- Modelo `Serveless` vs. Tradicional: A diferencia de la opción de MySQL (que usa instancias *`always-on`* tradicionales con cómputo fijo). `Postgres` opera bajo un modelo puramente `Serverless`. Esto significa que los recursos del cómputo (CPU y Memoria) se asignan dinámicamente en tiempo real según el volumen de consultas que reciba la aplicación.

### How scaling works
- Cómputo bajo Demanda: El sistema mide la carga en unidades de cómputo. Cuando la aplicación experimenta un pico de tráfico, los recursos de procesamiento de la base de datos se elevan de inmediato para absorber el impacto sin intervención manual.
- Escalado a Cero (`Scale to Zero`): Cuando la base de datos deja de recibir consultas por completo (por ejemplo, en entorno de desarrollo o `Preview Environments` durante la noche), el cómputo se apaga y escala a cero. Durante este periodo de inactividad, el consto por procesamiento es de $0. El cliente solo paga por el almacenamiento en disco de los datos guardados.
- Tiempo de Despertar (`Wake-up Latency`): Al recibir una nueva consulta HTTP que requiere interactuar con la base de datos dormida, `Postgres` se "despierta" de manera automática. Este proceso de arranque en frío (*`cold-start`*) toma típicamente entre 1 y 3 segundos para la primera conexión, volviendo a la latencia estándar en las peticiones subsecuentes.

### Creating a PostgreSQL database
El aprovisionamiento se realiza de forma visual desde el lienzo de infraestructura (`Add database > PostgreSQL`), configurando los siguientes parámetros esenciales:
1. `Database Name`: Identificador lógico dentro de tu organización.
2. `Maximum Computo` (Capacidad Límite): Define el techo máximo de unidades de cómputo que el motor puede consumir durante ráfagas de tráfico intenso. Actúa como un freno de costos, asegurando que una consulta ineficiente o un ataque no inflen la facturación de forma descontrolada.
3. `Region`: Debe coincidir estrictamente con la ubicación geográfica de tu `App cluster` para evitar penalizaciones por latencia de red y cargos por transferencia de datos entre regiones de AWS.

### Database backups
- **Respaldo automatizado continuo**: A diferencia de los esquemas de `snapshots` diarios fijos, el motor `Severless Postgres` de Laravel Cloud implementa una arquitectura de respaldo continuos con capacidad de recuperación en un punto en el tiempo (*`Point-in-Time Recovery o PITR`*).
- **Ventana de Retención**: El estándar de retención de datos históricos es de 7 días, permitiendo restaurar el estado exacto de la base de datos ante errores humanos o corrupciones de datos.
- ***Nota sobre Entornos Efímeros***: Para entornos de pruebas efímeros o de `Preview`, los respaldos se pueden desactivar por completo para optimizar costos de almacenamiento secundario.

### Monitoring databases cluster metrics
La plataforma expone gráficas de analíticas en tiempo real en la pestaña de monitoreo. Dado el comportamiento elástico del recurso, las métricas clave cambian: en lugar de ver el porcentaje fijo de una CPU física, se monitorea la fluctuación del uso de Unidades de Cómputo `Serverless` a lo largo del tiempo, facilitando la toma de decisiones para ajustar el *`Maximum compute`*.

### Deleting databases
La eliminación de base de datos `PostgreSQL` purga de inmediato el almacenamiento asignado y detiene cualquier cálculo de métricas. Es una acción destructiva e irreversible que requiere confirmación explícita en la consola.

### From your application
- **Inyección Nativa de Variables**: Al trazar la línea de conexión en el lienzo de infraestructura entre la base de datos `PostgreSQL` y tu entorno, Laravel Cloud inyecta de forma automática las variables de entorno de conexión estándar: `DB_CONNECTION=pgsql`, `DB_HOST`, `DB_PORT=5432`, `DB_DATABASE`, `DB_USERNAME` y `DB_PASSWORD`.
- **Compatibilidad de `Framework`**: Tu archivo de configuración `config/database.php` en Laravel procesa estas variables nativas de inmediato, permitiendo que `Eloquent` y las migraciones funcionen sin configuraciones ni extensiones adicionales de código.

### From your local machine
Privacidad por Defecto: Las bases de datos se instalan de forma aislada dentro de la red privada de la infraestructura, protegiéndolas de accesos externos maliciosos.

### PostgreSQL public endpoints
Si necesitas conectar clientes de escritorio locales (como TablePlus, DBeaver o PgAdmin), debes activar explícitamente el interruptor `"Public endpoint"` en la configuración de la base de datos dentro del panel de Laravel Cloud. Esto genera una URL de conexión segura accesible desde internet.

### PostgreSQL SSL connections
Por políticas estrictas de seguridad de datos, `PostgreSQL` rechaza cualquier intento de conexión externa que no utilice cifrado `SSL/TLS`. Es obligatorio configurar tu cliente de base de datos local para exigir `SSL` activo (`sslmode=require`) al apuntar al endpoint público.


Este comportamiento convierte a `Serverless PostgreSQL` en la opción ideal para optimizar al máximo los costos de infraestructura en empresas que manejan arquitectura multi-entorno, permitiendo tener decenas de entornos de desarrollo y `previews` operando con bases de datos independiente sin pagar por cómputo ocioso.
