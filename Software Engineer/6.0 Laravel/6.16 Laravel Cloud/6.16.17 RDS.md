## Introduction
- Para las organizaciones que operan dentro de Laravel Private Cloud, la plataforma ofrece la opción de aprovisionar bases de datos relacionales gestionadas directamente a través de Amazon RDS (`Relational Database Service`).
- **Entorno Aislado**: Estas bases de datos no corren en la infraestructura compartida tradicional de Laravel Cloud, sino se aprovisionan dentro de la VPC privada y la cuenta de AWS dedicada de tu organización, completamente aisladas de internet público y de otros clientes.

### Supported Engines
Laravel Private Cloud permite elegir entre los dos motores de base de datos relacionales más populares del ecosistema:
- `MySQL` (Versiones certificadas por la plataforma)
- `PostgreSQL` (Versiones certificadas por la plataforma)
- *Nota de Arquitectura: A diferencia del `Postgres` estándar de Laravel Cloud, las bases de datos de PostgreSQL aquí se gestionan mediante instancias RDS dedicadas en lugar del modelo `serverless`, priorizando un rendimiento crudo y predecible para cargas empresariales.*

### Multi-AZ Deployment
- **Alta Disponibilidad de Nivel Empresarial**: Al configurar una base de datos RDS en `Private Cloud`, se puede activar la opción de despliegue Multi-AZ (Múltiples Zonas de Disponibilidad).
- **Mecanismo de Resiliencia**: AWS replica de forma síncrona todos los datos en una instancia en estado de espera (`Stanby replica`) ubicada en una zona de disponibilidad física completamente distinta.
- **`Failover` Automático (Análisis Crítico)**: Si la zona de disponibilidad primaria sufre una interrupción de `hardware`, un corte de energía o problemas de red, Amazon RDS realiza una conmutación por error (`failover`) automática hacia la instancia de espera en cuestión de minutos. Los `endpoints` y las variables de conexión inyectadas en tu aplicación Laravel siguen siendo exactamente los mismos, garantizando una alta tolerancia a fallos con cero intervención manual del equipo de ingeniería.

### Storage Auto-scalling
- **Crecimiento Dinámico del Disco**: Para evitar que la aplicación quede fuera de línea debido a un disco lleno (`Out of Space`), RDS implementa el auto-escalado de almacenamiento.
- **Comportamiento**: Si el espacio disponible car por debajo de los umbrales de seguridad y la carga de datos es constante, RDS incrementa automáticamente el tamaño del volumen de almacenamiento asignado.
- **Freno de Costos**: Al configurar la base de datos, el administrador debe definir un Límite Máximo de Almacenamiento (*`Maximum Storage Threshold`*). Esto actúa como una barrera de seguridad financiera, impidiendo que procesos corruptos, `logs` infinitos o errores de software hinchen el almacenamiento de forma infinita y disparen la facturación.

### Database Backups
- **`Snapshots` Automatizados**: Amazon RDS realiza respaldos diarios completos de la base de datos de forma automática durante una ventana de tiempo de bajo tráfico definida por el usuario.
- **Transacciones Continuas**: Además del `snapshot` diario, se respaldan de forma continua los `logs` de transacciones de la base de datos.
- **`Point-in-Time Recovery (PITR)`**: Gracias a la combinación de `snaphots` y `logs` de transacciones, Private Cloud te permite restaurar la base de datos a cualquier segundo específico dentro de un periodo de retención (típicamente configurable entre 1 y 35 días), ofreciendo la máxima protección contra corrupciones de datos o eliminaciones accidentales de tablas en producción.

### Maintenance Windows
- **Modificaciones Controladas**: Las actualizaciones de seguridad del motor, parches del sistema operativo de la instancia o cambios de `hardware` obligatorios por parte de AWS requieren ventanas de mantenimiento. 
- **Planificación Horaria**: El administrador puede definir una ventana de mantenimiento semanal específica (día y hora en formato UTC). Las modificaciones que puedan provocar interrupciones momentáneas o degradación del servicio se pospondrán para ejecutarse exclusivamente dentro de ese bloque horario protegido.

### Network and Security
- **Aislamiento Perimetral Absoluto**: Por cuestiones estrictas de cumplimiento (*`compliance`*) y seguridad empresarial, las base de datos RDS en Private Cloud no tienen direcciones IP públicas ni `endpoints` expuestos a internet.
- **Acceso de la Aplicación**: El tráfico fluye de forma 100% privada e interna dentro de la VPC hacia el clúster de `Kubernetes` donde corren tus contenedores de Laravel.
- **Conectividad Externa Segura**: Si tu equipo necesita conectar herramientas de administración locales (como TablePlus) o extraer datos para analítica (BI), el acceso no se realiza abriendo un `endpoint` público, sino canalizando el tráfico a través de los puentes seguros configurados en tu Private Cloud, tales como `VPC Peering`, `AWS PrivateLink` o mediante túneles `VPN/Bastion` autorizados en tu propia cuenta de `AWS`.
