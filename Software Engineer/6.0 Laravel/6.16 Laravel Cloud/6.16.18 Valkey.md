### Introduction
- **Propósito:** Laravel Cloud ofrece **Valkey** como su almacén de datos clave-valor en memoria de nivel empresarial.
- **Origen Tecnológico:** Valkey es una bifurcación (_fork_) de código abierto, de alto rendimiento y totalmente compatible con Redis (creada tras el cambio de licencia de este último). Funciona como una alternativa directa y transparente para manejar la capa de caché, sesiones y almacenamiento de datos ultrarrápido en aplicaciones Laravel y Symfony.
- **Modelo Elástico:** Opera bajo un esquema completamente gestionado y optimizado para la nube, integrándose de forma nativa con los flujos de escalado e inmutabilidad de la plataforma.

### Creating a Valkey cache
El aprovisionamiento se realiza de forma visual desde el lienzo de infraestructura (`Add cache > Valkey`), configurando los siguientes parámetros esenciales:
1. **`Cache Name`:** El nombre identificador único del recurso dentro de tu organización.
2. **`Memory Allocation` (Tamaño de Instancia):** Permite seleccionar la cantidad de memoria RAM dedicada para el almacenamiento en caché. Las opciones escalan en niveles estándar (ej. 256 MiB, 512 MiB, 1 GiB, 2 GiB, etc.), donde la capacidad de procesamiento de CPU asignada crece de manera proporcional al tamaño de memoria seleccionado.
3. **`Region`:** Debe coincidir exactamente con la región geográfica de tu clúster de cómputo de la aplicación (`App cluster`) para eliminar latencias de red en consultas en memoria y evitar cargos adicionales por transferencia de datos inter-regional.

### Eviction Policies (Políticas de Desalojo)
Cuando el almacenamiento en caché se llena por completo y alcanza el límite de memoria RAM asignado, Valkey aplica una regla de comportamiento para decidir qué hacer con los datos nuevos y existentes. Puedes configurar estas políticas desde el dashboard de Laravel Cloud:
- **`volatile-lru` (Por defecto):** Elimina las llaves que tienen un tiempo de expiración (TTL) configurado, priorizando desalojar aquellas que se han usado menos recientemente (_`Least Recently Used`_). Es la opción recomendada para un uso estándar de caché de aplicaciones.
- **`allkeys-lru`:** Elimina cualquier llave (tenga o no tenga TTL configurado) basándose en el algoritmo LRU. Útil si usas Valkey única y exclusivamente como caché temporal volátil.
- **`noeviction`:** No elimina ningún dato de forma automática. Si la memoria se llena, Valkey rechazará cualquier operación de escritura y devolverá un error. **Análisis Crítico:** Esta opción es obligatoria si utilizas Valkey para almacenar datos que no puedes permitirte perder (como sesiones de usuario persistentes o contadores de negocio críticos), pero exige un monitoreo estricto de métricas para evitar caídas del servicio por falta de espacio.

### Monitoring Valkey metrics
- El panel de administración de Laravel Cloud expone gráficas de rendimiento específicas en tiempo real para el recurso.
- **Métricas Clave:** Permite supervisar el **Uso de Memoria RAM**, la **Tasa de Aciertos/Fallos de Caché** (_`Cache Hit/Miss Ratio`_) para evaluar la eficiencia de tu código, y las operaciones por segundo (comandos procesados), lo que facilita identificar cuándo es necesario realizar un escalado vertical del recurso.

### Deleting caches
sLa eliminación de una instancia de Valkey es una acción destructiva e irreversible. Remueve de inmediato el recurso del lienzo de infraestructura y purga instantáneamente todos los datos almacenados en memoria, interrumpiendo cualquier sesión activa vinculada.

### From your application
- **Inyección de Variables Automatizada:** Al trazar la línea de conexión en el lienzo entre Valkey y tu entorno, Laravel Cloud inyecta de forma automática las variables de entorno de conexión de caché:

```bash
REDIS_HOST=127.0.0.1 # Dirección interna aprovisionada
REDIS_PASSWORD=...
REDIS_PORT=6379
```

- **Compatibilidad Transparente:** Dado que Valkey mantiene compatibilidad total con el protocolo de Redis, el framework Laravel procesa estas variables de inmediato utilizando sus drivers nativos de Redis en `config/database.php`. No requieres alterar una sola línea de código en tu software para migrar de Redis a Valkey.

### From your local machine
**Aislamiento de Red:** Por defecto, las instancias de Valkey son estrictamente privadas e inaccesibles desde el internet público para proteger tus datos en memoria.

#### Valkey public endpoints
Si necesitas depurar el contenido de la caché localmente, ejecutar comandos CLI (`valkey-cli` o `redis-cli`) o conectar herramientas de escritorio (como RedisInsight o TablePlus), puedes habilitar el interruptor **"Public endpoint"** en los ajustes del recurso. Esto expondrá un punto de enlace seguro con credenciales exclusivas de acceso exterior.

#### Valkey SSL connections
Al igual que con las bases de datos de la plataforma, **Valkey exige obligatoriamente conexiones cifradas SSL/TLS** a través de su endpoint público. Cualquier intento de conexión local que no tenga habilitado el parámetro de seguridad SSL de forma explícita será rechazado automáticamente por la capa perimetral de Laravel Cloud.
