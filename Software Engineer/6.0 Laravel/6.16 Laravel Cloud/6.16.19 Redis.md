### Introduction
Laravel Cloud permite aprovisionar instancias de Redis totalmente gestionadas para ser utilizadas como almacenamiento en caché de alto rendimiento o base de datos clave-valor en memoria.

### Creating a Redis cache
- **Aprovisionamiento:** Se añade visualmente desde el lienzo de infraestructura (`Add resource > Cache > Redis`).
- **Parámetros:** Al crearlo se debe especificar un nombre para el recurso, la región de despliegue (que debe coincidir con la de la aplicación para evitar latencias) y el tamaño de la instancia basándose en la asignación de memoria RAM requerida.

### From your application
- **Configuración Automática:** Al conectar el recurso de Redis a un entorno en el lienzo gráfico, Laravel Cloud inyecta automáticamente las variables de entorno estándar en los contenedores:
    - `REDIS_HOST`
    - `REDIS_PASSWORD`
    - `REDIS_PORT` (6379 por defecto)
- **Conectividad:** La aplicación Laravel lee estas variables de forma nativa a través de su archivo `config/database.php` sin necesidad de modificar el código fuente.

### From your local machine
- **Seguridad:** Por defecto, las instancias de Redis se aprovisionan dentro de la red privada del entorno y no son accesibles desde el internet público.
- **Endpoint Público:** Si se requiere conectar herramientas de administración locales (como TablePlus o RedisInsight), se debe activar manualmente la opción **"Public endpoint"** en los ajustes del recurso para generar credenciales de acceso externo.
- **Requisito TLS/SSL:** Para proteger los datos en tránsito, **todas las conexiones a través del endpoint público exigen de forma obligatoria cifrado TLS/SSL**. Los intentos de conexión local en texto plano son rechazados automáticamente.
