### Introduction
- Permite crear *`buckets`* de almacenamiento de objetos compatibles con `S3` y asociarlos directamente a los entornos de la aplicación desde el panel de control.
- **Infraestructura Subyacente**: El servicio se ofrece en asociación con `Cloudflare R2`.
- **Compatibilidad**: En Laravel se interactúa de forma nativa mediante la `Facade Storage`, mientras que las aplicaciones `Symfony` pueden usar cualquier integración compatible con el protocolo `S3`.

### Prerequisites
**Dependencia Obligatoria**: Antes de utilizar el servicio, el proyecto debe incluir de forma explícita la librería `league/flysystem-aws-s3-v3` en las dependencias de `Composer`.

```bash
composer require league/flysystem-aws-s3-v3 "^3.0" --with-all-dependencies
```

### Creating buckets
Se adjunta desde el lienzo de infraestructura seleccionando `Add bucket > Laravel Object Storage`. Se requiere volver a desplegar el entorno (*`re-deploy`*) para que los cambios surtan efectos.

### Bucket disk names
- Al crear el `bucket` se define un nombre de disco (`disk name`), el cual mapea la llamada directa en el código: `Storage::disk('r2')->get('file.jpg')`.
- Se si marca como el disco predeterminado (*`default`*), se omite el parámetro en la `facade: Storage::get('file.jpg')`.

### Bucket file visibility
Se define la visibilidad de archivos a nivel de `bucket` entero (no soporta mezclar visibilidades dentro de un mismo `bucket`):
- **`Private buckets`**: Los archivos no son accesibles públicamente desde internet. Permite generar `URLs` temporales de acceso mediante `Storage::temporaryUrl(...)`. Ideal para documentos sensibles de usuarios.
- **`Public buckets`**: Todos los archivos son públicos y accesibles por internet mediante una `URL` provista por Laravel Cloud. Ideal para `assets` como avatares.
- **Restricción Técnica (Análisis Crítico)**: `Cloudflare R2` gestiona la visibilidad a nivel de `bucket` y no soporta cabeceras ACL individuales por objeto. Configurar `visibility: 'public'` en la configuración `Flysystem` hará que `R2` rechace la petición con un error `NotImplemented`.

### CORS policy
Laravel Cloud administra automáticamente las políticas `CORS (Cross-Origin Resources Sharing)` para permitir peticiones entre orígenes desde navegadores.

### Automatic domain inclusion
Todos los dominios predeterminados (`*.laravel.cloud`) y dominios personalizados asignados al entorno se agregan de forma automática a la política `CORS` del `bucket` sin configuración manual.

### Custom allowed origins
Permite añadir orígenes externos permitidos (por ejemplo, entornos locales tipo `http://example.test` o dominios de terceros) desde `Org > Resources > Object storage > Edit settings` (únicamente visible para `buckets` públicos).

### Technical details
La política `CORS` genérica aplicada permite los métodos `GET`, `POST`, `PUT`, `DELETE` y `HEAD`, autoriza todos los encabezados (`*`), y restringe orígenes a los dominios del entorno más los orígenes personalizados explícitos.

### Connecting to buckets
#### From your application
- Al enlazar un `bucket` a un entorno, Laravel Cloud inyecta automáticamente las variables de entorno necesarias (incluyendo `FILESYSTEM_DISK` y las credenciales `S3`).
- Detalle con `AWS_URL`: Para `buckets` públicos, la variable `AWS_URL` (la `URL` base pública) se muestra en los ajustes del `bucket` pero no se inyecta automáticamente. Si tu código la requiere, debes copiarla manualmente a las variables de entorno del proyecto.

#### From your local machine 
- Se pueden obtener las credenciales (`AWS_ENDPOINT`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_BUCKET`) desde `Resources > Object storage > View credentials`.
- Permite la conexión directa desde clientes de escritorio compatibles con `S3`/`Cloudflare R2` como `Cyberduck`.

#### Editing buckets
Se gestiona la configuración y orígenes permitidos desde `Org > Resources > Object storage > Edit settings`.

#### Deleting buckets
Operación destructiva disponible en `Org > Resources > Object storage > Delete bucket`. Purga el recurso y elimina el almacenamiento asociado.
