### Introduction
- La página `Usage` ofrece una vista consolidada a nivel de organización para monitorear el gasto actual, el uso de recursos y cómo se distribuyen los costos entre las distintas aplicaciones dentro del periodo de facturación seleccionado.
- Los datos de consumo se actualizan cada hora. El selector del encabezado permite navegar entre el periodo de facturación en curso o meses anteriores.

### Summary
Muestra un resumen del gasto acumulado de la organización. Dependiendo del plan, la consola despliega las siguientes tarjetas de control financiero:
- **`Spending limit`**: Muestra el umbral presupuestario configurado por el usuario y cuánto margen disponible queda antes de alcanzarlo.
- **`Bandwidth`**: Indica cuánto del ancho de banda acumulado se ha consumido.
- **`Credits`**: Muestra el saldo de créditos a favor que se aplicarán automáticamente en las próximas facturas.

### Bandwidth allowances
Las franquicias o asignaciones de ancho de banda se acumulan dinámicamente a medida que tus clústeres de cómputo se ejecutan y se comparten globalmente entre todas las aplicaciones de la organización. La sección detalla cuánta de esa cuota devengada ha sido consumida.

### Resources
Esta sección desglosa el uso actual y el costo estimado dividiéndose en cuatro pestañas de recurso:

#### Databases
Muestra el identificador de cada base de datos, su uso de almacenamiento físico en disco, el consumo de cómputo, el espacio ocupado por sus respaldos (*`backups`*) y el costo total derivado.

#### Caches
Desglosa cada instancia de caché especificando su tipo (`Redis/Valkey`), detalles de almacenamiento, uso de cómputo expresado en horas y su costo total.

#### Buckets
- Desglosa el uso de almacenamiento de objetos, separando el costo de almacenamiento en GB del volumen de peticiones `Clase A` y `Clase B`.
- Esto permite diagnosticar de forma inmediata si el costo de un `bucket` proviene de albergar archivos pesados o de un exceso de operaciones de lectura/escritura en el software.

#### Websockets
Monitorea las conexiones concurrentes máximas alcanzadas, el tiempo de uso del clúster en horas y el costo total de cada clúster de `WebSocket` (`Laravel Reverb`).

### Applications
Permite auditar el gasto de cómputo descendiendo de forma jerárquica por Aplicación y Entorno. Muestra explícitamente el identificador del entorno para poder conciliarlo directamente con los conceptos de las facturas. Agrupa el consumo de cómputo en tres tipos de clústeres:
- `App Clusters` (Tráfico web HTTP)
- `Worker Clusters` (Procesos de cola tradicionales)
- `Managed Queues` (Colas gestionadas elásticas)
Dentro de cada tipo, detallada el uso, la actividad de réplicas, las horas de CPU consumidas y el costo.
- Si un clúster escala horizontalmente usando múltiples réplicas, pasar el cursor sobre el `tooltip` de réplicas desglosa el consumo individual por instancia para entender el comportamiento del auto-escalado. Al fondo calcula el total por entorno, por aplicación y el consolidado global de software.

### Add-ons
Muestra de forma independiente cualquier servicio facturable adicional o costos de infraestructura especiales contratados (como soporte premium, `Private Cloud`, o complementos de red) listándolos como elementos de línea separados con su respectivo total.
