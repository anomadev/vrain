## Definición
`Laravel Herd` es un entorno de desarrollo de PHP nativo y ultra rápido para `macOS` y `Windows`. A diferencia de `Docker` o máquinas virtuales, `Herd` instala binarios de `PHP`, `Nginx` y `dnsmasq` directamente en tu sistema operativo, permitiéndote servir sitios `.test` de forma instantánea y sin configuración manual.

## Conceptos Clave
- **`Zero-Config Environment`**: No necesitas configurar servidores web ni editar archivos `hosts`; `Herd` lo hace por ti al detectar tus carpetas.
- **`Parked Directories`**: Carpetas específicas (como `~Herd` o `~/Sites`) donde cada subdirectorio se convierte automáticamente en un sitio web accesible vía `http://folder-name.test`.
- **`Isolation & Speed`**: Al ejecutarse de forma nativa (sin capas de virtualización como `Docker`), el rendimiento es significativamente superior, especialmente en la lectura/escritura de archivos.
- **`PHP Version Manager`**: Permite cambiar la versión de PHP por sitio o de forma global con un solo clic.
- **`Laravel Installer`**: La herramienta de línea de comandos (`CLI`) definitiva para crear aplicaciones con `Starter Kits` preconfigurados.

## Ejemplos Prácticos
##### Creating a new project via CLI
```bash
# Install the Laravel Installer globally
composer global require laravel/installer

# Create a new application with interactive prompts (database, testing, starter kits)
laravel new my-awesome-project

# Or create a project with specific preferences
laravel new my-app --git --pest --stack=livewire --dark
```
##### Useful Artisan commands for post-installation
```bash
# Run the local development server (if not using Herd's automatic serving)
php artisan serve

# Run database migrations
php artisan migrate

# Clear all application cache
php artisan optimize:clear
```

## Analogía
Imagina que `Docker` es como llevar un laboratorio científico completo dentro de un contenedor de carga solo para mezclar dos sustancias; funciona en cualquier lugar, pero es pesado y lento de mover. `Laravel Herd` es como tener un banco de trabajo profesional instalado permanentemente en tu garaje: entras, dejas tus herramientas y todo está listo para funcionar al instante a máxima velocidad.

## Buenas Prácticas
- **`Use the Laravel Installer`**: Siempre prefiere `laravel new` sobre `composer create-project`, ya que el instalador está optimizado para configurar `Git`, bases de datos y kits de inicio de forma interactiva.
- **`Static Sites with Herd`**: No solo sirve para `Laravel`; puedes poner cualquier proyecto de PHP plano o HTML estático en un `Parked Directory` y `Herd` lo servirá automáticamente.
- **`DBngin for Databases`**: `Herd` (versión gratuita) no incluye base de datos. La recomendación oficial es usar `DBngin` para gestionar instancias de `MySQL`, `PostgreSQL` o `Redis` de forma ligera y visual.

## Escenarios y Aplicaciones
- **`Local Development`**: Es el estándar actual para desarrollar en `macOS` y `Windows` debido a su nula latencia.
- **`Rapid Prototyping`**: Cuando necesitas levantar 5 proyectos diferentes en una tarde para probar ideas.
- **`Legacy Projects`**: Ideal cuando necesitas alternar rápidamente entre PHP 7.4 y PHP 8.4 sin lidiar con configuraciones complejas de servidores.

## Notas Personales 
Si ya tienes otros servicios usando el puerto 80 (como `Apache` o `Docker`), `Herd` podría tener conflictos. Asegúrate de detener otros servidores web antes de iniciarlo.
