## Definición
Es el proceso de **Bootstrapping** de una aplicación mediante el cual se genera la estructura de directorios, se gestionan las dependencias iniciales vía `Composer` y se configura el entorno base. En las versiones modernas, este proceso es altamente interactivo y permite elegir el `stack` tecnológico (`testing`, `database`, `frontend`) antes de escribir la primera línea de código.

## Conceptos Clave
- **`Laravel Installer`**: Herramienta global de `CLI` que ofrece la mejor experiencia (`DX`) para crear proyectos con opciones interactivas.
- **`Composer`**: Es gestor de dependencias de PHP que descarga el core del framework y paquetes de terceros.
- **`Starter Kits`**: Plantillas oficiales (`Breeze`, `Jetstream`) que incluyen autenticación, perfiles y diseño base.
- **`Environment File (.env)`**: Archivo donde residen las variables de configuración sensibles (`DB credentials`, `App Keys`).
- **`SQLite by Default`**: La nueva convención de Laravel para desarrollo local, eliminando la necesidad de configurar un servidor de base de datos externos al inicio.

## Ejemplos Prácticos
##### Using the Laravel Installer (Recommended)
```bash
# Install the installer globally if you haven't
composer global require laravel/installer

# Create a new project with an interactive wizard
# It will ask for Starter Kit, Testing Framework (Pest/PHPUnit), and DB
laravel new my-awesome-app
```
##### Manual Installation via Composer
```bash
# Standard command if you don't want the installer
composer create-project laravel/laravel my-app
```
##### Basic Verification
```bash
# Navigate to project
cd my-app

# Run migrations (creates the sqlite file automatically if chosen)
php artisan migrate

# Start the local server
php artisan serve
```

## Analogía
Crear un proyecto con el `Laravel Installer` es como usar un **configurador de coches de lujo online**. En lugar de recibir un coche genérico y tener que cambiarle tú mismo los asientos o el motor (instalación manual), el instalador te pregunta: "Quieres motor deportivo (`Pest`)? Asientos de cuero (`Breeze`)? Tracción total (`SQLite`)?". Cuando terminas, el coche sale de la fábrica listo para correr según tus especificaciones exactas.

## Buenas Prácticas
- **`Prefer Pest for Testing`**: Es el estándar moderno en el ecosistema `Laravel`. Su sintaxis es más legible y expresiva que `PHPUnit`.
- **`Initialize Git Immediately`**: El instalador ya ofrece hacerlos por ti. No trabajes sin control de versiones desde el segundo uno.
- **`Use the --git and --pest flags`**: Si quieres automatizar scripts de creación sin pasar por el `wizard` interactivo.
- **`Keep .env.example updated`**: Cada vez que añadas una variable nueva a tu `.env`, refléjala en el `.example` para que tu equipo (o tú en el futuro) sepa qué configurar.

## Escenarios y Aplicaciones
- Cuando inicias una idea desde cero y quieres la última versión estable.
- `**Proof of Concepts (PoC)`**: Usando `SQLite` y `Breeze`, puedes tener un prototipo funcional con autenticación en menos de 5 minutos.
- **`Learning/Tutorials`**: Instalar una versión limpia para probar un paquete específico o un patrón de diseño.

## Notas Personales
- Si el comando `laravel` no funciona tras instalarlo, es probable que tu `$PATH` no incluya el directorio de binarios de `Composer`.
- `Laravel 11+` ha simplificado enormemente la estructura inicial (menos archivos en `config/` y `app/`), lo que lo hace mucho más amigable para aplicar `Clean Architecture` sin tanto ruido.
