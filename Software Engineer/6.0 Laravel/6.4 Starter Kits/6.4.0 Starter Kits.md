## Definición
Son puntos de partida oficiales y de la comunidad que proporciona el "andamiaje" (`scaffolding`) necesario para una aplicación moderna. Incluye funcionalidades pre-construidas como **Autenticación** (registro, login, reset de password, verificación de email), gestión de perfiles y, en versiones avanzadas, soporte para equipos y `API tokens`.

## Conceptos Clave
- **`Laravel Breeze`**: El punto de partida minimalista. Es código simple, sin complicaciones, ideal para aprender y para proyectos que requieren personalización total.
- **`Laravel Jetstream`**: El "peso pesado". Diseñado para aplicaciones `SaaS` complejas; incluye gestión de equipos, autenticación de dos factores (`2FA`), sesiones de navegador y soporte para `API` (vía `Sanctum`).
- **`Stack Agnostic`**: Los kits oficiales permiten elegir entre `Blade`, `Livewire` o `Inertia` (`Vue`/`React`).
- **`SaaS Kits (Community)`**: Soluciones como `SaaSykits` que añaden capas de facturación (`Stripe`/`LemonSqueezy`), paneles de administración y suscripciones listas para usar.

## Ejemplos Prácticos
##### Instalación de `Breeze` (Minimalista)
```bash
# Ideal para Adaptaditto o WildSync
composer require laravel/breeze --dev

# Instalar con Livewire y soporte funcional de testing
php artisan breeze:install livewire --pest
```
##### Estructura de un Controlador de `Auth` (`Breeze`)
A diferencia de otros `frameworks` donde la lógica está oculta en un paquete, en `Breeze` el código es tuyo:
```php
// app/Http/Controllers/Auth/RegisteredUserController.php
public function store(Request $request): RedirectResponse
{
    $request->validate([...]);

    $user = User::create([
        'name' => $request->name,
        'email' => $request->email,
        'password' => Hash::make($request->password),
    ]);

    event(new Registered($user));
    Auth::login($user);

    return redirect(route('dashboard'));
}
```

## Analogía
Imagina que quieres abrir un restaurante:
- `Breeze` es como alquilar un local vacío con una cocina básica instalada. Tienes lo necesario para empezar a cocinar (`Auth`), pero tú decides la decoración y el menú (Arquitectura).
- `Jetstream` es una franquicia de lujo llave en mano. Ya viene con sistema de reservas, uniformes para los empleados y protocolos de seguridad (`Teams`, `2FA`, `API`). Es potente, pero cambiar algo estructural es más difícil.

## Buenas Prácticas
- **Elige `Breeze` si buscas `Clean Architecture`**: Al ser código tan simple, es mucho más fácil mover la lógica de los controladores a `Services` o `Actions` sin pelear con abstracciones complejas.
- **Elige `Jetstream` solo si necesitas Equipos**: Muchos desarrolladores instalan `Jetstream` para "tenerlo todo" y terminan abrumados por su complejidad. Si no vas a usar `Team Management`, quédate con `Breeze`.
- **`Don't touch the Core`**: Intenta no modificar demasiado los controladores de los kits si planeas actualizarlos en el futuro, o bien, extrae tu lógica a clases independientes.

## Escenarios y Aplicaciones
- **Proyectos Personales**: `Breeze` + `Livewire`. Es el equilibrio perfecto entre velocidad y control.
- **SaaS Empresarial**: `Jetstream` + `Inertia`. Ideal cuando la seguridad (`2FA`) y la gestión de colaboradores (`Teams`) son requisitos no negociables desde el día 1.
- **SaaS con Facturación**: Kits de la comunidad como `Filament-based kits` para tener un panel administrativo potente en minutos.

## Notas Personales
- Al instalar un kit, este sobrescribirá tus rutas en `web.php` y tus controladores de `Auth`. Instálalo siempre en un proyecto limpio antes de empezar a programar tu lógica de negocio.
