## Definición
`Laravel Jetstream` es un *`application scaffolding`* diseñado específicamente para aplicaciones SaaS de alto nivel. Proporciona una base robusta que incluye autenticación avanzada, gestión de perfiles, `Two-Factor Authentication (2FA)`, sesiones de navegador, soporte para API a través de `Laravel Sanctum` y, opcionalmente, un sistema completo de `Team Management`.

## Conceptos Clave
- **`Actions`**: Clases simples que ejecutan una única tarea (ej. `CreateNewUser`). Es el núcleo de la lógica de `Jetstream` y fomenta el `Single Responsability Principle`.
- **`Stacks (Livewire/Inertia)`**: Al igual que otros kits, permite elegir entre el `TALL stack` o el uso de `Vue` con `Inertia` ([[6.4.1 Inertia]]).
- **`Team Management`**: Un sistema nativo de roles y permisos para equipos, permitiendo que los usuarios creen, pertenezcan y cambien entre diferentes equipos.
- **`API Support`**: Integración *`out-of-the-box`* con `Sanctum` para permitir que los usuarios generen tokens de `API` con permisos específicos (`Abilities`).
- **`Security Features`**: Incluye de serie la eliminación de cuentas, `2FA` con códigos QR y gestiones de sesiones activas en otros dispositivos.

## Ejemplos Prácticos
##### `Installation with Team Support`
```bash
# Install Jetstream via Composer
composer require laravel/jetstream

# Install using Livewire stack with Teams feature enabled
php artisan jetstream:install livewire --teams --pest
```
##### `The Power of Actions (Clean Architecture)`
En lugar de lógica en el controlado, `Jetstream` usa clases dedicadas. Así se ve la lógica para crear un equipo:
```php
// app/Actions/Jetstream/CreateTeam.php
namespace App\Actions\Jetstream;

use App\Models\Team;
use App\Models\User;
use Illuminate\Support\Facades\Gate;
use Illuminate\Support\Facades\Validator;
use Laravel\Jetstream\Contracts\CreatesTeams;

class CreateTeam implements CreatesTeams
{
    public function create(User $user, array $input): Team
    {
        Gate::forUser($user)->authorize('create', Team::class);

        Validator::make($input, [
            'name' => ['required', 'string', 'max:255'],
        ])->validateWithBag('createTeam');

        return $user->ownedTeams()->create([
            'name' => $input['name'],
            'personal_team' => false,
        ]);
    }
}
```

## Analogía
Imagina que `Breeze` es un departamento "estudio" minimalista donde tú pones los muebles. `Jetstream` es un edificio inteligente de oficinas corporativas. Ya viene con recepción (`Login`), seguridad biométrica (`2FA`), salas de juntas compartidas (`Teams`) y un sistema de mensajería interna (`API`). Es mucho más potente, pero también requiere que entiendas cómo funcionan los planos (`Actions`) para no perderte.

## Buenas Prácticas
- **`Use Actions for Custom Logic`**: No muevas la lógica de las `Actions` a controladores. Si necesitas cambiar cómo se registra un usuario, modifica la `Action` correspondiente. Esto mantiene tu código desacoplado.
- **`Feature Flags`**: Revisa el archivo `config/jetstream.php`. Puedes desactivar funciones que no uses (como `profilePhotos` o `api`) para mantener la aplicación ligera.
- **`Policy-Based Access`**: Aprovecha que `Jetstream` ya integra `Laravel Policies` para la gestión de equipos. No intentes crear un sistema de permisos paralelo.

## Escenarios y Aplicaciones
- **`SaaS B2B`**: Aplicaciones donde las empresas (`Teams`) compran una suscripción y gestionan a sus propios empleados.
- **`High-Security Apps`**: Proyectos que requieren `2FA` y auditoría de sesiones de forma obligatoria por normativas.
- **`Developer Tools`**: Aplicaciones que necesitan ofrecer una `API` a sus usuarios finales desde el primer día.

## Notas Personales
`Jetstream` es mucho más "opinado" que `Breeze`. Si intentas cambiar radicalmente la UI o el flujo de autenticación, podrías terminar peleando con los componentes internos de `Action` o `Fortify`.
