## Definición
`Laravel Breeze` es  la implementación mínima sencilla de todas las funciones de autenticación de Laravel. A diferencia de `Jetstream`, `Breeze` "expulsa" el código directamente a tu aplicación. Es la máxima expresión de **Simplicidad y Control**, ideal para aplicar **Clean Architecture** desde el día uno.

## Conceptos Clave
- **`Scaffolding`**: Generación automática de rutas, controladores y vistas para el sistema de autenticación. 
- **`Minimalism`**: Código diseñado para ser el punto de partida, no una librería compleja. Es fácil de auditar y personalizar. 
- **`Authentication Flow`**: Incluye `Login`, `Registration`, `Password Reset`, `Email Verification` y `Password Confirmation`.
- **`Vite Integration`**: Configuración lista para procesar `assets` modernos de `CSS` y `JS` de forma ultra rápida.
- **`Publishable Code`**: Todo el código reside en `app/Http/Controllers/Auth`, lo que permite modificar la lógica de negocio sin restricciones de paquetes externos.

## Ejemplos Prácticos
##### `Installation & Stack Selection`
```bash
# Install Breeze via Composer
composer require laravel/breeze --dev

# Run the install command to choose your stack
# Options: blade, livewire, livewire-functional, inertia, react, vue
php artisan breeze:install livewire --pest --dark
```
##### `Customizing Auth Logic`
Como el código es tuyo, puedes añadir lógica extra fácilmente, por ejemplo, en el registro:
```php
// app/Http/Controllers/Auth/RegisteredUserController.php

public function store(Request $request): RedirectResponse
{
    $request->validate([
        'name' => ['required', 'string', 'max:255'],
        'email' => ['required', 'string', 'lowercase', 'email', 'max:255', 'unique:'.User::class],
        'password' => ['required', 'confirmed', Rules\Password::defaults()],
    ]);

    $user = User::create([
        'name' => $request->name,
        'email' => $request->email,
        'password' => Hash::make($request->password),
        'role' => 'editor', // Custom logic added easily
    ]);

    event(new Registered($user));
    Auth::login($user);

    return redirect(route('dashboard', absolute: false));
}
```

## Analogía
Imagina que vas a comprar una **computadora**. **`Breeze`** es como comprar las piezas y recibir una guía de ensamblaje básica: tienes el procesador, la RAM y el disco (`Auth`, `Routes`, `Controllers`), pero la caja está abierta y puedes cambiar cualquier componente o añadir luces LED (personalización) sin romper ninguna garantía. Es una máquina limpia y potente que tú controlas.

## Buenas Prácticas
- **`Keep it Simple`**: No añadas complejidad innecesaria. Si `Breeze` es suficiente para tu `Auth`, no saltes a `Jetstream` solo por "estética".
- **`Refactor to Actions`**: Si tu lógica de registro crece mucho, extrae la creación del usuario a una clase `CreateUserAction` para mantener el controlador limpio (siguiendo `SOLID`).
- **`Version Control`**: Haz un `commit` justo antes de ejecutar `breeze: install` y otro inmediatamente después para identificar exactamente qué archivos creó o modificó el comando.

## Escenarios
- **Proyectos Personalizados**: Donde necesitas un flujo de registro muy específico que se aleja del estándar.
- **Aprendizaje de Arquitectura**: Es el mejor recurso para estudiar cómo Laravel gestiona la seguridad por dentro sin distracciones.
- **`MVPs` Rápidos**: Cuando el tiempo es oro pero sabes que el proyecto escalará y no quieres "deuda técnica" de paquetes pesados. 

## Notas Personales
- Recuerda que `Breeze` sobrescribe archivos. Si ya tienes un `User.php` con mucha lógica o un `web.php` configurado, haz un respaldo.
- `Breeze` es ideal porque demuestra que prefieres soluciones elegantes y mantenibles sobre las "cajas negras" complejas.
