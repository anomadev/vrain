## Definición
El directorio `config/` contiene todos los archivos de configuración de tu aplicación. Cada archivo en un `array` de `PHP` ([[1.1 What is PHP]]) que devuelve valores que dictan cómo deben comportarse los diferentes servicios del `framework` (Base de Datos, Correos, Sesiones, etc.). Estos archivos actúan como un capa de abstracción sobre las variables de entorno (`.env`).

## Conceptos Clave
- **`env() Helper`**: Función utilizada dentro de los archivos de configuración para recuperar valores del archivo `.env`. Permite tener diferentes configuraciones para `Local`, `Staging` y `Producción`.
- **`config() Helper`**: Función utilizada en el código de tu aplicación (`Controllers`, `Services`) para acceder a los valores configurados. Regla de oro: Nunca usas `env()` fuera de los archivos de `config/`.
- **`Config Caching`**: El comando `php artisan config:cache` combina todos tus archivos de configuración en un solo archivo plano para que Laravel no tenga que leer múltiples archivo en cada `request`, mejorando drásticamente el rendimiento.
- **`Config Publishing`**: En `Laravel 11+`, si un archivo de configuración no existe en tu carpeta `config/`, puedes traerlo usando `php artisan config:publish`.

## Ejemplos Prácticos
##### Accediendo a valores de configuración
```php
// app/Services/PaymentService.php

// Accessing a value from config/services.php
$apiKey = config('services.stripe.key');

// Providing a default value if the config is missing
$timezone = config('app.timezone', 'UTC');
```
##### Estructura de un archivo de configuración (`config/services.php`)
```php
return [
    'stripe' => [
        'model' => App\Models\User::class,
        'key' => env('STRIPE_KEY'),
        'secret' => env('STRIPE_SECRET'),
    ],
];
```

## Analogía
Imagina que tu aplicación es un Avión de Combate: 
- El archivo `.env` es el Combustible y las Municiones que cargan en el hangar (datos sensibles y variables según la misión).
- El directorio `config/` es el Panel de Instrumentos en la cabina. El piloto no interactúa directamente con los tanques de combustibles, sino con los diales del panel que le dicen cuánta energía hay y cómo debe responder el motor.

## Buenas Prácticas
- **`Use Defaults`**: Siempre proporciona un segundo argumento al `helper config('key', 'default')` para evitar que tu aplicación falle si falta una variable de entorno.
- **`Never use env() in Logic`**: Si usas `env()` directamente en un `Controller`, el valor será `null` una vez que ejecutes `config:cache` en producción. Usa siempre `config()`.
- **`Custom Config Files`**: Si tu proyecto tienen configuraciones específicas (*ej. parámetros de sincronización offline*), crea tu propio archivo `config/{custom-file}.php` en lugar de ensuciar el `app.php`.

## Escenarios y Aplicaciones
- **`Multi-tenancy`**: Cambiar dinámicamente valores de configuración en tiempo de ejecución usando `config(['database.connections.tenant.database' => $dbName])`
- **`Third-party Services`**: Centralizar las credenciales de `APIs` para que sean fáciles de rotar sin tocar el código fuente.

## Notas Personales
Al crear un nuevo archivo de configuración, recuerda que debe devolver un `array`. Si olvidad el `return`, `config()` siempre devolverá un `null`.
