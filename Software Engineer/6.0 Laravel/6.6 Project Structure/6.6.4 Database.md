## Definición
El directorio `database/` contiene todos los archivos relacionados con la estructura y la población de la base de datos. Es el centro de control para el control de versiones del esquema (`Migrations`), la generación de datos de prueba (`Factories`) y la inserción de datos iniciales o maestros (`Seeders`).

## Conceptos Clave
- `Migrations`: Archivos de control de versiones para tu base de datos. Permiten que todo el equipo trabajo sobre el mismo esquema sin compartir volcados de SQL manualmente.
- `Factories`: "Planos" para generar registros de modelos de forma masiva y aleatoria. Son esenciales para el `Testing` y el desarrollo de interfaces.
- `Seeders`: Clases encargadas de insertar datos específicos de las tablas. Se usan para configurar datos maestros (*ej. roles, categorías*) o llenar la `app` de contenido `dummy`.
- `database.sqlite`: El archivo de base de datos por defecto en instalaciones nuevas. Vive físicamente aquí, eliminando la dependencia de servidores externos para empezar a prototipar.

## Ejemplos Prácticos
##### Una Migración Atómica
```php
//database/migrations/2026_03_09_000000_create_projects_table.php
public function up(): void
{
	Schema::create('projects', function (Blueprint $table) {
		$table->id(); $table->string('name');
		$table->text('description')->nullable();
		$table->foreignId('user_id')->constrained()->onDelete('cascade');
		$table->timestamps();
	});
}
```
##### Factory para `testing`
```php
// database/factories/ProjectFactory.php
public function definition(): array
{
	return [
		'name' => $this->faker->$sentence(3);
		'description' => $this->faker->paragraph(),
		'user_id' => User::factory(), // Recursive factory!
	];
}
```
#####  `Seeder` Principal
```php
// database/seeders/DatabaseSeeder.php
public function run(): void
{
	User::factory(10)->create()->each(function ($user) {
		Project::factory(3)->create(['user_id' => $user->id]);
	});
}
```

## Analogía
Imagina que estás construyendo una **Urbanización**.
- Las `Migrations` son los planos de ingeniería que dicen dónde van las tuberías y las paredes.
- Los `Seeders` son el mobiliario básico que pones en la casa piloto para que se vea funcional.
- Las `Factories` son máquinas de clonación que pueden generar miles de inquilinos con nombres y apariencias diferentes (`Faker`) para probar si los ascensores y servicios aguantan la carga.

## Buenas Prácticas
- **`Atomic Migrations`**: Nunca modifiques una migración que ya ha sido ejecutada en producción. En su lugar, crea una nueva migración para alterar la tabla.
- **`Use Factories in Seeders`**: No Escribas `User::create([...])` manualmente en tus `seeders`. Usa las `Factories` para mantener la coherencia y facilitar cambios futuros.
- **`Don't ignore down()`**: Siempre completa el método `down()` en tus migraciones para permitir retrocesos (`migrate:rollback`) seguros durante el desarrollo.

## Escenarios y Aplicaciones
- **`Testing`**: Usar `RefreshDatabase` en tus `test` para limpiar y volver a migrar la base de datos en cada ejecución asegurando un entorno puro.
- **`Onboarding de Devs`**: Un nuevo desarrollador solo tiene que ejecutar `php artisan migrate --seed` para tener la aplicación lista para trabajar con datos reales.
- **`Local Persistence`**: Para proyectos, usar `SQLite` dentro de este directorio permite una portabilidad total sin configurar `MySQL/Postgres`.

## Notas Personales
Cuidado con `php artisan migrate:fresh`. Borra todas las tablas y vuelve a ejecutarlas. Nunca lo uses en producción o perderás todos los datos reales.
