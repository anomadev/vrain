## Definición
El directorio `routes/` contiene las definiciones de todas las rutas de tu aplicación. Estas rutas actúan como el `mapper` que asocia la `URL` y un verbo `HTTP` (`GET`, `POST`, etc.) con una pieza específica de lógica (un `Controller`, o un `Closure`). Es la primera capa de lógica que el `Request` encuentra después de pasar por el proceso de *`bootstraping`*.

## Conceptos Clave
- **`web.php`**: El hogar de las rutas para la interfaz web. Estas rutas pasan por el grupo de `middleware web`, proporcionando estado de sesión, protección `CSRF` y `cookies`.
- **`api.php`**: (Opcional en L11+) Diseñado para rutas `stateless`. Por defecto, una el `middleware` de `Sactum` para autenticación basada en `tokens`.
- **`console.php`**: Aquí se definen los comandos de `Artisan` basados en `Closures`. Es donde registras tareas que se ejecutan desde la terminal.
- **`Route Model Binding`**: Técnica que permite a Laravel inyectar automáticamente una instancia de un modelo de `Eloquent` basándose en el `ID` presente en la `URL`.
- **`Named Routes`**: Asignar un nombre único a una ruta para poder referenciarla en toda la aplicación sin `hardcodear` la `URL`.

## Ejemplos Prácticos
##### Definición de rutas y `Named Routes`
```php
// routes/web.php
use App\Http\Controllers\ProjectController;
use Illuminate\Support\Facades\Route;

// Standard route with a name
Route::get('/projects', [ProjectController::class, 'index'])
	->name('projects.index');

// Route with parameters and Route Model Binding
Route::get('/projects/{project}', [ProjectController::class, 'show'])
	->name('projects.show');
```
##### `Routes Groups` y `Middleware`
```php
// routes/web.php
Route::middleware(['auth', 'verified'])->group(function () {
    Route::prefix('admin')->name('admin.')->group(function () {
        Route::get('/dashboard', [AdminController::class, 'index'])
	        ->name('dashboard');
        Route::resource('users', UserController::class);
    });
});
```
##### `Enabled API routes in Laravel 11+`
Si no ves el archivo `php.api`, es porque Laravel moderno es minimalista. Debes invocarlo:
```bash
php artisan install:api
```

## Analogía
Imagina que tu aplicación es un **Gran Hotel**. El directorio `routes/` es el **Panel de la Recepción**.
- Cuando un huésped (`Request`) llega y dice "Quiero ir a la piscina" (`URL`), el recepcionista mira el panel y dice: "Piso 2, puerta A" (`Controller`).
- Si el huésped intenta ir a la "Bóveda" (Ruta protegida), el recepcionista primero verifica si tiene la llave (`Middleware`).
- Las `Named Routes` son como llamar a la habitación por su nombre ("Suite Presidencial") en lugar de decir siempre "La habitación 402 del cuarto piso"; si el hotel cambia la numeración, el nombre sigue apuntando al lugar correcto.

## Buenas Prácticas
- **`Always use Named Routes`**: Nunca uses `URLs` fijas en tus vistas o controladores. Usa `route('projects.index')`. Esto te permite cambiar la `URL` en un solo archivo sin romper nada.
- **`Avoid Closures in Routes`**: Para aplicaciones reales, mueve la lógica a `Controllers`. Las rutas con `Closures` no pueden ser cacheadas (`routes:cache`), lo que penaliza el rendimiento.
- **`Use Route Model Binding`**: Limpia tus controladores dejando que Laravel busque el modelo por ti. Si el registro no existe, Laravel devolverá un 404 automáticamente.

## Escenarios y Aplicaciones
- **`SEO & Marketing`**: Usar `web.php` para definir `URLs`amigables (`slugs`) que mejoran el posicionamiento de tu marca personal.
- **`Microservices / Mobile Apps`**: Usar `api.php` para exponer datos en formato `JSON` consumibles por una app en `Flutter` o un `frontend` desacoplado.
- **`Task Automation`**: Usar `console.php` para crear comandos rápidos que limpien `logs` o sincronices datos de clima.

## Notas Personales
- Si añades una ruta y Laravel te da un 404, asegúrate de que no haya otra ruta más genérica arriba que la esté "atrapando". El orden de las rutas importa.
- En Laravel 11, la ausencia de `api.php` por defecto refuerza la idea de que Laravel es, ante todo, un framework pragmático. Si no vas a construir una `API`, no te ensucia el proyecto con archivos que no usas.
