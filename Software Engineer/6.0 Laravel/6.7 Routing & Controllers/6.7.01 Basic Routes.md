## Definición 
El `Routing` es el mecanismo que acepta una petición `HTTP` (`Request`), analiza la `URI` y el `HTTP Verb` (`GET`, `POST`, etc.), y la dirige al destino correcto, ya sea un `Closure` o, preferiblemente, un método dentro de un `Controller`. Es la primera capa de lógica después del `middleware`.

## Conceptos Clave
- **`HTTP Verbs`**: Los métodos estándar para interactuar con recursos: `GET` (recuperar), `POST` (crear), `PUT/PATCH` (actualizar) y `DELETE` (eliminar).
- **`Route Parameters`**: Segmentos dinámicos de la `URL` (ej. `id`) que se capturan y se pasan como argumentos a la lógica de la ruta.
- **`Named Routes`**: Asignar un nombre único a una ruta para generar `URLs` o direcciones de forma pragmática, evitando el "`hardcoding`" de `strings`.
- **`Redirect Routes`**: Rutas dedicadas exclusivamente a redirigir de una `URI` a otra.
- **`View Routes`**: Rutas directas para renderizar una vista de `Blade` sin necesidad de pasar por un controlador si no hay lógica compleja.

## Ejemplos Prácticos
##### `Basic Routing with Controllers (Industry Standard)`
```php
// routes/web.php
use App\Http\Controllers\ProjectController;
use Illuminate\Support\Facades\Route;

// Simple GET route
Route::get('/projects', [ProjectController::class, 'index'])
	->name('projects.index');

// Route with a required parameter
Route::get('/projects/{id}', [ProjectController::class, 'show'])
	->name('projects.show');

// Route with an optional parameter and default value
Route::get('/user/{name?}', function (string $name = 'Guest') {
    return "Hello, {$name}";
})->name('user.profile');
```
##### `Generating URLs using Named Routes`
```php
// In a Controller or Blade view
$url = route('projects.show', ['id' => 1]); // Output: https://app.test/projects/1

return redirect()->route('projects.index');
```

## Analogía
Imagina que tu aplicación es un **Edificio de Oficinas**. El sistema de `Routing` es el Directorio en la recepción.
- La `URI` (`/projects`) es el número de oficina.
- El `HTTP Verb` es la acción del visitante: si viene a "entregar un paquete" (`POST`), a "pedir información" (`GET`) o a "hacer mantenimiento" (`PUT`).
- El `Controller` es el empleado especializado que atiende esa oficina específica.
- El `Named Route` es como decir "Ve a la oficina de Contabilidad" en lugar de decir "Ve a la puerta 302 del piso 3"; si Contabilidad se muda, el nombre sigue siendo el mismo pero la ubicación cambia en el registro central.

## Buenas Prácticas
- **`Avoid Closures in Production`**: Nunca dejes lógica pesada en `web.php` usando funciones anónimas. Mueve todo a `Controllers` para habilitar el `Route Caching` y mejorar el rendimiento.
- **`Use Named Route Always`**: Esto previene que tu aplicación se rompa si decides cambiar una `URL` de `/users` a `/accounts` en el futuro.
- **`KISS (Keep It Simple, Stupid)`**: Las rutas deben ser legibles. Si una `URI` es demasiado compleja, considera si el recurso está bien definido arquitectónicamente.

## Escenarios y Aplicaciones
- **`SEO-Friendly URLs`**: Crear estructuras como `/blog/{slug}` para mejorar el posicionamiento en buscadores.
- **`API Development`**: Definir puntos de acceso claros para aplicaciones móviles o `frontends` desacoplados en `api.php`.
- **`Marketing / Shorlinks`**: Implementar redirecciones rápidas para campañas temporales usando `Route::redirect()`.

## Notas Personales
- El orden de las rutas importa. Si defines `Route::get('/projects/{id}')` antes de `Route::get('/projects/create')`, Laravel intentará tratar `create` como si fuera un `ID`. Pon siempre la rutas estáticas antes que las dinámicas.
- El uso de `Route Model Binding` hará que las rutas sean extremadamente potentes, inyectando modelos directamente desde la `URI`.
