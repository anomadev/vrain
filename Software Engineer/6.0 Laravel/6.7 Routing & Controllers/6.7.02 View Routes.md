## Definición 
Las `View Routes` son un método abreviado proporcionado por la fachada `Route` para definir rutas que solo necesitan renderizar una vista de `Blade`. Eliminan la necesidad de definir un `closure` o un método en un controlador, permitiendo pasar datos simples directamente a la plantilla.

## Conceptos Clave
- **`Route::view()`**: El método estático que recibe tres parámetros: la `URI`, el nombre de la vista y un `array` de datos (opcional).
- **`Static Content`**: Ideal para páginas que no dependen de lógica compleja o consultas pesadas a la base de datos.
- **`Data Injection`**: Permite inyectar variables simples (como títulos o meta-datos) sin salir del archivo de rutas. 
- **`HTTP Method`**: Por defecto, estas rutas solo responden a peticiones de tipo `GET`.

## Ejemplos Prácticos
##### Definición básica vs Paso de datos
```php
// routes/web.php
use Illuminate\Support\Facades\Route;

// Simple view rendering
Route::view('/about', 'pages.about')->name('about');

// Rendering with static data passing
Route::view('/terms', 'pages.terms', [
    'page_title' => 'Terms and Conditions',
    'last_updated' => '2026-03-15'
])->name('terms');
```
##### Comparativa: `Route::get` vs `Route::view`
```php
// Verbose way (Avoid for simple views)
Route::get('/privacy', function () {
    return view('pages.privacy');
});

// Clean way (Senior approach)
Route::view('/privacy', 'pages.privacy');
```

## Analogía
Imagina que vas a una Máquina Expendedora (`View Route`). Tú presionas un botón (`URI`) y la máquina te entrega directamente el producto (`View`) sin que nadie tenga que intervenir. En cambio, ir a un Restaurante (`Controller`) implica que un mesero tome tu pedido, se lo lleve al chef, este lo prepare y luego te lo traigan. Si lo que quieres es solo una lata de refresco, la máquina expendedora es mucho más eficiente.

## Buenas Prácticas
- **`Use for Static Pages`**: Úsalas para "`About Us`", "`Contact`" (si el formulario se maneja por `API` o `Livewire`), "`FAQ`" y "`Privacy Policy`".
- **`Don't fetch data here`**: Nunca intentes llamar a modelos de `Eloquent` o realizar cálculos complejos dentro del `array` de datos de una `Route::view`. Si necesitas datos de la `DB`, usa un `Controller`.
- **`Keep it readable`**: Si el `array` de datos que pasas es muy grande, es una señal de que la ruta está dejando de ser "simple" y deberías considerar moverla a un controlador.

## Escenarios y Aplicaciones
- **`Landing Pages`**: Perfectas para la página de inicio, si el contenido es mayormente informativo y estático.
- **`Maintenance/Legal`**: Paginas de "Términos de Servicio" o "Política de Cookies" que rara vez cambian su estructura lógica.
- **`Error Pages Customization`**: Puede previsualizar tus páginas de error personalizadas rápidamente con este método.

## Notas Personales
- A veces olvidamos que al usar `Route::view`, los datos que pasamos son estáticos. Si necesitas que el `page_title` sea dinámico basado en un registro de la base de datos, esta función no es la adecuada.
- Usa estas rutas para las páginas de marketing. Mantendrás tu `web.php` corto y fácil de leer para otros ingenieros que revisen tu código.
