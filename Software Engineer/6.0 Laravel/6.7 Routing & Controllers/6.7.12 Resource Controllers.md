## Definición
Un `Resource Controller` es un controlador que agrupa todas las acciones relacionadas con un "recurso" específico (como artículos, usuarios, productos) siguiendo las convenciones de una `API RESTful`. Al declarar una ruta de recurso, Laravel registra automáticamente siete rutas para manejar las operaciones comunes del recurso.
### Mapeo de Acciones (`CRUD`)
| Verbo HTTP | URI | Acción (Método) | Propósito
| :--- | :--- | :--- | :--- |
| `GET` | `/photos` | `index` | Listar todos los registros |
| `GET` | `/photos/create` | `create` | Mostrar el formulario de creación |
| `POST` | `/photos` | `store` | Almacenar un nuevo registro en la DB |
| `GET` | `/photos/{photo}` | `show` | Mostrar un registro especifico |
| `GET` | `/photos/{photo}/edit` | `edit` | Mostrar el formulario de edición |
| `PUT/PATCH` | `/photos/{photo}` | `update` | Actualizar un registro específico |
| `DELETE` | `/photos/{photo}` | `destroy` | Eliminar un registro la DB |

## Ejemplos Prácticos
##### Generación del Controlador
Para crear un controlador con todos los métodos `CRUD` ya definimos, se utiliza el `flag --resource`:
```bash
php artisan make:controller PostController --resource
```
Este comando genera los métodos vacíos listos para ser implementados.
##### Definición de la Ruta
En lugar de escribir siete líneas de `Route::get` o `Route::post`, se utiliza una sola:
```php
// routes/web.php
use App\Http\Controllers\PostController;
use Illuminate\Support\Facades\Route;

Route::resource('posts', PostController::class);
```
Laravel entiende automáticamente qué método llamar basándose en la `URL` y el verbo `HTTP`.
##### Implementación del método `Store` (Validación y Creación)
```php
// app/Http/Controllers/PostController.php
public function store(Request $request)
{
    // Validar los datos de entrada
    $validated = $request->validate([
        'title' => 'required|max:255',
        'body' => 'required',
    ]);

    // Crear el registro usando el modelo
    Post::create($validated);

    return redirect()->route('posts.index')
                     ->with('success', 'Post created successfully');
}
```
Es una buena práctica validar los datos antes de persistirlos.
##### `Partial Resources Routes`
Si no necesitas las siete rutas, puedes limitar cuáles registrar:
```php
Route::resource('photos', PhotoController::class)
	->only(['index', 'show']);
Route::resource('photos', PhotoController::class)
	->except(['create', 'store', 'update', 'destroy']);
```
##### `API Resources`
Si estás construyendo una `API`, el comando `php artisan make:controller API/PostController --api` excluye las rutas que devuelven `HTML` (`create` y `edit`), dejando solo los `endpoints` de datos.
##### `Nested Resources`
Útil para recursos relacionados, como comentarios de un `post`:
```php
Route::resource('posts.comments', CommentController::class);
// Genera URLs como: /posts/{post}/comments/{comment}
```

## Analogía
Imagina que un `Resource Controller` es un Mostrador de Trámites Gubernamentales. En lugar de tener una ventanilla distinta para "Pedir el formulario", otra para "Entregarlo", otra para "Consultar estado" y otra para "Cancelar", el `Resource Controller` es una Ventanilla Única. Dependiendo del "documento" que lleves (Verbo `HTTP`) y de si ya tienes un "número de folio" (`ID`), el empleado sabe exactamente qué proceso ejecutar.

## Buenas Prácticas
- **`Use Route Model Binding`**: Combina `Resource Contollers` con `Route Model Binding` para que métodos como `show`, `edit`, `update` y `destroy` reciban directamente la instancia del modelo en lugar de solo el `ID`.
- **`Naming Consistency`**: Mantén los nombres de los recursos en plural (`posts`, `users`) para seguir la convención estándar de Laravel.
- **`Controller Bloat`**: Si un `Resource Controller` empieza a tener demasiados métodos personalizados fuera de los siete estándar, considera si el recurso debe dividirse en controladores más pequeños.

## Notas Adicionales
- El uso de `--resource` en el comando `make:controller` ahorra tiempo de desarrollo y asegura que el equipo siga el mismo estándar de nombres de métodos.
- Aunque Laravel crea siete métodos por defecto, siempre puedes añadir métodos personalizados al controlador si el recurso requiere acciones adicionales, aunque se recomienda mantenerlo simple.
