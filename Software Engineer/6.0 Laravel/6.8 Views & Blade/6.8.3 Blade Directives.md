## Definición
Las directivas de `Blade` son comandos que comienzan con el símbolo `@` y son procesados por el motor de plantillas antes de que la página se envíe al navegador. El motor compila estas directivas en código `PHP` optimizado, lo que significa que no hay una penalización de rendimiento al usarlas.
### Estructuras de Control (Condicionales)
Permiten renderizar contenido solo si se cumple una condición específica.
- `@if`, `@elseif`, `@else`, `@endif`: La estructura básica de decisión.
- `@unless`, `@endunless`: El inverso de `@if`. Solo renderiza si la condición es falsa.
- `@isset`, `@endisset`: Comprueba si una variable está definida y no es nula.
- `@empty`, `@endempty`: Comprueba si una variable está vacía.
```php
@if ($greeting == 'hello')
	<p>Hi from inside the if statement</p>
@endif
```
### Bucles (Iteración)
Ideales para listar datos que vienen de la base de datos (como una lista de usuarios o productos).
- `@foreach`, `@endforeach`: La forma más común de iterar sobre una colección o `array`.
- `@forelse`, `@empty`, `@endforelse`: Una versión mejorada de `@foreach`. Si la colección está vacía, ejecuta automáticamente el bloque `@empty`. Es el estándar de oro para evitar errores de "`count`".
- `@for`, `@enfor`: El bucle tradicional basado en un contador.
- `@while`, `@endwhile`: Itera mientras se cumpla una condición.
```php
<ul>
	@foreach ($ninjas as $ninja)
		<li>{{ $ninja['name'] }} - {{ $ninja['id'] }}</li>
	@endforeach
</ul>
```
### Directivas de Autenticación
Atajos rápidos para verificar el estado del usuario sin escribir lógica manual.
- `@auth`, `@endauth`: El contenido solo es visible para usuarios autenticados.
- `@guest`, `@endguest`: El contenido solo es visible para usuarios no autenticados (invitados).
- `@can`, `@cannot`: Verifican permisos específicos del usuario (`Policies`/`Gates`).
### Gestión de `Layouts` y Secciones
Permiten construir arquitecturas de vistas modulares y reutilizables.
- `@extends`: Define de qué plantilla "muestra" o `layout` hereda la vista actual.
- `@section`, `@endsection`: Define un bloque de contenido que se inyectará en el `layout`.
- `@yield`: Se usa en el `layout` principal para marcar el lugar donde se inyectará el contenido de las secciones.
- `@include`: Incluye una "sub-vista" dentro de la vista actual (útil para componentes pequeños como botones o alertas).
### Directivas de Entorno y Utilidad
- `@production`, `@env('staging')`: Renderizan contenido solo en entornos específicos.
- `@csrf`: Genera automáticamente el campo de `token` oculto necesario para proteger formularios contra ataques `Cross-Site Request Forgery`.
- `@method('PUT')`: Simula verbos `HTTP` que los navegadores no soportan nativamente en formularios.

## Buenas Prácticas
- **Evita la lógica compleja**: Si tu directiva `@if` tiene más de 2 o 3 condiciones lógicas, es mejor mover ese cálculo al `Controller` o al `Model`.
- **Indentación**: Siempre indenta el `HTML` dentro de las directivas para mantener la legibilidad.
- **Nombres semánticos**: Usa directivas como `@auth` en lugar de `@if(Auth::check())` para que el código sea más "natural" de leer.
