Las sentencias condicionales en `Blade` son directivas que permiten controlar la estructura del `HTML` generado basándose en la lógica de la aplicación. Al ser procesadas por el motor de plantillas de Laravel, estas se compilan en código PHP eficiente, eliminando la necesidad de abrir y cerrar etiquetas de PHP manualmente (`<?php ... ?>`).

## `Blade`: Sentencias Condicionales (`if, else`)
### Estructura Básica: `@if`
La directiva `@if` funciona exactamente igual que la sentencia `if` en PHP. Se utiliza para renderizar un bloque de contenido solo si una condición se evalúa como verdadera.
```php
@if ($count === 1)
    <p>Hay un elemento disponible.</p>
@elseif ($count > 1)
    <p>Hay múltiples elementos disponibles.</p>
@else
    <p>No hay elementos disponibles.</p>
@endif
```
**Concepto Clave**
- Directivas de Cierre: Todas las sentencias condicionales en `Blade` deben cerrarse con su correspondiente directiva `@endif`.
- Legibilidad: `Blade` permite mantener el indentado del `HTML` limpio, lo que facilita la lectura de estructuras complejas.
### Directivas Especializadas (`Clean Code`)
Para evitar escribir lógica repetitiva como `if (count($items) > 0)` o `if (isset($var))`, Laravel proporciona directivas semánticas que hacen el código más expresivo.
`@unless` Es el opuesto a `@if`. El contenido se mostrará solo si la condición es falsa.
```php
@unsless (Auth::check())
	<p>Por favor, inicia sesión para continuar.</p>
@endunless
```
*Equivale a: `if (! Auth::check())`*.
`@isset` y `@empty` Estas directivas son atajos para las funciones `isset()` y `empty()` de PHP.
```php
// Se muestra si la variable existe y no es null
@isset($records)
    <p>Los registros están definidos.</p>
@endisset

// Se muestra si la variable está vacía (null, array vacío, string vacío, etc.)
@empty($records)
    <p>No hay registros para mostrar.</p>
@endempty
```
### Atajos de Autenticación y Entorno
Laravel incluye directivas específicas para los casos de uso más comunes en el desarrollo web profesional.
- `@auth`: Renderiza el contenido si el usuario está autenticado.
- `@guest`: Renderiza el contenido si el usuario es un invitado (no autenticado).
- `@production`: Renderiza el contenido solo si la aplicación se ejecuta en el entorno de producción.
- `@env('staging')`: Renderiza si el entorno coincide con el parámetro enviado.

## Mejores Prácticas
1. **Evitar Lógica Compleja**: Si tu `@if` contiene múltiples operadores lógicos (`&&`, `||`) o llamadas a métodos pesados, traslada esa lógica al `Controller` o al `Model`. La vista solo debe decidir "si se muestra", no "cómo se calcula".
2. **Uso de `@auth`**: En lugar de `@if (Auth::check())`, utiliza siempre `@auth`. Es más semántico y facilita la lectura a otros desarrolladores.
3. **No anidar en exceso**: Si tiene más de 3 niveles de `@if` anidados, considera separar ese bloque en un `Blade Component` o un `Partial` para mejorar la mantenibilidad.

## Escenarios de Uso
- **Dashboards**: Mostrar botono de "Editar" o "Eliminar" solo si el usuario tiene los permisos correcto usando `@can` (una extensión de la lógica condicional).
- **Formularios**: Mostrar mensajes de error de validación solo si existen errores para un campo específico usando `@error('field')`.
- **Vistas de Usuario**: Alternar entre los botones de "`Login/Registro`" y "`Perfil/Logout`" usando `@auth` y `@guest`.
