## Definición
En el desarrollo profesional con Laravel, la gestión de `Layouts` es la base para mantener la consistencia visual y aplicar el principio `DRY (Don't Repeat Yourself)`. Un `layout` actúa como el "armazón" de tu aplicación, conteniendo los elementos comunes (`HTML base`, `head`, `scripts`, navegación) mientras delega el contenido específico a las vistas individuales.
### Los dos enfoques de `Layouts`
Laravel permite estructurar `layouts` de dos maneras principales. Aunque ambas son válidas, la industria prefiere los componentes en versiones modernas (`10`, `11` y `12`).
#### Herencia de Plantillas (Tradicional)
Utiliza directivas para definir "huecos" que las páginas hijas rellenarán.
- `@extends`: Indica de qué `layout` hereda la vista.
- `@yield`: Define una sección que será inyectada por la vista hija.
- `@section` / `@endsection`: Envuelve el contenido que se enviará al `layout`.
#### `Layout Components` (Moderna)
Trata al `layout` como un componente de `Blade` regular. Es más intuitivo porque utiliza etiquetas similares a `HTML` y el concepto de `Slots`.
- `<x-layout>`: Envuelve la página completa.
- `{{ slot }}`: Variable donde se renderiza el contenido principal.
- `Named slots`: Para inyectar contenido en lugares específicos (ej. títulos o `scripts`).

## Ejemplo Comparativo
##### Enfoque de Componente (Recomendado)
Este método es más limpio y facilita el paso de variables (como el título de la página) mediante atributos.
```html
<!DOCTYPE html>
<html>
<head>
    <title>{{ $title ?? 'Mi Aplicación' }}</title>
    @stack('styles')
</head>
<body>
    <nav>...</nav>
    
    <main>
        {{ $slot }}
    </main>

    <footer>...</footer>
    @stack('scripts')
</body>
</html>

<x-layout title="Inicio">
    <h1>Bienvenido a la página principal</h1>
    <p>Este contenido se inyecta en la variable $slot.</p>
    
    @push('scripts')
        <script src="home.js"></script>
    @push
</x-layout>
```
### Directivas de Inyección: `Stacks` y `Pushes`
A diferencia de las secciones, los `Stacks` permiten acumular contenido. Esto es vital para cargar archivos `JavaScript` o `CSS` específicos solo en las páginas que los necesitan, sin ensuciar el `layout` global.
- `@stack('name')`: Define el lugar en el `layout` donde se renderizará el contenido acumulado.
- `@push('name')`: Añade contenido al final del `stack`.
- `@prepend('name')`: Añade contenido al principio del `stack`.

## Analogía
Imagina que estás construyendo una **Revista**.
- El **`layout`** es la plantilla de diseño fija: el logo en la esquina, el número de página abajo y la tipografía estándar.
- La **Herencia (`@extends`)** es como tener una hoja con agujeros recortados donde pegas fotos por detrás.
- El **`Layout Component`** es como un marco digital: tú le envías la foto y el texto (el contenido), y el marco sabe exactamente cómo presentarlo. Es más flexible y fácil de cambiar.

## Buenas Prácticas
1. **`Layouts Múltiples`**: No intentes meter todo en un solo `app.blade.php`. Crea `layouts` específicos para invitados (`guest.blade.php`), para el panel de administración (`admin.blade.php`) y para la aplicación principal.
2. **`Meta Tags Dinámicos`**: Usa `slots` o variables para permitir que cada página define su propio `description` y `title` para mejorar el SEO.
3. **Fragmentación**: Si tu `layout` tiene una navegación muy compleja, extrae la barra de navegación a un componente anónimo (`<x-nav />`) para mantener el archivo del `layout` legible.
4. **Uso de `@once`**: Cuando insertes `scripts` dentro de componentes que se usan varias veces en una página, envuélvelos en `@once` para evitar que el navegador cargue el mismo `script` múltiples veces.
