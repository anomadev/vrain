## Definición
`Eloquent` es el mapeador objeto-realcional (`ORM`) por defecto de Laravel. Sigue el patrón `Active Record`, lo que significa que el modelo no solo contiene los datos de una fila, sino que también sabe cómo guardarse actualizarse o eliminarse a si mismo en base de datos.

## Conceptos Clave
- **`Conventions over Configuration`**: Por defecto, un modelo `User` buscará la tabla `users`. Si sigues las convenciones (nombres en plural, claves primarias `id`, `timestamps created_at`/ `updated_at`), no tienes que configurar nada.
- **`Mass Assignment Protection`**: `Eloquent` protege tu base de datos de inyecciones de datos maliciosos mediante las propiedades `$fillable` (lista blanca) o `$guarded` (lista negra).
- **`Attribute Casting`**: Permite transformar datos automáticamente al salir o entrar a la DB (ej. convertir un `JSON` de la `DB` en un `Array` de PHP, o un `string` en un objeto `Carbon`).

## Ejemplos Prácticos
##### El nuevo estándar: `Accesors & Mutators (Laravel 11/12)`
Olvida los antiguos métodos `getFirstNameAttribute`. El enfoque moderno usa la clase `Attribute` para definir lógica de entrada y salida en un solo lugar:
```php
use Illuminate\Database\Eloquent\Casts\Attribute;

protected function fullName(): Attribute
{
    return Attribute::make(
        get: fn (mixed $value, array $attributes) => "{$attributes['first_name']} {$attributes['last_name']}",
        set: fn (string $value) => [
            'first_name' => explode(' ', $value)[0],
            'last_name' => explode(' ', $value)[1] ?? '',
        ],
    );
}
```
##### `Query Scopes` Modernos (`Laravel 12`)
Laravel 12 introduce atributos PHP para definir `scopes`, lo que hace el código mucho más limpio:
```php
use Illuminate\Database\Eloquent\Attributes\Scope;

#[Scope]
protected function featured($query)
{
    return $query->where('is_featured', true);
}

// Uso: Post::featured()->get();
```

## Analogía
Imagina que tu base de datos es un Almacén Gigante (Tablas) lleno de Cajas (Filas).
- **Sin `ORM` (`SQL` Puro)**: Tienes que escribir una carta formal al encargado del almacén cada vez que quieres una caja, describiendo exactamente dónde está y qué tiene dentro.
- **Con `Eloquent`**: Cada caja es un Robot Inteligente. Si quieres cambiar el contenido, solo le dices al robot: "*Oye, ahora guarda esto*" (`$box->save()`). El robot sabe exactamente en qué estante vive y cómo actualizar su propio inventario sin que tú tengas que mover un dedo.

## Buenas Prácticas
- **Evitar el Problema `N+1`**: Es el error más común. Si vas a mostrar 20 posts y sus autores, usa `Post::with('author')->get()`. Sin el `with()`, Laravel hará `1` consulta para los posts y 20 consultas adicionales para los autores (una por cada uno).
- **`Select Only What You Need`**: No hagas `User::all()` si solo necesitas el email. Usa `User::select('id', 'email')->get()` para ahorrar memoria RAM en el servidor.
- **Usa `exists()` vs `count()`**: Si solo quieres saber si hay registros, `User::where('active', true)->exists()` es mucho más rápido que contar todos los registros.
- **`Chucking` para Grandes Volúmenes**: Si necesitas procesar 100,000 registros, nunca uses `get()`. Usa `chuck(100, function($users) { ... })` o `lazy()` para procesar los datos en pequeños lotes y no saturar la memoria.

## Escenarios y Aplicaciones
- **`SaaS Multi-tenant`**: Usar `Global Scopes` para asegurar que cada consulta filtre automáticamente los datos por el `ID` de la empresa del usuario actual, evitando fugas de datos.
- **`API Resources`**: Transformar tus modelos de `Eloquent` en `JSON` limpios y estandarizados para que tu `frontend` (`Vue`/`React`) reciba exactamente lo que necesita.
