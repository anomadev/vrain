## Definición
`CRUD` es el acrónimo de las cuatro funciones básicas de la persistencia de datos: `Create`, `Read`, `Update` y `Delete`. `Eloquent` implementa estas funciones mediante métodos intuitivo que mapean directamente a sentencias `SQL`.

### `Create`
Existen dos formas principales de insertar datos en la base de datos:
##### Instanciación y Guardado Manual
Crea una nueva instancia del modelo, asigna valores a sus propietarios y llama al método `save()`.
```php
$user = new User;
$user->name = 'G Developer';
$user->email = 'g@example.com';
$user->save();
```
##### Creación Masiva (`Mass Assignment`)
Permite crea un registro en una sola línea pasando un `array` de datos. Requiere configurar la propiedad `$fillable` (lista blanca) o `$guarded` (lista negra) en el modelo para seguridad. 
```php
// En el Modelo: protected $fillable = ['name', 'email'];
$user = User::create([
    'name' => 'G Developer',
    'email' => 'g@example.com',
]);
```
### `Read`
`Eloquent` ofrece múltiples métodos para recuperar datos, devolviendo una instancia del modelo (para un solo resultado) o una Colección (para múltiples resultados).
- `all()`: Recupera todos los registros de la tabla
- `find($id)`: Busca un registro por su clave primaria
- `first()`: Devuelve el primer registro que coincida con los criterios 
- `get()`: Ejecuta la consulta y devuelve una colección de resultados
- `where()`: Filtra los registros por condiciones específicas
```php
// Ejemplo de consulta avanzada
$articles = Article::where('status', 'published')
                   ->orderBy('created_at', 'desc')
                   ->take(3)
                   ->get();
```
### `Update`
Al igual que en la creación, puedes actualizar registros de forma individual o masiva.
##### Actualización Individual
Recupera el modelo, modifica sus atributos y guarda los cambios.
```php
$user = User::find(1);
$user->email = 'new-email@example.com';
$user->save();
```
##### Actualización Masiva
Actualiza todos los registros que coincidan con una condición.
```php
User::where('active', false)->update(['status' => 'archived']);
```
### `Delete`
Para eliminar registros, puedes usar el método `delete` sobre una instancia o `destroy` si conoces la clave primaria.
- `$model->delete()`: Elimina la instancia actual
- `User::destroy(1)`: Elimina el registro con `ID: 1` sin necesidad de recuperarlo primero
- `User::destroy([1, 2, 3])`: Elimina múltiples registros por `ID`

## Conceptos Críticos de Ingeniería
##### `Mass Assignmet Vulnerability`
Es vital configurar `$fillable` o `$guarded`. Si pasas todo el input del usuario (`$request->all()`) directamente al método `create` sin protección, un atacante podría enviar campos inesperados (como `is_admin = true`) y modificar columnas sensibles.
##### `Query Builder` vs. `Eloquent`
`Eloquent` es ideal para operaciones `CRUD` básicas y lógica de negocio gracias a sus métodos legibles. Sin embargo, para consultas extremadamente complejas o procesamiento de grandes volúmenes de datos donde el rendimiento es crítico, el uso de `Raw SQL` o el `Query Builder` puro puede ser más eficiente.
