## Definición
Las relaciones en `Eloquent` son métodos definidos en tus clases de modelo. Debido a que las relaciones sirven como potentes constructores de consultas (`Query Builders`), definirlas como métodos proporciona capacidades robustas de encadenamiento y carga de datos.

### Relaciones Básicas
##### `One To One` (Uno a Uno)
Una relación básica donde un registro posee exactamente otro registro. Por ejemplo un `User` tiene un `Profile`.
- **Método**: `hasOne` (en el padre) y `belongsTo` (en el hijo).
- **Clave Foránea**: Por defecto, Laravel busca `user_id` en la tabla de perfiles
```php
// User.php
public function profile() {
    return $this->hasOne(Profile::class);
}

// Profile.php
public function user() {
    return $this->belongsTo(User::class);
}
```
##### `One To Many` (Uno a Muchos)
Es la relación más común. Un solo modelo es el padre de múltiples modelos hijos. Por ejemplo, un `Post` tiene muchos `Comments`.
- **Método**: `hasMany` (en el padre) y `belongsTo` (en el hijo)
```php
// Post.php
public function comments() {
    return $this->hasMany(Comment::class);
}
```
##### `Many to Many` (Muchos a Muchos)
Relaciones complejas donde muchos registros de una tabla se relacionan con muchos de otra. Requiere una **tabla pivote** (ej. `role_user`).
- **Método**: `belongsToMany`
- **Convención de tabla pivote**: Nombres de modelos en singular, orden alfabético, separados por guion bajo (`role_user`).
### Relaciones Avanzadas
| Tipo               | Propósito                                                                     | Ejemplo                               |
| :----------------- | ----------------------------------------------------------------------------- | ------------------------------------- |
| `Has One Through`  | Acceder a un registro lejano a través de una relación intermedia única.       | `User -> Mechanic -> CarOwner`        |
| `Has Many Through` | Acceder a muchos registros lejanos a través de un intermediario.              | `Country -> Users -> Posts`           |
| `Polymorphic`      | Un modelo puede pertenecer a más de un tipo de modelo en una sola asociación. | `Image` (pertenece a `User` o `Post`) |
### Rendimiento: `Eager Loading`
El mayor peligro al usar relaciones es el problema de las consultas `N+1`. Esto sucede cuando cargas 20 registros y luego, en un bucle, pides una relación de cada uno, disparando 21 consultas a la base de datos. **La solución**: usa el método `with()`
```php
// MAL (N+1): Genera 1 consulta para libros + N consultas para autores.
$books = Book::all(); 

// BIEN (Eager Loading): Genera solo 2 consultas en total.
$books = Book::with('author')->get();
```
### Tips de Ingeniería
- **`Dynamic Properties` vs `Methods`**:
	- `$user->posts` (propiedad): Devuelve una colección de resultados ya ejecutados.
	- `$user->posts()` (método): Devuelve una instancia de la relación, permitiéndote seguir filtrando: `$user->posts()->where('active', 1)->get()`.
- **`Constraining Member Styles`**: Puedes filtrar relaciones directamente al cargarlas: `User::with(['posts' => fn(query) => $query->where('title', 'like', '%Laravel%')])->get()`.
- **`Touching Timestamps`**: Si quieres que al actualizar un `Comment` se actualice la fecha `updated_at` del `Post` padre, usa la propiedad `protected $touches = ['post']` en el modelo hijo.
