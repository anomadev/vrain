### `Accessors` y `Mutators` (`Attributes`)
Los `Accessors` permiten transformar un valor de `Eloquent` cuando es accedido, mientras que los `Mutators` transforman el valor antes de que sea persistido en la base de datos.
##### La Sintaxis Moderna
Anteriormente se usaban métodos separados (`getFooAttribute`, `setFooAttribute`). El estándar actual utiliza la clase `Illuminate\Database\Eloquent\Casts\Attribute`.
```php
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    /**
     * Determina cómo se comporta el atributo 'first_name'.
     */
    protected function firstName(): Attribute
    {
        return Attribute::make(
            // Accessor: Se ejecuta al leer $user->first_name
            get: fn (string $value) => ucfirst($value),
            
            // Mutator: Se ejecuta al asignar $user->first_name = 'nombre'
            set: fn (string $value) => strtolower($value),
        );
    }
}
```
### `Attribute Casting`
El `Casting` proporciona una funcionalidad similar a los `Accessors`/`Mutators` pero de forma declarativa para tipos de datos comunes. Le indica a `Eloquent` cómo debe tratar una columna específica al convertirla de la base de datos a un tipo de PHP y viceversa.
##### `Casts` Comunes
- `boolean`: Convierte `0` o `1` en `false` o `true`
- `datetime` / `date`: Convierte `string` de fecha en objetos `Carbon`
- `array` / `json`: Convierte automáticamente `strings JSON` de la `DB` en `arrays` de `PHP`
- `encrypted`: Encripta el valor al guardarlo y lo desencripta al leerlo de forma transparente
- `hashed`: (Común para contraseñas) Aplica el algoritmo de `hashing` automáticamente
##### Implementación (`Laravel 11+`)
El Laravel `11` y `12`, se recomienda definir los castas mediante un método `casts()` en lugar de la propiedad `$casts`. lo que permite llamar a métodos estáticos y pasar argumentos.
```php
protected function casts(): array
{
    return [
        'is_admin' => 'boolean',
        'options' => 'array',
        'published_at' => 'datetime:Y-m-d',
        'password' => 'hashed',
    ];
}
```
### `Custom Casts`
Cuando la lógica de transformación es demasiado compleja para un simple `cast` o un `accessor`, puedes crear una clase de `Cast` personalizada. Esto es ideal para objetos de valor (`Value Objects`).
```bash
php artisan make:cast AddressCast
```
Esta clase implementará los métodos `get` y `set`, permitiendo, por ejemplo, convertir tres columnas de la `DB` (`calle`, `numero` y `ciudad`) en un solo objeto `Address` en PHP.

## Analogía
Imagina que `Eloquent` es un Traductor Universal.
- La Base de Datos habla un idioma rudo y simple (`strings`, enteros, ceros y unos).
- Tu Aplicación habla un idioma sofisticado (Objetos, Colecciones, Booleanos).
- El `Casting` es el diccionario que dice: "Cada vez que veas un 1, di Verdadero".
- Los `Accessors`/`Mutators` son el intérprete que dice: "Cada vez que me digan un nombre, lo escribiré en minúsculas en el registro, pero cuando te lo presente, te lo diré con la primera letra en mayúscula".

## Buenas Prácticas
- **Prioriza `Casting` sobre `Accessors`**: Si solo necesitas convertir un tipo (ej. `string` a `boolean`), usa siempre `casts()`. Es más rápido y estándar.
- **Evita `Side Effects`**: Un `Accessor` nunca debería modificar otros atributos o realizar consultas a la base de datos. Debe ser una función pura basada en el valor actual.
- **`Encrypted Data`**: Usa el `cast` `encrypted` para datos sensibles como `tokens` de `API` de terceros o información personal identificable (`PII`), pero recuerda que no podrás realizar búsquedas por valor (`WHERE`) sobre columnas encriptadas fácilmente.
- **`Casting` de `Enums`**: Laravel soporta nativamente el `casting` a `Enums` de PHP, lo que proporciona una seguridad de tipos (`Type Safety`) superior en toda la aplicación.
