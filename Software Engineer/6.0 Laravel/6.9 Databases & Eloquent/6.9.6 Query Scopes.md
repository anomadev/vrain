## Definición
Un `Query Scope` es un método que permite definir restricciones comunes de consulta que se pueden reutilizar en toda la aplicación. En lugar de repetir clausulas `where`, `orderBy` o `join` en múltiples controladores, se definen una vez en el modelo y se invocan como métodos encadenables en el `Query Builder`.

### `Local Scopes`
Son alcances que el desarrollador invoca manualmente cuando los necesita. Son ideales para filtros que se aplican frecuentemente pero no en todas las situaciones.
##### Definición y Uso
Para definir un `local scope`, el nombre del método debe comenzar con el prefijo `scope`. Al invocarlo, no es necesario incluir dicho prefijo.
```php
// app/Models/Post.php
public function scopePublished($query)
{
    return $query->where('status', 'published')
                 ->where('published_at', '<=', now());
}

// Uso en el controlador
$posts = Post::published()->orderBy('title')->get();
```
##### `Scopes Dinámicos`
Puedes pasar parámetros adicionales a tus `scopes` para hacerlos más flexibles.
```php
public function scopeOfType($query, string $type)
{
    return $query->where('type', $type);
}

// Uso
$videoPosts = Post::published()->ofType('video')->get();
```
### `Global Scopes`
Los `Global Scopes` se aplican automáticamente a todas las consultas de un modelo específico. Son fundamentales para implementar lógicas que deben ser transversales, como multi-tenencia o el filtrado de registros "borrados" (`Soft Deletes`).
##### Implementación mediante Clases
Es la forma más organizada. Requiere crear una clase que implemente la interfaz `Scope`.
```php
// app/Models/Scopes/ActiveScope.php
public function apply(Builder $builder, Model $model)
{
    $builder->where('active', true);
}

// En el modelo
protected static function booted(): void
{
    static::addGlobalScope(new ActiveScope);
}
```
##### `Scopes Globales` Anónimos
Para lógicas simples, se pueden definir directamente en el método `booted` del modelo.
```php
protected static function booted(): void
{
    static::addGlobalScope('ancient', function ($builder) {
        $builder->where('created_at', '<', now()->subYears(1));
    });
}
```
##### Eliminar `Scope Globales`
A veces es necesario realiza una consulta que ignore estas restricciones automáticas:
```php
$allUsers = User::withoutGlobalScope(ActiveScope::class)->get();
// O para todos los scopes
$allUsers = User::withoutGlobalScopes()->get();
```
### Decisiones de Arquitectura: Local o Global

| Escenario                                                 | Recomendación  |
| :-------------------------------------------------------- | -------------- |
| Filtros opcionales (ej. `populares`, `recientes`)         | `Local Scope`  |
| Restricciones de seguridad (ej. `pertenece_a_mi_empresa`) | `Global Scope` |
| Lógica de visibilidad (ej. `solo_registros_aprobados`)    | `Global Scope` |
| Filtros dinámicos basados en `input` del usuario          | `Local Scope`  |

## Analogía
Imagina que `Eloquent` es una **Cafetería**
- Un `Local Scope` es como un **Botón de Selección** (Espresso, Latte). Solo se aplica si tú presionas el botón específicamente. Tú decides cuándo quieres esa "restricción" de sabor.
- Un `Global Scope` es como el **Filtro de Agua** interno de la cafetera. Está ahí siempre, filtrando cada gota de café que sale, sin que tengas que recordarle que lo haga. Si por alguna razón necesitas agua sin filtrar, tendrías que "desarmar" o desactivar el filtro manualmente (`withoutGlobalScope`)

## Buenas Prácticas
1. **`Naming Semantico`**: Los nombres de los `scopes` deben ser verbos o adjetivos que describa un estado (`active`, `published`, `priority`), no la implementación técnica.
2. **Encadenamiento**: Asegúrate de que tus `local scopes` siempre retornen la instancia de `$query` para permitir el encadenamiento de métodos.
3. **No abusar de `Global Scopes`**: Un exceso de `scopes` globales puede hacer que las consultas sean difíciles de depurara, ya que añaden clausulas `WHERE` "invisibles" que pueden confundir a otros desarrolladores.
4. **`Unit Testing`**: Los `Query Scope` son excelentes candidatos para pruebas unitarias de modelos, asegurando que la lógica de filtrado sea correcta antes de llegar a los controladores.
