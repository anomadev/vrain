## Definición
El `Query Builder` es un servicio de Laravel que permite interactuar con la base de datos sin necesidad de definir modelos de `Eloquent`. Retorna resultados como objetos de clase estándar de PHP (`stdClass`), lo que reduce el consumo de memoria al evitar la hidratación de modelos pesados, siendo ideal para reportes, procesos masivos o consultas de solo lectura.

### Operaciones de Recuperación (`Read`)
El punto de entrada es el `facade` `DB::table()`.

| Método             | Propósito                                     | Resultado                             |
| :----------------- | --------------------------------------------- | ------------------------------------- |
| **`get()`**        | Recuperar todos los registros que coincidan   | `Collection` de objetos `stdClass`    |
| **`first()`**      | Recuperar solo el primer registro             | Objeto único o `null`                 |
| **`value('col')`** | Obtener un único valor de una sola columna    | Valor escalar (`string`, `int`, etc.) |
| **`find($id)`**    | Buscar un registro por su clave primaria `ID` | Objeto único                          |
| **`pluck('col')`** | Obtener una lista de valores de una columna   | `Collection` de valores               |
```php
use Illuminate\Support\Facades\DB;

$users = DB::table('users')->get();
$email = DB::table('users')->where('name', 'John')->value('email');
```
### Clausulas de Filtrado y Orden
El `Query Builder` permite encadenar métodos para construir la sentencia `WHERE` de forma dinámica.
- **`where`**: Filtro básico. Si se omiten los operadores, asume igualdad.
- **`orWhere`**: Añade una condición `OR`.
- **`whereIn`** / **`whereNotIn`**: Filtra por un `array` de valores.
- **`whereBetween`**: Filtra en un rango de valores.
- **`orderBy`** / **`latest`** / **`oldest`**: Manejo del orden de los resultados.
##### Subconsultas (`Sub-queries`)
Se pueden pasar funciones anónimas a las clausulas `where` para crear subconsultas complejas de manera elegante.
```php
$films = DB::table('films')
    ->where('title', 'like', 'K%')
    ->orWhere('title', 'like', 'Q%')
    ->whereIn('language_id', function ($query) {
        $query->select('language_id')
              ->from('languages')
              ->where('name', 'English');
    })
    ->orderBy('title')
    ->get();
```
### Agregaciones y `Joins`
El `Builder` maneja funciones de agregación directamente en la base de datos, lo cual es mucho más eficiente que procesar colecciones en PHP.
- **Agregados**: `count()`, `max()`, `min()`, `avg()`, `sum()`
- **`Joins`**: Permite combinar tablas de forma tradicional
```php
$users = DB::table('users')
    ->join('contacts', 'users.id', '=', 'contacts.user_id')
    ->select('users.*', 'contacts.phone')
    ->get();
```
### Escritura de Datos (CUD)
Aunque `Eloquent` es más común para esto, el `Query Builder` es extremadamente rápido para inserciones o actualizaciones masivas.
- **`Insert`**: `DB::table('users')->insert(['email' => 'john@example.com']);`
- **`Update`**: `DB::table('users')->where('id', 1)->update(['active' => true]);`
- **`Delete`**: `DB::table('users')->where('votes', '<', 100)->delete();`
### Depuración (`Debbuging`)
Para ver la consulta `SQL` que se está generando sin ejecutarla, se pueden usar métodos de inspección:
- **`toSql()`**: Devuelve el `string` del `SQL` (sin los valores `bindeados`)
- **`dd()`**: `"Dump and Die"`. Muestra el `SQL` y los parámetros, y detiene la ejecución
- **`dump()`**: Muestra la información pero continúa la ejecución

## Notas
- **Manejo de Transacciones**: Siempre usa `DB::transaction(function () { ... })` cuando realices múltiples operaciones de escritura dependientes para asegurar la integridad de los datos.
- **`Raw Expressions`**: Si necesitas una función de base de datos muy específica que el `Builder` no soporta, usa `DB::raw()`, pero ten cuidado extremo con la sanitización de los datos.
