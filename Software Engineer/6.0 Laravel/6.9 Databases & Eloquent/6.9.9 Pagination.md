## Definición
La paginación es el proceso de dividir un conjunto de resultados extenso en partes más pequeñas (páginas) para mejorar el rendimiento del servidor, reducir el tiempo de carga del navegador y ofrecer una mejor experiencia de usuario (UX). Laravel integra esto nativamente tanto en el `Query Builder` como en `Eloquent ORM`.

### Métodos de Paginación en el Servidor
Laravel ofrece tres formas principales de paginar datos, cada una adecuada para diferentes escenarios de rendimiento y diseño.
##### El método `paginate()`
Es el más común. Realiza dos consultas a la base de datos: una para contar el total de registros y otra para obtener los registros de la página actual.
- **Resultado**: Un objeto `LengthAwarePaginator`
- **Ventaja**: Permite mostrar enlaces a todas las páginas (1,2,3... 50) y el total de resultados
```php
// En el Controlador
$users = User::paginate(15);
// [00:00:44] Cambias ->get() por ->paginate(n)
```
##### El método `simplePaginate()`
Solo realiza una consulta para obtener los registros actuales `+1` (para saber si hay una página siguiente)
- **Resultado**: Un objeto `Paginator`
- **Ventaja**: Más eficiente que `paginate()` porque no cuenta el total de registros
- **Uso ideal**: Enlaces de "Anterior" y "Siguiente" únicamente
##### El método `cursorPaginate()`
Utiliza un "puntero" o cursor basado en una columna ordenada (usualmente `ID` o fecha) en lugar de desplazamiento (`offset`).
- **Ventaja**: Rendimiento constante independientemente de si estás en la página 1 o en la 1000. Proviene problemas de registros duplicados si se insertan datos mientras el usuario navega
- **Uso Ideal**: `Scroll` infinito o conjunto de datos masivos
### Renderizado en el `Frontend (Blade)`
Una vez que pasa los resultados paginados a la vista, mostrar los controles de navegación es extremadamente sencillo.
```html
{{-- resources/views/users/index.blade.php --}}
<div class="container">
    @foreach ($users as $user)
        {{ $user->name }}
    @endforeach
</div>

{{-- Genera automáticamente el HTML de la navegación --}}
{{ $users->links() }}
```
Laravel inyecta dinámicamente todo el `template` y la funcionalidad necesaria.
### Configuración y Estilos
Por defecto, Laravel utiliza `Tailwind CSS` para los estilos de paginación, pero puedes cambiarlo globalmente si usas otros `frameworks` como `Bootstrap`.
```php
// app/Providers/AppServiceProvider.php
use Illuminate\Pagination\Paginator;

public function boot(): void
{
    Paginator::useBootstrapFive();
}
```
### Paginación en `APIs` (`JSON`)
Al devolver un `paginator` desde una ruta de `API` o un controlador, Laravel lo convierte automáticamente en una estructura `JSON` que incluye metadatos útiles para el `frontend` (como `Vue` o `React`).
```json
{
   "data": [ ... ],
   "links":{
       "first": "http://example.com/api/users?page=1",
       "last": "http://example.com/api/users?page=50",
       "prev": null,
       "next": "http://example.com/api/users?page=2"
   },
   "meta":{
       "current_page": 1,
       "from": 1,
       "last_page": 50,
       "path": "http://example.com/api/users",
       "per_page": 15,
       "to": 15,
       "total": 750
   }
}
```

## Buenas Prácticas
1. **Limitar columnas**: Si solo necesitas el nombre y el email, usa `User::select('id', 'name', 'email')->paginate(15)` para reducir el uso de memoria.
2. **Evitar el `"Deep Paging"` con `offset`**: En tablas con millones de filas, ir a la página 10,000 con `paginate()` es muy lento. Usa `cursorPaginate()` en estos casos.
3. **Preservar filtros**: Si el usuario está buscando algo, asegúrate de mantener los parámetros de búsqueda al cambiar de página: `{{ $users->appends(request()->query())->links() }}`.
4. **Carga Diferida (`Lazy Loading`)**: Siempre usa `Eager Loading` (`with()`) junto con la paginación para evitar el problema de las consultas `N+1` en cada página cargada.
