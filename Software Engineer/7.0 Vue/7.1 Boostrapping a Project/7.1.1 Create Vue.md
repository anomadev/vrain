## Definición 
Es el proceso inicial de andamiaje (`scaffolding`) de una aplicación `Vue` moderna utilizando la herramienta oficial `create-vue`. A diferencia del antiguo `Vue CLI`, este se basa en `Vite`, un `build tool` de próxima generación que ofrece una experiencia de desarrollo extremadamente rápida gracias al uso de `Native ESM modules`.

## Conceptos Clave
- **`Vite`**: El motor que maneja el HMR (`Hot Module Replacement`) y el empaquetado.
- **`Single File Components (SFC)`**: Archivos `.vue` que encapsulan `Logic` (`<script>`), `Template` (`<template>`) y `Styles` (`<style>`).
- **`Progressive Enhancement`**: La capacidad de empezar con una configuración mínima e ir añadiendo herramientas (`Router`, `Pinia`, etc.) según la escala del proyecto.
- **`Build-step vs. No-build`**: La distinción entre usar `Vue` mediante un CDN o mediante un proceso de compilación necesario para aplicaciones de grado profesional.

## Ejemplos Prácticos
Inicialización del proyecto.
```bash
# Execute the latest scaffolding tool
npm create vue@latest

# Recommended selections for Senior Architecture:
# - Project Name: stencil-core (or any specific project)
# - Add TypeScript? Yes (Standard for scalability)
# - Add Vue Router? Yes (SPA requirement)
# - Add Pinia? Yes (State management)
# - Add Vitest? Yes (Unit testing)
# - Add ESLint/Prettier? Yes (Code quality)
```
Estructura de carpetas recomendada (`Clean Architecture Ready`)
```
src/
├── assets/          # Static assets (global styles, images)
├── components/      # Shared/Presentational components
├── composables/     # Reusable business logic (Composition API)
├── router/          # Navigation configuration
├── stores/          # Pinia state definitions
├── views/           # Page-level components
├── App.vue          # Root component
└── main.ts          # Application entry point
```

## Analogía
Imagina que vas a construir una casa. `create-vue` no es solo el camión que trae los ladrillos; es un **kit de construcción prefabricado de alta tecnología**. En lugar de forjar tus propios clavos (configurar `Webpack` manualmente), el kit te entrega las paredes ya estructuradas con los ductos de electricidad (`Router`) y tuberías (`Pinia`) listos para conectar, permitiéndote enfocarte en el diseño y la funcionalidad del hogar.

## Buenas Prácticas
- `Strict TypeScript`: Mantén siempre `strict: true` en el `tsconfig.json`. Para proyectos que apuntan a mercados competitivos como Australia o Nueva Zelanda, el código tipado no es opcional, es el estándar de calidad.
- `Environment Variables`: Nunca guardes credenciales en el código, Usa archivos `.env` y accede a ellos mediante `import.meta.env`.
- `Linting on Commit`: Configura `Husky` o `lint-staged` para asegurar que ningún código sin formato llegue al repositorio.
- `Dependency Management`: Prefiere `pnpm` sobre `npm` para ahorrar espacio en disco y acelerar la instalación de dependencias en proyectos múltiples.

## Escenarios y Aplicaciones 
- `Greenfield Projects`: Es el punto de partida obligatorio para cualquier aplicación nueva.
- `Migration`: Cuando necesitas mover una lógica `legacy` de Vue 2 (`Options API`) a una arquitectura moderna de Vue 3 (`Composition API`).
- `PoC (Proof of Concept)`: Ideal para prototipar rápidamente una idea de negocio y validar la viabilidad técnica antes de escalar.

## Notas Personales
- `Vite Cold Start vs Hot Reload`: Es increíblemente rápido, pero ten cuidado con las dependencias muy pesadas que no son `ESM` compatibles; `Vite` tendrá que pre-empaquetar usando `esbuild`.
- `Tooling`: No olvides instalar la extensión `Vue - Official` en `VS Code`. Sin ella, el soporte de `TypeScript` dentro de los archivos `.vue` será deficiente.
- `Architecture Tip`: Aunque `create-vue` sugiere una carpeta `views/` y `components/`, para proyectos complejos suelo aplicar una estructura por módulos o capas para evitar que la carpeta `components/` se convierta en un vertedero.
