## Definición
Los `Components` son la unidad fundamental de una aplicación `Vue`. Son instancias de `Vue` reutilizables con un nombre definido, que encapsulan su propia estructura (`HTML`), lógica (`JS/TS`) y estilo (`CSS`). En una arquitectura limpia, actúan como los "átomos" o "moléculas" que conforman la interfaz de usuario, permitiendo una separación de preocupaciones (*`Separation of Concerns`*)clara y mantenible.

## Conceptos Clave
- `SFC (Single File Components)`: Es estándar de la industria. Archivos `.vue` que contienen `<script setup>`, `<template>` y `<style scoped>`.
- `Props`: El mecanismo para pasar datos de un componente padre a un hijo (Flujo de datos hacia abajo).
- `Emits`: La forma en que un componente hijo comunica eventos al padre (Flujo de datos hacia arriba).
- `Slots`: Espacios reservados para distribuir contenido dentro de un componente, permitiendo una composición flexible.
- `Component Registration`: La distinción entre registro **Global** (disponible en toda la app) y **Local** (importado explícitamente donde se necesita).

## Ejemplos Prácticos
##### Child Component con Props y Emits (`BaseButton.vue`)
```typescript
<script setup lang="ts">
// Definig props with Typescript interfaces for better DX
interface Props {
	label: string;
	variant?: 'primary' | 'secondary';
}

const props = withDefaults(defineProps<Props>(), {
	variant: 'primary'
});

// Definig emits
const emit = defineEmits<{
	(e: 'click', id: number): void;
}>();

const handleClick = () => {
	emit('click', Date.now());
};
</script>

<template>
	<button :class="['btn', props.variant]" @click="handleClick">
		{{ props.label }}
	</button>
</template>

<style scoped>
.btn { padding: 0.5rem 1rem }
.primary { background: blue; color: white; }
</style>
```
##### Local Registration en un Parent Component
```typescript
<script setup lang="ts">
import BaseButton from './components/BaseButton.vue';

const handleAction = (id: number) => {
	console.log(`Action triggered with ID: ${id}`);
};
</script>

<template>
	<main>
		<BaseButton label="Save Changes" variant="primary" @click="handleAction" />
	</main>
</template>
```

## Analogía
Imagina que los componentes son piezas de LEGO personalizadas. Cada pieza tiene una forma especifica y conectores (`Props` y `Emits`). No necesitas saber cómo se fabricó la pieza por dentro; solo necesitas saber qué conectores tiene para unirla con otras y construir estructuras complejas como un castillo.

## Buenas Prácticas
- `Local Registration > Global Registration`: Evita registrar todo globalmente. El registro local permite un mejor *`Tree-shaking`* (`Vite` eliminará el código que no uses) y facilita el seguimiento de dependencias.
- `PascalCase` para Nombres: Usa `MyComponent.vue` en lugar de `mycomponent.vue`. En el `template`, se recomienda usar `PascalCase` (`<MyComponent />`) para distinguirlos de los elementos `HTML` nativos.
- `Props Read-Only`:  Nunca intentes mutar una `Prop` dentro del componente hijo. Sigue el principio de `One-Way Data Flow`. Si necesitas cambiarlo, emite un evento al padre.
- `Scoped Styles`: Usa siempre `<style scoped>` para evitar que el `CSS` de un componente "se filtre" y afecte a otros elementos de la aplicación.

## Escenarios y Aplicaciones
- `UI Libraries`: Creación de botones, inputs y modales reutilizables en todo el proyecto.
- `Feature Modules`: Dividir una página compleja en sub-componentes más pequeños y testeables.
- `Layouts`: Estructuras repetitivas como `Navigation Bars` o `Sidebars` que envuelven el contenido principal. 

## Notas Personales
- `Over-componentization`: Cuidado con crear componentes para todo. Si un pedazo de código no se va a reutilizar y no es complejo, mantenerlo en el mismo archivo puede reducir la carga cognitiva.
- `Naming is hard`: Para componentes globales de base, suelo usar prefijos como `Base`, `App` o `V` (ej. `BaseInput.vue`). Esto los agrupa alfabéticamente en la carpeta `components/`.
- `Volar/Vue-Official`: Asegúrate de que los nombres de tus componentes coincidan con el nombre del archivo para que las herramientas de autocompletado en `VS Code` funcionen a la perfección.
