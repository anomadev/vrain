## Definición
El `Attribute Inheritance` ocurre cuando se pasan atributos a un componente que no han sido declarados explícitamente como `props` o `emits` (ej. `class`, `style`, `id` o eventos nativos como `@click`). Estos se denominan `Fallthrough Attributes` y, por defecto, `Vue` los aplica automáticamente al elemento raíz (*`root element`*) del componente hijo.

## Conceptos Clave
- **`Fallthrough Attributes`**: Atributos y oyentes de eventos que se "pasan de largo" porque el componente no los reclama.
- **`Root Element Inheritance`**: Comportamiento por defecto donde el primer elemento del `template` recibe los atributos.
- **`Class & Style Merging`**: A diferencia de otros atributos, si el hijo ya tiene clases o estilos, los del padre se **fusionan** en lugar de sobrescribirse.
- `inheritAttrs: false`: Opción para desactivar el comportamiento automático, permitiendo un control granular.
- `$attrs`: Objeto que contiene todos los atributos que han "caído" al componente. Se usa para `bindearlos` manualmente.

## Ejemplos Prácticos
##### Comportamiento por defecto
Si tenemos un `BaseButton.vue`
```html
<template>
	<button class="btn-primary">Click me</button>
</template>
```
Al usarlo en el padre: `<BaseButton class="large" id="main-btn" />`, el resultado renderizado será:
```html
<button class="btn-primary large" id="main-btn">Click me</button>
```
##### Desactivar Herencia y Control Manual (Avanzado)
Ideal para componentes tipo "`Wrapper`" donde el elemento raíz es un `div` pero quieres que los atributos vayan al `input`.
```html
<script setup lang="ts">
import { useAttrs } from 'vue';

// Disable automatic inheritance
defineOptions({
	inheritAttrs: false
});

// Access attributes in logic if needed
const attrs = useAttrs();
console.log('Inherited attributes', attrs);
</script>

<template>
	<div class="input-wrapper">
		<label>User Field</label>
		<input v-bind="$attrs" class="inner-input" />
	</div>
</template>
```

## Analogía
Imagina que pides un **paquete por Amazon** (Atributos) para alguien que vive en un edificio (Componente). El portero (Elemento Raíz) recibe el paquete por defecto. Sin embargo, si el portero tiene instrucciones de "No recibir paquetes" (`inheritAttrs: false`), él simplemente te deja pasar para que tú mismo le entregues el paquete directamente a la persona correcta en el apartamento 301 (`v-bind="$attrs"`).

## Buenas Prácticas
- **`Single Root Requeriments`**: La herencia automática solo funciona si el componente tiene un único elemento raíz. Si hay múltiples raíces, `Vue` no sabe a quién dárselos y arrojará un `warning` a menos que uses `v-bind="$attrs"` explícitamente.
- **Usa `inheritAttrs: false` para UI Kits**: Al construir bibliotecas de componentes, suele ser mejor desactivar la herencia automática para evitar que atributos como `id` o `aria-label` se quedan en un `div` contenedor en lugar del elemento interactivo real.
- **`Avoid Logic in $attrs`**: Usa `$attrs` principalmente para pasar atributos de presentación o accesibilidad. Si necesitas lógica de negocio, usa `props`.

## Escenarios y Aplicaciones
- **`Wrapper Components`**: Si envuelves una librería de gráficas, querrás que los atributos de tamaño pasen directamente al contenedor de la gráfica.
- **`Form Validation`**: Pasar atributos como  `required`, `placeholder` o `maxlength` desde el padre hasta un `input` anidado.
- **`Accessibility (A11y)`**: Asegurar que atributos `aria-*` lleguen al elemento semántico correcto.

## Notas Personales
- Los oyentes de eventos nativos (ej. `@click`) también están en `$attrs`. Si los `bindeas` manualmente a un elemento interno, el evento se disparará desde ahí.
- Al crear `inputs` personalizados, desactivar la herencia automática te permite mantener el `DOM` limpio, evitando que atributos "sucios" se queden en contenedores decorativos.
- Si tienes un componente con dos `div` al mismo nivel, debes elegir uno para ponerle `v-bind="$attrs"`, de lo contrario, los atributos pasados se perderán.
