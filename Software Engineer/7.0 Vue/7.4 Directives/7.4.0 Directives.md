## Definición
Las `Directives` son atributos especiales con el prefijo `v-` que se añaden al `HTML` para indicarle a `Vue` que aplique un comportamiento reactivo o una transformación específica al elemento `DOM`. Son el puente principal entre tu lógica de `JavaScript` y la representación visual, permitiendo manipular el `DOM` de forma declarativa sin tocarlo directamente (imperativamente).

## Conceptos Clave
- **`Arguments (:)`**: Indican qué propiedad o evento del elemento estamos afectando (ej. `:href` o `@click`).
- **`Modifiers (.)`**: Sufijos que aplican comportamientos especiales (ej. `.prevent`, `.stop`, `.capture`).
- **`Conditional Rendering`**: `v-f`, `v-else-if`, `v-else` y `v-show`.
- **`List Rendering`**: `v-for`, siempre acompañado de un `:key` único.
- **`Two-Way Binding`**: `v-model` para sincronizar `inputs` con el estado.
- **`One-Time Rendering`**: `v-once` y `v-memo` para optimizaciones de rendimiento.

## Ejemplos Prácticos
##### `Conditional Rendering: v-if vs v-show`
```html
<script setup lang="ts">
const isVisible = ref<boolean>(true);
const userRole = ref<'admin' | 'user' | 'guest'>('admin');
</script>

<template>
  <div v-if="userRole === 'admin'">
    Welcome, Admin.
  </div>
  <div v-else-if="userRole === 'user'">
    Welcome, User.
  </div>
  <div v-else>
    Access Denied.
  </div>

  <div v-show="isVisible">
    This is toggled frequently.
  </div>
</template>
```
##### `List Rendering with v-for`
```html
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.name }}
    </li>
  </ul>
</template>

<script setup lang="ts">
const items = ref([{ id: 1, name: 'Item A' }, { id: 2, name: 'Item B' }]);
</script>
```

## Analogía
Imagina que las directivas son etiquetas inteligentes que pegas en los objetos de tu oficina. La etiqueta `v-if="es_invierno"` en un ventilador dice: "Si no es invierno, el ventilador existe; si lo es, desmóntalo y guárdalo". La etiqueta `v-show="esta_encendido"` en una lámpara dice: "La lámpara siempre está ahí, pero si no está encendida, simplemente apaga la luz (`display: none`)".

## Buenas Prácticas
- **`v-if` vs `v-for`**: Nunca los uses en el mismo elemento. `Vue` prioriza `v-if` antes que `v-for`, lo que puede llevar a errores de lógica o performance. Si necesitas filtrar una lista, usa una `computed property`.
- **`The Key is Sacred`**: En `v-for`, nunca uses el `index` como `:key` si la lista es dinámica (filtros, ordenamiento). Usa `IDs` reales de la base de datos para evitar bugs de renderizado y pérdida de estado en componentes.
- **`v-show for Frequent Toggles`**: Si un elemento cambia la visibilidad muchas veces por segundo (ej. un `tooltip` o un menú), usa `v-show` para evitar el costo de destruir y recrear nodos del `DOM`.
- **`v-once for Static Content`**: Si tienes un bloque de contenido que nunca cambiará tras el renderizado inicial, usa `v-once` para que `Vue` lo ignore en futuros ciclos de actualización, ahorrando memoria.

## Escenarios y Aplicaciones
- **`Dashboard Logic`**: Usar `v-if` para mostrar diferentes secciones según los permisos del usuario.
- **`Form Handling`**: `v-model` para capturar datos.
- **`Dynamic Menus`**: `v-for` para renderizar elementos de navegación basados en un objeto de configuración.

## Notas Personales
- Ojo con `v-html`. Aparte del riesgo de `XSS`, abusar de él impide que `Vue` optimice ese pedazo del `DOM`.
- Aprende a crear `Custom Directives` (`v-focus`, `v-scroll`). Son excelentes para encapsular lógica que interactúa directamente con el `DOM` (como librerías de terceros o `analytics`) sin ensuciar tus componentes con `refs` manuales.
- Investiga `v-memo`. En listas gigantes (`1000+ items`), permite saltarse las actualizaciones de sub-árboles enteros si ciertas dependencias no han cambiado.
