## Definición
La directiva `v-html` se utiliza para actualizar el `innerHTML` de un elemento con una cadena de texto que contiene `HTML` real. A diferencia de la interpolación de bigotes `{{ }}` o `v-text`, que escapan el contenido para tratarlo como texto plano, `v-html` renderiza las etiquetas como nodos reales `DOM`.

> [!Warning]
> Renderizar `HTML` dinámico de forma arbitraria ex extremadamente peligroso porque puede conducir fácilmente a ataques de `XSS (Cross-Site Scripting)`.

## Conceptos Clave
- **`innerHTML Binding`**: Funciona exactamente como la propiedad nativa de `JavaScript`, reemplazando todo el contenido del elemento.
- **`XSS Risk`**: Si el contenido proviene de un usuario o una fuente no confiable, un atacante podría inyectar scripts maliciosos.
- **`Scoped CSS Limitation`**: Los estilos definidos con `<style scoped>` no se aplican al contenido insertado mediante `v-html`, ya que `Vue` no procesa ese `HTML` dinámico para añadirle los atributos de `scope (data-v-xxxx)`.
- **`Virtual DOM Bypass`**: Al usar `v-html`, te saltas parte de la optimización del `Virtual DOM` de `Vue` para ese nodo específico.

## Ejemplos Prácticos
##### Uso básico (Contenido confiable)
```html
<script setup lang="ts">
// Content from a trusted source (e.g., a secure CMS)
const trustedMarkup = ref<string>(
  '<p>This is a <strong>formatted</strong> description.</p>'
);
</script>

<template>
  <div v-html="trustedMarkup"></div>
</template>
```
##### Estilizando contenido de `v-html (Deep Selector)`
```html
<style scoped>
/* Scoped styles won't work normally on v-html content. 
   We must use the :deep() pseudo-selector. */
.content-wrapper :deep(p) {
  color: #2c3e50;
  line-height: 1.6;
}
</style>

<template>
  <div class="content-wrapper" v-html="trustedMarkup"></div>
</template>
```

## Analogía
Usar interpolación normal `{{ }}` es como recibir un **paquete sellado**: puedes ver lo que dice la etiqueta, pero no puedes abrirlo. Usar `v-html` es como recibir una llave maestra de tu casa y dársela a un mensajero. Si el mensajero es de confianza, abrirá la puerta y acomodará los muebles (renderiza el `HTML`). Si el mensajero es un extraño (`input` de usuario), podría entrar y robarse todo (ejecutar un `script` malicioso).

## Buenas Prácticas
- **`Sanitize First`**: Si absolutamente debes renderizar `HTML` de terceros, pásalo primero por una librería de sanitización como `DOMPurity`.
- **`Avoid on User Inputs`**: Nunca, bajo ninguna circunstancia, uses `v-html` con datos de un usuario que haya escrito directamente.
- **`Use Slots instead`**: En la mayoría de los casos, lo que intentas resolver con `v-html` se puede lograr de forma más segura y estructurada usando `Slots` o componentes funcionales.
- **`Tailwind / Global Styles`**: Dado que el `CSS scoped` no funciona, usa clases globales o utilidades de `Tailwind` si el contenido inyectado necesita diseño. 

## Escenarios y Aplicaciones
- **`CMS Integration`**: Renderizar el contenido de un post de blog o una página de ayuda que viene de una base de datos administrada.
- **`Legacy Systems`**: Integrar pedazos de código de aplicaciones antiguas que devuelven fragmentos de `HTML` pre-construidos.
- **`Markdown Previewers`**: Renderizar la salida de un `parser` de `Markdown` (después de sanitizarla).

## Notas Personales
- Un error común es intentar anidar componentes de `Vue` dentro del `string` de `v-html`. No funcionará. `Vue` trata el contenido como `HTML` estático nativo; no activará directivas ni montará componentes que estén dentro del `string`.
- El uso de `v-html` suele ser un "`smell`" de que la arquitectura de componentes podría estar fallando. Pregúntate siempre: "¿Puedo hacer esto con un componente recursivo o con `slots`?".
